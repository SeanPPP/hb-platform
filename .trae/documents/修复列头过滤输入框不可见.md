## 问题定位
- 组件已使用 `headerRenderer` 与 `headerRowHeight={56}`，但输入框仍不可见，通常由以下原因导致：
  1) 未引入 react-data-grid 的官方样式，表头行高与单元格布局未生效；
  2) 所用版本不支持 `headerRowHeight` 属性，需要改用 CSS 变量 `--rdg-header-row-height`；
  3) 表头单元格的 overflow 被裁剪，需强制可见；
  4) 冻结列与容器样式冲突，表头内容被压缩。

## 修复方案
1) 引入样式
- 在应用入口（如 `src/app.tsx` 或 `src/main.tsx`）引入：`import 'react-data-grid/lib/styles.css'`

2) 强制表头行高
- 在 DataGrid 的 `style` 中设置 CSS 变量：`style={{ height: '100%', ['--rdg-header-row-height' as any]: '56px' }}`
- 同时保留 `headerRowHeight={56}`，双重保障。

3) 表头渲染安全样式
- 在 `headerRenderer` 容器上增加 `minHeight: 48` 与 `overflow: 'visible'`，确保输入框不被压缩或裁剪。

4) 兜底方案（若仍不可见）
- 在表格上方渲染一行“外部过滤栏”（对应列的输入框），沿用当前过滤逻辑 `filters`，保证用户可用；图片/序号不显示。

## 验证
- 打开明细页，列头下方应显示输入框；输入关键字或数值区间即时过滤；点击列头正常排序。
- 冻结列与选择列显示正常，右侧操作列仍位于末尾。
