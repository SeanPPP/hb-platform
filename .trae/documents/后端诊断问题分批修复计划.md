## 问题分类与影响
- 空引用风险：在多处 `.Contains`、`.Select`、`FirstOrDefault`、属性/集合访问时参数或源可能为 null，产生运行时异常风险（示例：ProductService.cs:67、368，UserService.cs:128-129，CartMappingProfile.cs:49）。
- 可空类型与泛型约束不匹配：将 `string?` 作为 `Dictionary<TKey,...>` 的键或 `ToDictionary` 的键（需要 `notnull`），触发编译器警告（示例：ProductSyncService.cs:250，WarehouseProductBatchService.cs:169-171，ProductSetCodeReactService.cs:423）。
- 异步方法未使用 await：多个方法标记为 async，但内部未 await，逻辑可能同步运行且异常传播不符合预期（示例：RoleService.cs:998/1016/1057，StoreSyncService.cs:125/252，WarehouseProductBatchService.cs:699/709，ContainerReactService.cs:324）。
- 条件表达式恒真/恒假：与 `bool?` 的比较导致恒真/恒假（示例：WarehouseCategoryReactService.cs:119，WarehouseCategoryService.cs:163）。
- 其他：格式化提示（LocalSupplierInvoicesReactService.cs:1009），可容忍但可顺手修复。

## 修复策略（按优先级）
1) 空值防护与集合过滤
- 在所有容易为 null 的源、参数处添加防护：
  - 在 `.Contains(value)` 前先做 `!string.IsNullOrWhiteSpace(value)` 或 `value != null`；对集合源 `list?.Contains(x) == true`。
  - 对 LINQ 源与键：`Where(x != null)` 后再 `.Select(x => x!)`，避免 `string?` 键进入 `ToDictionary`。
  - 访问集合或属性前先判空：`if (obj?.Prop != null) {...}` 或 `obj?.Prop ?? defaultValue`。
- 受影响文件与位置：
  - ProductService.cs:67、368；ProductSyncService.cs:47、246、366、250（ToDictionary 键）；UserService.cs:128-129；CartMappingProfile.cs:49；DomesticSupplierService.cs:42、43、272、279；ProductLocationService.cs:74；StoreMultiCodePricesReactService.cs:528；WarehouseCategoryReactService.cs:284、298；HqProductTranslationReactService.cs:37；WarehouseProductBatchService.cs:165。

2) 修复可空键与 ToDictionary 使用
- 模式：`source.Where(k => !string.IsNullOrEmpty(k)).ToDictionary(k => k!, v => ...)`；若键来自对象属性，先 `Select` 再过滤。
- 受影响文件：ProductSyncService.cs:250；WarehouseProductBatchService.cs:169-171；ProductSetCodeReactService.cs:423。

3) 异步方法语义一致化
- 对标记为 async 的方法，确保内部调用使用 `await`；若无异步调用，移除 async 并返回非 Task，或包裹为 `await Task.Run(...)`（CPU 密集型）。
- 受影响文件：RoleService.cs:998/1016/1057；StoreSyncService.cs:125/252；WarehouseProductBatchService.cs:699/709；ContainerReactService.cs:324。

4) 纠正恒真/恒假条件
- 使用显式判空：`if (flag == true)` 或 `(flag ?? false)`，避免与 `null` 比较。
- 受影响文件：WarehouseCategoryReactService.cs:119；WarehouseCategoryService.cs:163。

5) 代码格式与微调
- LocalSupplierInvoicesReactService.cs:1009 格式化插入换行；顺手统一字符串/空值处理风格。
