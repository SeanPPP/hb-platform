## 添加门店供应商销售统计功能到前端

### 1. 后端 API 控制器修改

**在 `StatisticsJobTriggerController.cs` 中添加：**
- `POST /api/StatisticsJobTrigger/trigger-store-supplier` 端点，调用 `UpdateStoreSupplierStatistics`
- `POST /api/StatisticsJobTrigger/batch-update-store-supplier` 端点，调用 `BatchUpdateStoreSupplierStatistics`
- 新增 `StoreSupplierJobTriggerRequest` DTO 类
- 新增 `BatchStoreSupplierUpdateRequest` DTO 类

### 2. 前端服务文件修改

**在 `statisticsJob.ts` 中添加：**
- `StoreSupplierJobTriggerRequest` 接口
- `BatchStoreSupplierUpdateRequest` 接口
- `triggerStoreSupplierStatistics()` 函数
- `batchUpdateStoreSupplierStatistics()` 函数

### 3. 前端页面修改

**在 `StatisticsJobTrigger/index.tsx` 中添加：**
- 导入新的服务函数
- 扩展 `JobType` 类型，添加 `'store-supplier'` 和 `'batch-store-supplier'`
- 在 `jobTypeOptions` 中添加两个新选项：
  - "门店供应商统计"
  - "批量更新门店供应商统计"
- 在 `handleTrigger` 函数中添加对应的 case 处理
- 在任务说明 Alert 中添加新的 Tag 说明
- 添加条件渲染逻辑，支持新的任务类型（显示分店和供应商选择器）

### 4. 功能特性

- 单日触发：支持指定日期、分店列表、供应商列表
- 批量触发：支持日期范围、分店列表、供应商列表
- 支持按分店和供应商筛选统计
- 错误处理和成功消息提示
