## 原因分析
- 即使后端返回了数据，前端仍可能取不到 `Items/Total`，常见原因：
  - ASP.NET Core 默认 JSON 命名策略为 camelCase，控制器匿名对象属性 `Items/Total` 在序列化后可能变成 `items/total`，导致前端仍读不到大写键。
  - 列表 DTO 属性在 JSON 中若为 `StoreName/ProductName...`（PascalCase），而前端列绑定使用小驼峰 `storeName/productName`，会导致 Handsontable 全列空白。
  - 前端未对返回结构做容错，`loadData` 仅读取大写键。位置：`ReactUmi/my-app/src/pages/PosAdmin/StoreRetailPrices/index.tsx:164-166`。

## 修复方案（前端容错，推荐）
- 在页面 `loadData` 中同时兼容大小写键：
  - `const d = res.data?.data; const items = d?.Items ?? d?.items ?? []; const totalVal = d?.Total ?? d?.total ?? 0;`。
- 若发现列表项属性是 PascalCase（如 `StoreName`），增加一次规范化映射为小驼峰：
  - `const norm = (obj) => ({ uuid: obj.UUID || obj.uuid, storeName: obj.StoreName || obj.storeName, supplierName: obj.SupplierName || obj.supplierName, productName: obj.ProductName || obj.productName, itemNumber: obj.ItemNumber || obj.itemNumber, barcode: obj.Barcode || obj.barcode, purchasePrice: obj.PurchasePrice ?? obj.purchasePrice, storeRetailPriceValue: obj.StoreRetailPriceValue ?? obj.storeRetailPriceValue, discountRate: obj.DiscountRate ?? obj.discountRate, isActive: obj.IsActive ?? obj.isActive, isAutoPricing: obj.IsAutoPricing ?? obj.isAutoPricing, isSpecialProduct: obj.IsSpecialProduct ?? obj.isSpecialProduct, updatedBy: obj.UpdatedBy || obj.updatedBy, updatedAt: obj.UpdatedAt || obj.updatedAt, productCode: obj.ProductCode || obj.productCode })`，然后 `setData(items.map(norm))`。
- 增加一次调试输出便于确认：在 `loadData` 里 `console.log('grid payload', res.data)`。

## 备用方案（后端统一为保留大小写，谨慎）
- 在 `Program.cs` 设置 `options.JsonSerializerOptions.PropertyNamingPolicy = null;`，让返回的 `Items/Total` 以及 DTO 属性保留原大小写。但这会影响整个 API 的命名一致性，不建议在已有前端依赖 camelCase 的情况下采用。

## 验证步骤
- 打开页面点击查询：确认 `items.length > 0` 且表格显示列值（如 `storeName/productName`）。
- 切换排序、筛选与分页（`updatedAt` 降序，`storeName` 升序，关键词过滤），确认数据正常变化。
