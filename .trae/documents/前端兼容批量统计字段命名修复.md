## 原因分析
- 后端在 `/Services/React/StoreRetailPriceReactService.cs:728–736` 返回 `BatchResultDto`，服务端序列化为 JSON 时字段很可能采用 camelCase（`inserted/updated/failed/errors`）。
- 前端在 `index.tsx` 提示统计使用的是 PascalCase（`stat?.Inserted/Updated/Failed`），当收到 camelCase 时这三项为 `undefined`，提示回退为 0，出现“Updated 是 1，但前端显示为 0”。
- 佐证：前端在网格接口已做过大小写兼容（`Items/Total` 与 `items/total`），但在批量保存统计未做同样兼容。

## 修复方案（前端）
- 在 `ReactUmi/my-app/src/pages/PosAdmin/StoreRetailPrices/index.tsx` 保存成功后构建提示时，兼容两种命名：
  - `const inserted = stat?.Inserted ?? stat?.inserted ?? 0`
  - `const updated = stat?.Updated ?? stat?.updated ?? 0`
  - `const failed = stat?.Failed ?? stat?.failed ?? 0`
- 同时兼容 `res.data?.data` 与 `res.data` 两种包装，防止拦截器或控制器包装差异导致读取不到 `stat`。
- 保留现有对增量刷新与折扣率百分比转换的修复，不再更改后端序列化策略。
