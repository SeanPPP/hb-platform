## 问题概述
- 前端：`src/services/localSupplierInvoiceService.ts:15` 报错，原因是 `AxiosRequestConfig` 中未声明 `skipBizReject`。现有写法 `skipBizReject: true as any` 仍触发“未知属性”校验。
- 后端：大量可空性与异步用法警告，属于潜在运行时风险，需要分批治理。

## 前端修复计划
1. 通过类型声明合并为 `AxiosRequestConfig` 增加可选字段：
   - 新增文件 `src/types/axios.extend.d.ts`，内容：
     - `declare module 'axios' { export interface AxiosRequestConfig { skipBizReject?: boolean } }`
   - 保持拦截器兼容逻辑（`src/utils/request.ts`）不变，使用类型安全的 `config.skipBizReject`。
2. 移除局部的 `as any` 变通写法，保留调用处：
   - `src/services/localSupplierInvoiceService.ts:12-16` 的第二参数维持 `{ method: 'POST', data, skipBizReject: true }`。
3. 重新运行诊断并确认无 TypeScript 错误。

## 后端治理计划（分批）
1. 优先级高（易引发 NRE/逻辑错误）：
   - 可空性与集合操作：在访问集合前做非空校验；对 `Contains/TryGetValue` 保证 key 非 null；统一 `List<string?>` 与 `List<string>` 的可空性一致。
   - 异步缺少 `await`：为“缺少 await”的方法补充 `await` 或改为同步方法，避免假异步。
2. 中优先级（代码风格与可读性）：
   - 按提示拆分长逻辑行，对齐空值处理，提升可维护性（如 `LocalSupplierInvoicesReactService.cs:832,905,916` 等）。
3. 低优先级（广泛通知性警告）：
   - 逐步为字符串操作添加非空断言或提前过滤；为 ToDictionary/HashSet 使用 `notnull` 约束的键值源进行预处理。
