## 变更方案
- 批内重复数列：
  - 修改渲染逻辑：仅当计数>1时显示 `Tag`，否则显示空字符串；排序逻辑保持不变。
- 新增列（插入在数量/进货价之后，与详情页字段对齐）：
  - `上次进货价`（只读）：取 `supplierDetect.currentPurchasePrice`，用于对比当前导入的 `进货价`。
  - `是否自动定价`（可编辑）：`Switch` 控件，默认值规则：
    - 新商品（`supplierDetect.exists === false`）默认 `true`。
    - 已有商品默认 `false`。
    - 值持久在行对象中（扩展 `ItemRow`）。
  - `定价浮率`（只读）：来自自动价格服务返回的 `rate`，保留两位小数展示。
  - `新自动零售价`（可编辑）：初始值来自自动价格服务返回的 `retailPrice`，使用 `InputNumber` 可修改。
- 自动定价计算：
  - 新增按钮“计算自动定价”：对当前列表中 `autoPricing === true` 且 `price > 0` 的行，调用 `evaluatePricing({ purchasePrice: row.price, storeCode, supplierCode })`，填充 `pricingFloatRate` 与 `newAutoRetailPrice`。
  - 当用户编辑行内 `进货价` 时，若该行 `autoPricing === true`，可自动触发重新计算（防抖）。
- 操作列（枚举 + 默认建议）：
  - 枚举值：`新建商品` | `更新进货价` | `待操作` | `不操作`（使用 `Select`，可编辑）。
  - 默认建议规则：
    - `supplierDetect.exists === false && barcodeDetect.matched === false` → `新建商品`。
    - `supplierDetect.exists === false && barcodeDetect.matched === true` → `待操作`。
    - `supplierDetect.exists === true && barcodeDetect.matched === false` → `待操作`。
    - `supplierDetect.exists === true && barcodeDetect.matched === true` → `不操作`。
    - 价格变更提示：若已有商品且 `price > 0` 与 `supplierDetect.currentPurchasePrice` 不一致 → 默认 `更新进货价` 优先于上述规则。
  - 默认值在检测结果写入时计算（`onDetectSupplierItem`/`onDetectBarcode`/`onDetectAll` 完成后）。
- 保存行为：
  - 目前创建请求 `CreateInvoiceRequest.items` 仅包含 `quantity` 与 `price`（见 `src/types/localSupplierInvoice.ts:72-80`）。此迭代不修改后端请求结构，新增列仅用于前端辅助与决策提示；后续若需落库再扩展。
