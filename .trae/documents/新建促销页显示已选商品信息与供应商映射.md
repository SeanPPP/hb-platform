## 目标
- 适用店维持多选（当前已是 `Select mode="multiple"`）。
- 在“已选商品”行内显示：供应商（名称）、商品货号、商品名称、零售价。

## 方案
- 引入商品详情与供应商名称映射：
  - 使用 `getProductById(productCode)` 拉取商品详情并缓存到 `productInfoMap`。
  - 使用 `getActiveLocalSuppliers()` 构建 `localSupplierCode -> name` 的 `supplierNameMap`。
- 更新交互：
  - 当用户通过 `ProductPicker` 单选或批量选择后，为每个 `productCode` 调用一次详情加载，填充 `productInfoMap`。
  - 打开编辑已有促销时，遍历 `dto.products` 中的 `productCode` 预加载对应详情。
- UI 展示：
  - 在 `Form.List name="products"` 每行下方追加只读信息区：显示供应商名称（映射展示）、货号、名称、零售价（保留两位小数）。
  - 保持原有 `productCode` 与 `unitWeight` 表单字段与保存逻辑不变。

## 受影响文件
- `src/pages/PosAdmin/Promotions/index.tsx`：新增状态与加载函数，增强 `onPick/onPickMany`，并在 `Form.List` 行渲染商品信息。

## 验证
- 打开“新建促销”→ 添加商品：每行显示供应商名称、货号、名称与零售价；批量选择同样显示。
- 编辑已有促销：已存在商品行也显示上述信息。
