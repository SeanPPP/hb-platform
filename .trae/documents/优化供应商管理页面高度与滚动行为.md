**问题分析**

* 最外层滚动条出现在页面容器 `div` 上，原因是 `wrapRef` 固定高度与内部真实内容（工具栏、表格、分页）高度计算不一致，产生溢出。

* 现有实现在 `src/pages/PosAdmin/SupplierManagement/index.tsx` 中通过 JS 计算 `scroll.y`（33–41 行），使用了经验常量 `48` 作为预留，容易低估分页和边距，导致页面级滚动条出现。

* 页面不应随表格数据高度变化而改变，应保持视口固定高度，内部仅由表格区域滚动。

**优化思路**

* 将页面内容容器固定为视口高度并禁止外层溢出：`overflow: hidden`。

* 使用 Flex 布局，让表格区域占满剩余空间：`flex: 1; min-height: 0;`，避免外层出现滚动条。

* 取消基于 `vh - top` 的高度推导，改为基于容器可用高度的精确计算（`wrapRef.clientHeight - toolbarHeight - 预留卡片内边距`）。

* 动态测量分页高度，或将表格放入独立可滚动容器，避免页面滚动。

**具体改动（文件与关键行）**

* 文件：`src/pages/PosAdmin/SupplierManagement/index.tsx`

  1. 在内容容器 `wrapRef` 所在行（约 63 行）增加：`overflow: 'hidden'`，并保持 `display: 'flex', flexDirection: 'column'`。
  2. 在表格外包一层容器（约 66–97 行之间插入），样式：`style={{ flex: 1, minHeight: 0 }}`，用于承载滚动表格。
  3. 重写高度计算（33–41 行）：改为基于 `wrapRef.current.clientHeight` 与 `toolbarRef.current.getBoundingClientRect().height` 计算 `available`；去除 `vh - top` 与魔数 `48`，改用明确常量或测量分页高度：

     * `const containerH = wrapRef.current?.clientHeight || 0;`

     * `const tbarH = toolbarRef.current?.getBoundingClientRect().height || 0;`

     * `const pagerH = wrapRef.current?.querySelector('.ant-pagination')?.getBoundingClientRect().height || 0;`

     * `const available = containerH - tbarH - pagerH - 8; // 边界预留8px`

     * `setTableScrollY(Math.max(available, 200));`
  4. 保持 `Table` 的固定表头与内部滚动：`scroll={{ y: tableScrollY }}`（约 71 行）。

**页面不随表格高度变化的保证**

* `wrapRef` 高度固定（视口计算）+ `overflow:hidden`，阻止外层滚动条；表格区域在独立 Flex 子容器中滚动。

* 当数据增多时，`scroll.y` 仅改变表格内部滚动，不影响页面总高度。
