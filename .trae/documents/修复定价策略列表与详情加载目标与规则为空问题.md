## 问题分析
- 现象：数据库存在策略、规则(`PricingStrategyDetail`)、目标(`PricingStrategyTarget`)数据，但后端接口返回的 `Targets` 与 `Details` 为空，导致列表的分店/供应商与计数为 0，编辑弹窗也无数据。
- 代码路径：
  - 列表：`d:/Development/cline/blazor/BlazorApp.Api/Services/React/PricingStrategyReactService.cs:1-119`
  - 详情：`d:/Development/cline/blazor/BlazorApp.Api/Services/React/PricingStrategyReactService.cs:121-169`
- 可能根因：
  - 导航加载 `.Includes(...)` 未生效或与 `InSingleAsync` 组合异常，导致 `s.Targets/s.Details` 未填充；
  - 使用导航属性在查询过滤中：`query.Where(s => s.Targets.Any(...))` 可能无法由 SqlSugar 正确翻译为 SQL，影响结果；
  - 连接字符串或上下文指向与插入数据不同的库（需确认，但优先从查询方式修复）。

## 修复方案（后端）
- 详情接口 `GetByIdAsync`：
  - 保留 `.Includes(...)`，同时显式查询子表作为兜底，确保不受导航加载影响：
  - 通过 `_context.PricingStrategyDetailDb.GetListAsync(d => d.StrategyId == id)` 与 `_context.PricingStrategyTargetDb.GetListAsync(t => t.StrategyId == id)` 取数，构造 `PricingStrategyDetailDto`。
- 列表接口 `GetGridAsync`：
  - 过滤：移除导航属性上的 `Any`，改为子查询或联表过滤：
    - `storeCode`：`query = query.Where(s => SqlFunc.Subqueryable<PricingStrategyTarget>().Where(t => t.StrategyId == s.Id && t.TargetType == "Store" && t.TargetCode == val).Any());`
    - `supplierCode`：同理使用子查询。
  - 数据组装：
    - 先按请求分页取出 `items` 与 `ids`，再一次性批量查询子表：
      - `details = await _context.PricingStrategyDetailDb.AsQueryable().Where(d => ids.Contains(d.StrategyId)).ToListAsync()`
      - `targets = await _context.PricingStrategyTargetDb.AsQueryable().Where(t => ids.Contains(t.StrategyId)).ToListAsync()`
    - 按 `StrategyId` 分组，回填：
      - `StoreCodes = targets.Where(t => t.TargetType == "Store").Select(t => t.TargetCode).Distinct().ToList()`
      - `SupplierCodes = targets.Where(t => t.TargetType == "Supplier").Select(t => t.TargetCode).Distinct().ToList()`
      - `DetailsCount = details.Count`，`TargetsCount = targets.Count`
    - 这样避免导航加载不稳定与 N+1 问题。

## 详细修改点
