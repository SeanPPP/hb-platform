## 优化 UpdateSupplierStatistics 方法（包含 200 号供应商单独聚合）

### 目标
修改 `UpdateSupplierStatistics` 方法，使用 `PosmProductSupplierMapping` 表进行连表查询，并按照特定规则进行两次聚合统计。

### 变更计划
1.  **修改 POSM 查询逻辑**：
    *   联接 `SalesOrder`, `SalesOrderDetail`, 和 `PosmProductSupplierMapping`。
    *   查询出原始数据列表，包含：`Date`, `ProductCode`, `Quantity`, `Amount`, `LocalSupplierCode`, `ChinaSupplierCode`。

2.  **执行两次聚合逻辑**：
    *   **聚合 1：本地供应商聚合**
        *   按 `LocalSupplierCode` 分组。
        *   **注意**：即使 `LocalSupplierCode` 是 "200"，也要作为一个普通的本地供应商参与此次聚合。
        *   统计每个 `LocalSupplierCode` 的总销量、总金额、门店数。
    
    *   **聚合 2：国内供应商聚合（仅针对 200 号）**
        *   筛选出 `LocalSupplierCode == "200"` 且 `ChinaSupplierCode` 不为空的记录。
        *   按 `ChinaSupplierCode` 分组。
        *   统计每个 `ChinaSupplierCode` 的总销量、总金额、门店数。
        *   标记 `IsDomestic = true`。

3.  **合并与保存**：
    *   将上述两组聚合结果合并。
    *   从主数据库批量获取供应商名称（LocalSupplier 和 ChinaSupplier）。
    *   保存到 `SupplierSalesStatistic` 表。

### 为什么这样做？
用户要求 `LocalSupplierCode == "200"` 要单独聚合一次，同时国内供应商也要分别再单独聚合一次。这意味着 "200" 这个供应商代码会在统计表中有一条记录（汇总所有 200 的销量），同时每个具体的国内供应商（如 C001, C002）也会有各自的记录。这是一种"双重统计"的需求。

### 代码逻辑预览
```csharp
// 1. 获取基础数据
var rawData = await query.Select(...).ToListAsync();

// 2. 本地供应商聚合 (包含 200)
var localStats = rawData
    .GroupBy(x => x.LocalSupplierCode)
    .Select(...);

// 3. 国内供应商聚合 (仅 200 下的细分)
var chinaStats = rawData
    .Where(x => x.LocalSupplierCode == "200" && !string.IsNullOrEmpty(x.ChinaSupplierCode))
    .GroupBy(x => x.ChinaSupplierCode)
    .Select(...);

// 4. 合并结果并保存
var allStats = localStats.Concat(chinaStats);
```
