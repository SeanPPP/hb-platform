# 自动定价策略实施计划（优化版）

## 1. 数据库模型设计（主子表结构）

### 主表：`PricingStrategy` (定价策略头信息)

位置：`BlazorApp.Shared/Models/HBweb/PricingStrategy.cs`

* **ID**: 主键
* **Name**: 策略名称（如“耐克专供规则”）
* **Level**: 策略级别 (枚举：Supplier, Store, Global)
* **TargetCode**: 关联代码（供应商编码 或 分店代码，Global则为空）
* **Priority**: 优先级数值（用于同级别排序）
* **IsEnabled**: 是否启用

### 子表：`PricingStrategyDetail` (定价规则明细)

位置：`BlazorApp.Shared/Models/HBweb/PricingStrategyDetail.cs`

* **StrategyId**: 关联主表ID
* **MinPrice**: 价格区间下限（如 1.00）
* **MaxPrice**: 价格区间上限（如 10.00）
* **StartRate**: 起始上浮率（如 2.5）
* **EndRate**: 结束上浮率（如 2.2，若相等则为固定浮率）
* **Algorithm**: 算法类型（枚举：Linear线性, Exponential指数, Step阶梯）

## 2. 核心服务实现

### 服务：`AutoPricingService`

位置：`BlazorApp.Api/Services/Pricing/AutoPricingService.cs`

* **功能1：策略匹配 (Match Strategy)**
  按照 **供应商 > 分店 > 默认** 的顺序，查找首个启用的策略。
  * *逻辑*：先查 `Level=Supplier` 且 `TargetCode=当前供应商`；没找到再查 `Level=Store`...

* **功能2：价格计算 (Calculate)**
  输入进货价，根据匹配到的策略明细计算零售价。
  * *算法*：
    * **线性**：$Rate = Start + (End - Start) \times \frac{Price - Min}{Max - Min}$
    * **指数**：$Rate = Start \times (\frac{End}{Start})^{\frac{Price - Min}{Max - Min}}$
  * *保护*：包含单调性检查，防止价格倒挂。

## 3. 业务集成点

### 场景A：进货录入 (LocalSupplierInvoicesReactService)
* 在识别商品（`DetectSupplierItem`）时，自动带出建议零售价。
* 在手动修改进货价时，重新计算。

### 场景B：商品管理 (ProductReactService / StoreRetailPriceReactService)
* 提供“一键重算零售价”的功能接口。
