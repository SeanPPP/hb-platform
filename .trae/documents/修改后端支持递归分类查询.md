# 实现分类过滤包含子分类

用户要求在点击分类时，过滤结果不仅包含当前分类的商品，还要包含所有子分类的商品。

## 1. 现状分析

*   **前端**:
    *   `src/services/warehouseCategory.ts` 中的 `getProductsByCategory` 直接调用 API，传递 `categoryGuid`。
    *   目前还不确定前端是否传递了 `includeSubCategories` 参数（通常需要增加这个参数）。

*   **后端**:
    *   `WarehouseProductService.cs` 中似乎已经有处理子分类的逻辑（`GetCategoryAndSubCategories` 方法），但这可能是用于其他控制器的。
    *   关键是 `WarehouseCategoryReactService.cs`（React 专用的服务）。
    *   在 `WarehouseCategoryReactService.cs` 的 `GetProductsByCategoryAsync` 方法中（约第 222 行），目前的逻辑是：
        ```csharp
        query = query.Where(w =>
            SqlSugar.SqlFunc.Subqueryable<Product>()
                .Where(p => p.ProductCode == w.ProductCode && p.WarehouseCategoryGUID == categoryGuid)
                .Any()
        );
        ```
        这里只匹配了 `p.WarehouseCategoryGUID == categoryGuid`，即只匹配当前分类，**没有**包含子分类。

## 2. 解决方案

我们需要修改后端 `WarehouseCategoryReactService.cs`，使其在查询时递归获取所有子分类的 ID，并使用 `IN` 查询。

### 修改 `WarehouseCategoryReactService.cs`

1.  引入或复制递归获取子分类的逻辑。由于 `WarehouseProductService` 中已有 `GetCategoryAndSubCategories`，我们可以参考或复用。考虑到微服务/模块化，最好是在 `WarehouseCategoryReactService` 中实现类似的逻辑，或者调用已有的 Category Service。
2.  修改 `GetProductsByCategoryAsync` 方法：
    *   获取 `categoryGuid` 的所有子分类 ID 列表。
    *   将查询条件改为 `categoryIds.Contains(p.WarehouseCategoryGUID)`。

### 详细步骤

1.  **添加辅助方法**: 在 `WarehouseCategoryReactService` 中添加 `GetAllSubCategoryIds` 方法。
    ```csharp
    private List<string> GetAllSubCategoryIds(string categoryGuid)
    {
        var allCategories = _context.WarehouseCategoryDb.AsQueryable().ToList();
        var result = new List<string> { categoryGuid };
        GetSubCategoriesRecursive(categoryGuid, allCategories, result);
        return result;
    }

    private void GetSubCategoriesRecursive(string parentGuid, List<WarehouseCategory> allCategories, List<string> result)
    {
        var children = allCategories.Where(c => c.ParentGUID == parentGuid).Select(c => c.CategoryGUID).ToList();
        foreach (var childId in children)
        {
            if (!result.Contains(childId))
            {
                result.Add(childId);
                GetSubCategoriesRecursive(childId, allCategories, result);
            }
        }
    }
    ```

2.  **修改查询逻辑**:
    ```csharp
    if (!string.IsNullOrWhiteSpace(categoryGuid))
    {
        var categoryIds = GetAllSubCategoryIds(categoryGuid);
        query = query.Where(w =>
            SqlSugar.SqlFunc.Subqueryable<Product>()
                .Where(p => p.ProductCode == w.ProductCode && categoryIds.Contains(p.WarehouseCategoryGUID))
                .Any()
        );
    }
    ```

## 3. 验证计划

*   **代码修改**：修改 `WarehouseCategoryReactService.cs`。
*   **验证**：
    1.  选择一个包含子分类的父分类。
    2.  确认商品列表显示了该父分类下的商品以及其子分类下的商品。

**注意**: 这是一个后端逻辑变更，不需要修改前端代码，除非我们想让这个行为变成可选的（例如加一个 checkbox "包含子分类"）。根据用户描述“分类 过滤 需要 包含所有子分类”，这听起来像是一个默认行为的变更请求。

我将直接修改后端服务以支持此默认行为。
