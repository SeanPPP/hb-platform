## 实施计划：个人用户信息管理功能

### 1. 后端 API 分析与实现
- **当前状态**：
  - `AuthController.GetCurrentUser` (`GET /api/Auth/current`)：已存在，用于获取当前登录用户信息。
  - `UsersController.UpdateUserByGuid` (`PUT /api/Users/guid/{guid}`)：已存在，用于修改用户信息。
  - **缺失**：修改密码 API 仅支持管理员直接重置（`UpdateUserPassword`），或者重置为随机密码。**需要实现用户自行修改密码的 API**，验证旧密码。

- **后端变更**：
  - **IAuthService / AuthService**：
    - 添加 `ChangePasswordAsync(string userGuid, ChangePasswordDto dto)` 方法。
    - 逻辑：验证旧密码是否正确 -> 更新为新密码。
  - **AuthController**：
    - 添加 `[HttpPost("change-password")]` 端点。

### 2. 前端路由与页面
- **路由**：
  - 在 `.umirc.ts` (或路由配置文件) 中添加 `/system/profile` 路由。
- **页面组件 `src/pages/UserProfile/index.tsx`**：
  - **基本信息卡片**：
    - 显示：用户名、角色、分店等（只读）。
    - 编辑：全名、邮箱（表单）。
    - 提交调用 `updateUser` 接口。
  - **修改密码卡片**：
    - 表单：当前密码、新密码、确认新密码。
    - 提交调用新的 `changePassword` 接口。

### 3. 前端服务
- **services/auth.ts**：
  - 添加 `changePassword(data: ChangePasswordDto)` 方法。

### 4. SystemLayout 改造
- **SystemLayout.tsx**：
  - 添加右上角用户区域：显示 `Avatar` 和 `Dropdown`。
  - Dropdown 菜单项：
    - 👤 个人中心（跳转 `/system/profile`）
    - 🚪 退出登录（调用 logout）

### 5. 实施步骤
1.  **后端**：在 `AuthService` 和 `AuthController` 中实现修改密码功能。
2.  **前端服务**：更新 `auth.ts`，添加修改密码 API。
3.  **前端页面**：创建 `UserProfile` 页面。
4.  **前端布局**：更新 `SystemLayout.tsx`，添加用户头像下拉菜单。
5.  **前端路由**：注册 `/system/profile`。
