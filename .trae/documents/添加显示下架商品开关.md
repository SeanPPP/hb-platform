# 添加“是否显示下架商品”开关

用户希望在分店订货页面的“查询按钮后”添加一个开关，用于控制是否显示已下架（Inactive）的商品，默认状态为“不显示”（即默认只显示上架商品）。

## 1. 现状分析

*   **前端页面**: `src/pages/StoreOrder/index.tsx` (尚未查看，但这是主入口) 或 `src/pages/StoreOrder/components/ProductGrid.tsx`。
*   **API 服务**: `src/services/warehouseCategory.ts` 中的 `WarehouseProductFilterDto` 已经包含 `isActive` 字段（布尔类型）。
*   **后端逻辑**: `WarehouseCategoryReactService.cs` 已经支持 `isActive` 过滤：
    ```csharp
    if (filter.IsActive.HasValue)
    {
        query = query.Where(w => w.IsActive == filter.IsActive.Value);
    }
    ```

## 2. 需求拆解

1.  **UI 变更**: 在搜索栏（查询按钮后）添加一个 `Switch` 组件，Label 为“显示下架商品”。
2.  **状态管理**: 在组件状态中添加一个变量（如 `showInactive`），默认值为 `false`。
3.  **API 调用**:
    *   当开关为 `false` (默认) 时，`filter.isActive` 应为 `true` (只显示活跃商品)。
    *   当开关为 `true` (显示下架) 时，`filter.isActive` 应为 `undefined` (显示所有) 还是 `false` (只显示下架)？
    *   通常“显示下架商品”意味着“包含下架商品”（即显示所有），或者“只显示下架商品”。
    *   结合用户描述“默认不显示”，意味着默认过滤 `IsActive = true`。如果打开开关，可能是想看到所有商品（包括下架），或者想专门查找下架商品。
    *   **推断**: 大多数场景下，“显示下架商品”意味着取消 `IsActive = true` 的过滤条件，从而显示所有商品。或者，如果后端支持，可以明确传递 `isActive = null` 来获取所有。
    *   查看后端代码：
        ```csharp
        if (filter.IsActive.HasValue)
             query = query.Where(w => w.IsActive == filter.IsActive.Value);
        ```
        如果前端传递 `isActive: null` 或 `undefined`，后端就不会应用 `IsActive` 过滤，从而返回所有商品（包括 Active 和 Inactive）。这符合“显示下架商品”且保留上架商品的直觉（即“显示全部”）。如果用户的意思是“只显示下架”，那逻辑就不同了。
    *   **确认策略**: 开关打开 -> `isActive = undefined` (显示全部)。开关关闭 -> `isActive = true` (只显示上架)。
