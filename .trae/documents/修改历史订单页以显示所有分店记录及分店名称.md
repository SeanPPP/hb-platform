我将修改“历史订单”页面，以显示用户关联的所有分店记录，并展示分店名称。

### 1. 修改 StoreHistoryOrders/index.tsx

*   **获取所有分店信息**：
    *   引入 `getUserStores` API，用于获取当前用户关联的所有分店列表。
    *   在组件加载时调用该 API，并将分店列表存储在 `userStores` 状态中。
    *   创建一个 `storeMap`（Record<string, string>），用于将 storeCode 映射为 storeName。

*   **调整查询逻辑**：
    *   修改 `ProList` 的 `request` 方法。
    *   **不再**从 `initialState?.selectedStore?.storeCode` 获取 `storeCode` 参数。
    *   如果 API 支持查询“该用户所有分店的订单”，则不传 `storeCode`（或者传递特殊标识），让后端返回用户有权限的所有订单。
    *   如果后端 API `getOrderList` 强制要求 `storeCode`，则需要确认 API 行为。根据 `getOrderList` 定义，`storeCode` 是可选的 (`StoreOrderListFilterParams`)。如果不传 `storeCode`，后端应该返回该用户有权限的所有订单。
    *   因此，我将修改请求参数，移除 `storeCode` 过滤，从而获取所有分店的订单。

*   **展示分店名称**：
    *   在 `ProList` 的 `metas.subTitle` 或 `description` 中，使用 `storeMap` 根据 `row.storeCode` 显示对应的分店名称。
    *   例如：在订单号下方或旁边显示 `<Tag icon={<ShopOutlined />}>{storeMap[row.storeCode] || row.storeCode}</Tag>`。
