# 收银用户条码管理页面实施计划

## 功能概述
为分店管理员添加收银用户条码管理页面，允许分店管理员查看、修改和创建自己相关分店的收银条码。

## 后端实现

### 1. 创建 DTOs (BlazorApp.Shared/DTOs/)
- `CashRegisterUserReactDtos.cs`
  - `CashRegisterUserListDto` - 列表DTO
  - `CashRegisterUserDetailDto` - 详情DTO
  - `CreateCashRegisterUserDto` - 创建DTO
  - `UpdateCashRegisterUserDto` - 更新DTO
  - `CashRegisterUserFilterDto` - 查询过滤DTO

### 2. 创建服务接口 (BlazorApp.Api/Interfaces/React/)
- `ICashRegisterUserReactService.cs`
  - `GetGridDataAsync` - 网格查询（支持分页、过滤、排序）
  - `GetByUuidAsync` - 根据UUID获取详情
  - `CreateAsync` - 创建收银用户
  - `UpdateAsync` - 更新收银用户
  - `DeleteAsync` - 删除收银用户
  - `BatchDeleteAsync` - 批量删除

### 3. 创建服务实现 (BlazorApp.Api/Services/React/)
- `CashRegisterUserReactService.cs`
  - 实现上述接口方法
  - 权限过滤：分店管理员只能看到自己分店的数据
  - 使用 SqlSugar 进行数据库操作

### 4. 创建控制器 (BlazorApp.Api/Controllers/React/)
- `ReactCashRegisterUserController.cs`
  - 路由：`/api/react/v1/cash-register-users`
  - 端点：
    - `POST /grid` - 网格查询
    - `GET /{uuid}` - 获取详情
    - `POST /` - 创建
    - `PUT /{uuid}` - 更新
    - `DELETE /{uuid}` - 删除
    - `DELETE /batch-delete` - 批量删除

### 5. 注册服务 (BlazorApp.Api/Program.cs)
- 在依赖注入中注册 `ICashRegisterUserReactService`

## 前端实现

### 1. 创建服务层 (ReactUmi/my-app/src/services/)
- `cashRegisterUser.ts`
  - `getGrid` - 网格查询
  - `getByUuid` - 获取详情
  - `create` - 创建
  - `update` - 更新
  - `delete` - 删除
  - `batchDelete` - 批量删除

### 2. 创建页面 (ReactUmi/my-app/src/pages/PosAdmin/CashRegisterUsers/)
- `index.tsx` - 主页面
  - 功能：
    - 搜索栏：分店、操作用户、用户条码、状态
    - 表格：显示收银用户列表
    - 新建按钮：打开创建弹窗
    - 编辑按钮：打开编辑弹窗
    - 删除按钮：删除单条记录
    - 批量删除：删除多条记录
  - 列配置：
    - 分店名称
    - 操作用户
    - 用户条码
    - 登录角色
    - 条码打印次数
    - 状态
    - 备注
    - 创建时间
    - 操作（编辑/删除）

### 3. 添加国际化文本 (ReactUmi/my-app/src/locales/zh-CN.ts)
- 添加收银用户相关翻译键
- 页面标题、列名、按钮文本、提示信息等

### 4. 添加菜单路由 (ReactUmi/my-app/.umirc.ts 或 config/routes.ts)
- 添加菜单项：`menu.posAdmin.cashRegisterUsers`
- 路由：`/pos-admin/cash-register-users`

## 权限控制
- 分店管理员：只能查看/修改/创建自己分店的收银用户条码
- Admin/WarehouseManager：可以查看所有分店的收银用户条码

## 技术要点
- 后端使用 GridRequestDto/GridResponseDto 模式
- 前端使用 Ant Design Table + Modal
- 权限过滤在服务层实现
- 支持分页、搜索、排序
- 表单验证
