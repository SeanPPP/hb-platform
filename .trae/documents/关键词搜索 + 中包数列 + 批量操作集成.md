原因分析
- 关键字搜索未生效：前端请求体未携带 globalSearch，当前页面只在 useEffect 中根据 isActive、分类与列头过滤构建请求，缺少全局关键字字段，导致无法统一查询。
- “中包数”列缺失：后端 React 列表 DTO 未包含 MinOrderQuantity 字段（BlazorApp.Shared/DTOs/ReactTableDtos.cs:24-42），查询投影也未选择该字段（BlazorApp.Api/Services/React/ProductWarehouseReactService.cs:583-602）。前端类型与列也未定义该字段（ReactUmi/my-app/src/services/productWarehouse.ts:83-99、src/pages/WarehouseCategories/index.tsx:65-83）。
- 批量上/下架与批量设置中包数：后端已提供批量状态接口（BlazorApp.Api/Controllers/WarehouseProductBatchController.cs:285-301；服务实现 BlazorApp.Api/Services/WarehouseProductBatchService.cs:612-638）以及支持 MinOrderQuantity 的批量更新接口（Controller: 102-137；Service: 252-368），但前端页面未提供按钮与调用方法。

1) 后端契约扩展（只读列表数据）
- 在 React 列表 DTO 增加 MinOrderQuantity 字段：
  - BlazorApp.Shared/DTOs/ReactTableDtos.cs: 在 WarehouseProductReactListDto 增加 public int? MinOrderQuantity { get; set; }。
- 在列表查询 Select 投影补充字段：
  - BlazorApp.Api/Services/React/ProductWarehouseReactService.cs: 在 .Select(...) 中添加 MinOrderQuantity = w.MinOrderQuantity（约 595-602 行附近）。
- 排序支持（可选）：
  - 同文件排序映射处（543-575 行），增加 sort == "minorderquantity" 时对 w.MinOrderQuantity 的 OrderBy。

2) 前端服务类型与接口
- ReactUmi/my-app/src/services/productWarehouse.ts：
  - WarehouseProductReactListDto 增加 minOrderQuantity?: number（83-99 行附近）。

    - batchSetMinOrderQuantity(productCodes: string[], qty: number): POST /api/v1/WarehouseProductBatch/batch-update，构造 { products: [{ productCode, minOrderQuantity: qty }] }。

3) 页面 UI 与统一查询
- 关键字搜索：
  - src/pages/WarehouseCategories/index.tsx：在工具栏 isactive 单选框前添加 Input.Search（或 Input），新增 globalSearch 状态；useEffect 依赖加入 globalSearch；loadTable 请求体增加 globalSearch 字段（100-111 行附近）。输入变更时重置页码到 1。
