# 解决编辑失败与列表数据显示异常问题

**问题分析**

1.  **编辑无法打开 (Span 点击无效)**
    *   **现象**: 点击列表中的“编辑”按钮无反应。
    *   **原因**: 数据解析路径错误。
        *   `openEdit` 函数中：`const dto: PricingStrategyDetailDto | undefined = (res as any)?.data?.data`
        *   **关键点**: 与列表加载 (`loadData`) 的情况类似，`getById` 后端返回的是 `ApiResponse<PricingStrategyDetailDto>`。
        *   Controller 代码: `return Ok(res);`
        *   如果 Umi Request 拦截器或 axios 响应中，`res` 包含 `data`，那么 `res.data` 就是 `ApiResponse`。
        *   `res.data.data` 确实是正确路径。但是，如果 Umi request 配置了 error adapter 或者 response interceptor 自动解包，`res` 可能直接就是响应体。
        *   **最可能原因**: 前端代码中 `res.data` 实际上是 `undefined` (如果拦截器已解包)，或者 `res.data.data` 不存在。
        *   **检查**: 在之前的 `fix-pricing-strategies-response-parsing` 任务中，我们修复了 `loadData`，意识到后端直接返回 DTO 或 API Response。
        *   **验证**: `ReactPricingStrategiesController.Get` 返回 `ApiResponse`。
        *   如果拦截器未解包：`res.data` (ApiResponse) -> `res.data.data` (DTO)。
        *   如果拦截器已解包：`res` (ApiResponse) -> `res.data` (DTO)。
        *   **结论**: 需要像修复 `loadData` 那样，兼容两种路径：`(res as any)?.data?.data || (res as any)?.data`。

2.  **列表数据为 0 (`td` 显示异常)**
    *   **现象**: 列表中的“规则数”和“目标数”显示为 0，尽管可能有数据。
    *   **原因**: 属性大小写不匹配。
        *   **后端 DTO**: `PricingStrategyListDto` 定义了 `DetailsCount` 和 `TargetsCount` (PascalCase)。
        *   **前端代码**: `columns` 中 `dataIndex: 'detailsCount'`, `dataIndex: 'targetsCount'` (camelCase)。
        *   **序列化**: ASP.NET Core 默认序列化为 camelCase，所以后端 `DetailsCount` -> JSON `detailsCount`。这部分通常是正确的。
        *   **真正原因**: 可能是后端查询时没有正确 `Include` 或计算 `Count`。
        *   **后端代码检查**:
            ```csharp
            // PricingStrategyReactService.cs
            var list = items.Select(s => new PricingStrategyListDto {
                // ...
                DetailsCount = s.Details?.Count ?? 0,
                TargetsCount = s.Targets?.Count ?? 0,
            })
            ```
            查询时：`.Includes(s => s.Details).Includes(s => s.Targets)`。
            这看起来是正确的。
        *   **另一种可能**: 之前因为主键类型变更，导致旧数据关联丢失。如果 `PricingStrategyDetail` 表中的 `StrategyId` 还是旧的 Int 值或者为空，那么关联查询结果为空，Count 自然为 0。
        *   **结论**: 如果是新创建的数据，应该能正确计数。如果是旧数据，Count 为 0 是预期的（因为关联已断）。
        *   **前端显示问题**: 如果字段值是 undefined，Antd Table 可能不显示或显示空。如果显示 '0'，说明值确实是 0。
        *   **需要确认**: 是否真的是新创建的数据？如果是新建的数据，且规则/目标已填，Count 仍为 0，则后端保存逻辑有问题（未正确保存明细）。
        *   **回顾 `CreateAsync`**:
            ```csharp
            var id = UuidHelper.GenerateUuid7();
            // ... insert strategy
            // ... insert details with StrategyId = id
            ```
            逻辑正确。
        *   **疑点**: 之前 `editingId` 类型错误导致保存时可能有些问题？
        *   **行动**: 我将先修复编辑功能，这样用户可以打开详情查看是否真的有明细数据。如果详情里有数据但列表 Count 为 0，那是后端查询问题。如果详情里也没数据，那是保存问题。
