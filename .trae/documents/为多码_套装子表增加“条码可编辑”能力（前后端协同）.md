## 原因与现状
- 子表已统一使用 `ProductSetCode` 数据源，并支持编辑 `SetPurchasePrice/SetRetailPrice`；但后端当前未提供条码编辑接口，条码仅可预览。
- 模型已包含 `SetBarcode` 字段（BlazorApp.Shared/Models/HBweb/ProductSetCode.cs:36-41），适合支持编辑。

## 后端改造
- 控制器：BlazorApp.Api/Controllers/React/ReactProductSetCodesController.cs
  - 新增接口：`PUT /api/react/v1/product-set-codes/batch-barcodes`
  - 入参 DTO：`BatchUpdateBarcodesDto { List<BatchUpdateBarcodesItemDto> Items }`，`BatchUpdateBarcodesItemDto { string Id; string? SetBarcode; }`
  - 权限：`[Authorize(Roles = "Admin")]`
  - 返回：`{ success: boolean, message?: string }`
- 服务接口：BlazorApp.Api/Interfaces/React/IProductSetCodeReactService.cs
  - 新增方法：`Task<ApiResponse<bool>> BatchUpdateBarcodesAsync(List<BatchUpdateBarcodesItemDto> items, string updatedBy)`
- 服务实现：BlazorApp.Api/Services/React/ProductSetCodeReactService.cs
  - 实现 `BatchUpdateBarcodesAsync`：
    - 校验：空字符串允许清空或保持？建议允许空即清空；长度 ≤50。
    - 唯一性：确保 `SetBarcode` 在 `ProductSetCode` 表内不重复（可选：联表 `Product` 按供应商范围唯一）。
    - 更新：批量更新对应 `SetCodeId` 的 `SetBarcode`、`UpdatedAt`、`UpdatedBy`。
    - 返回成功更新条数与错误信息。

## 前端改造
- 服务层：ReactUmi/my-app/src/services/multiCodeSet.ts
  - 新增方法：`export async function batchUpdateBarcodes(data: { items: { id: string; setBarcode?: string }[] })`
  - 调用 `PUT /api/react/v1/product-set-codes/batch-barcodes`
- 编辑弹窗页面：ReactUmi/my-app/src/pages/PosAdmin/ProductManagement/index.tsx
  - 子表列：将 `setBarcode` 改为可编辑 `Input`，编辑写入本地 `setPriceEdits` 或新增 `setExtraEdits` 键值 `{ id: string; setBarcode?: string }`。
  - 保存：
    - 先构造条码变更列表并调用 `batchUpdateBarcodes`；
    - 再构造价格变更列表并调用 `batchUpdateSetPrices`；
    - 两者均成功后提示“更新成功”并刷新。
  - 交互校验：
    - 条码输入长度 ≤ 50；可选校验 EAN-13 格式（如需）。
    - 重复条码后端会返回错误；前端 `message.error` 显示。

## 验证方案
- 套装商品与多码商品均可在编辑弹窗中修改条码与价格；保存后刷新列表，子表显示更新后的条码与价格。
- 验证重复条码与超长条码的后端校验与前端提示。

## 注意事项
- 保持更新接口为 Admin 权限，避免普通用户误改条码。
- 若需要支持“新增条码”功能（新增 `ProductSetCode` 行），可另行增设创建接口；本次仅支持编辑已有条码。
