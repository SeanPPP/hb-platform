浏览器标签页图标未更新的原因可能是配置仅添加到了 `layout` 中，而未作为全局配置生效，或者是因为浏览器缓存了旧图标。

我将更新 Umi 配置，显式设置全局 Favicon。

### 计划
1.  **更新 `.umirc.ts`**: 在配置根节点（`layout` 外部）添加 `favicons` 配置。这是 Umi 中设置 HTML 头部图标的标准方式。
2.  **保留 Layout 配置**: 保持 `layout.favicon` 不变，以确保布局组件也能正确识别。

### 实施步骤
1.  **修改 `.umirc.ts`**:
    *   在 `defineConfig` 对象中添加 `favicons: ['/favicon.svg']`。

### 用户提示
修改配置后，开发服务器通常会自动重载。但**浏览器对 Favicon 的缓存非常顽固**。如果修改后图标仍未变化，请尝试：
*   **强制刷新**: 按 `Ctrl + F5`。
*   **无痕模式**: 在无痕/隐私窗口中打开页面测试。
*   **重启服务**: 如果以上无效，可能需要重启开发服务器。
