## 问题分析
- 后端 `GetContainerProductsAsync` 已通过导航 `.Includes(x => x.LocalProduct)` 与 `.Includes(x => x.Product)` 加载了本地商品与国内商品，但 `ContainerDetail -> ContainerDetailDto` 的映射未把 `Product(Local: DomesticProduct)` 写入 `ContainerDetailDto.商品信息`，也没有导出本地商品是否存在的标识。
- `ContainerProductInfoDto` 中当前无“条形码”字段，`DomesticProduct -> ContainerProductInfoDto` 的映射也未映射条码，导致前端无法展示条码。
- 前端 `src/pages/ContainerDetail/index.tsx` 的列定义不存在“条码”与“是否新商品”。

## 后端改动
- 修改 DTO：
  - 在 `BlazorApp.Shared.DTOs.ContainerDto.cs` 的 `ContainerProductInfoDto` 中新增 `条形码?: string` 字段。
  - 在 `ContainerDetailDto` 中新增 `是否新商品?: bool` 字段。
- 修改 AutoMapper 映射（`BlazorApp.Api/Mappings/Profiles/ContainerMappingProfile.cs`）：
  - 为 `CreateMap<ContainerDetail, ContainerDetailDto>` 增加：
    - `.ForMember(dest => dest.商品信息, opt => opt.MapFrom(src => src.Product))` 把国内商品信息映射到 `商品信息`。
    - `.ForMember(dest => dest.是否新商品, opt => opt.MapFrom(src => src.LocalProduct == null))` 用于标识新商品。
  - 为 `CreateMap<DomesticProduct, ContainerProductInfoDto>` 增加：
    - `.ForMember(dest => dest.条形码, opt => opt.MapFrom(src => src.Barcode))`。
- 无需改动 `ContainerReactService.GetContainerProductsAsync`（`BlazorApp.Api/Services/React/ContainerReactService.cs:182-197`），现有导航加载已满足，映射补齐后即可返回所需字段。

## 前端改动
- 类型：修改 `ReactUmi/my-app/src/types/container.ts`：
  - 在 `ContainerProductInfo` 中新增 `条形码?: string`。
  - 在 `ContainerDetail` 中新增 `是否新商品?: boolean`。
- 页面：修改 `ReactUmi/my-app/src/pages/ContainerDetail/index.tsx` 的列定义：
  - 在“货号”列后新增“条码”列，渲染 `row.商品信息?.条形码 || '-'`。
  - 新增“是否新商品”列，渲染为 `row.是否新商品 ? <Tag color="red">新商品</Tag> : '-'`。

- 后端本地编译，确保映射新增字段生效；调用 `GET /api/react/v1/containers/{containerGuid}/products`，确认返回 JSON 中存在 `商品信息.条形码` 与 `是否新商品`。
