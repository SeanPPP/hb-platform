# 解决 504 Gateway Timeout 问题

**原因分析**
前端报错 `504 Gateway Timeout` 表示前端开发服务器无法连接到后端 API。
经检查：
- 前端代理配置正确指向 `http://localhost:5001`。
- 后端启动配置正确设置为监听 `http://0.0.0.0:5001`。
- **最可能的原因是后端服务未启动。**

**执行计划**
1. **检查端口状态**：确认 5001 端口是否处于空闲状态。
2. **启动后端服务**：在新的终端中运行后端 API 项目。
3. **验证连接**：等待服务启动后，您即可重试登录。
