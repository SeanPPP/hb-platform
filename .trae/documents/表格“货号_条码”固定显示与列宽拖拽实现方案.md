分析原因
- 目前列隐藏主要来源是 AntD Table 的 responsive 配置与横向滚动时的可视区域限制。我们之前只为部分非关键列设置了 responsive:['md']，但在小屏情况下，视口有限，用户需要横向滚动才能看到所有列，导致“看起来被隐藏”。
- “货号”和“条码”列（src/pages/WarehouseCategories/index.tsx:70-71）目前没有 responsive 配置，但未做固定列处理，因此在小屏或横向滚动时可能不在首屏可见。
- AntD Table原生不支持列宽拖拽，需要通过自定义表头单元或第三方方案实现。

实施方案
1) 货号/条码列固定在左侧，始终可见
- 在列定义为“货号”和“条码”两列添加 fixed:'left'，并设置合理初始宽度（如 160-180），确保横向滚动时这两列不移出视口。
- 保持这两列不使用 responsive 隐藏策略。

2) 支持列宽度拖拽调整
- 引入 react-resizable 作为轻量方案（标准做法），不更换表格组件。
- 定义 ResizableTitle 作为自定义表头单元（Table.components.header.cell），为每一列添加 width 和 onResize。
- 将各列宽度放入本组件的 state（或 useRef），用户拖拽改变后即时更新列定义并触发表格重渲染。
- 设定最小/最大宽度约束，避免极端拖拽导致布局破坏。
- 可选：将用户调整后的宽度持久化到 localStorage，页面刷新后还原。

3) 保持现有响应式与横向滚动策略
- 保留非关键列的 responsive:['md'] 隐藏（如供应商、贴牌价格、创建/更新日期等），但保证“货号/条码”固定在左侧始终可见。
- scroll={{ x }} 继续启用，使小屏可左右滑动；sticky 维持表头固定。
