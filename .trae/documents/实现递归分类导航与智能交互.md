# 实现 StoreLayout 递归分类导航

## 问题分析
用户要求在 `StoreLayout.tsx` 中实现增强的分类导航功能，具体包括：
1.  **递归菜单生成**: 支持无限层级的子分类（目前代码仅支持一级子菜单）。
2.  **智能触发**: 根据设备类型（桌面 vs 触摸）决定是悬停还是点击触发菜单。
3.  **样式适配**: 配合已有的 `StoreLayout.less` 样式，确保嵌套菜单显示正确。

检查发现 `StoreLayout.less` 已经包含了所需的样式（z-index, mobile padding），但 `StoreLayout.tsx` 仍然使用旧的平面映射逻辑。

## 实施计划

### 1. 修改 `src/layouts/StoreLayout.tsx`
*   **添加 `supportsHover` 检测**: 使用 `window.matchMedia` 检测设备是否支持 hover。
*   **实现 `buildMenuItems` 函数**: 递归转换 `WarehouseCategoryDto` 为 Antd `MenuProps['items']`，支持多级嵌套。
*   **更新渲染逻辑**:
    *   使用 `buildMenuItems` 生成菜单项。
    *   根据 `isHoverSupported` 动态设置 `Dropdown` 的 `trigger` 属性（桌面 hover，移动端 click）。
    *   设置 `menu.triggerSubMenuAction` 以适配移动端子菜单交互。

### 2. 验证与测试
*   由于后端数据可能不够丰富，我将在 `fetchCategories` 中注入一些测试用的嵌套数据（如用户所述的“示例嵌套分类结构”），以便验证多级菜单功能。

## 代码变更预览
```typescript
// Helper
const supportsHover = () => window.matchMedia('(hover: hover)').matches;

// Component
const [isHoverSupported, setIsHoverSupported] = useState(true);

// Recursive Builder
const buildMenuItems = (cats) => {
  return cats.map(c => ({
    key: c.categoryGUID,
    label: c.categoryName,
    children: c.children ? buildMenuItems(c.children) : undefined,
    onClick: !c.children ? () => handleClick(c.categoryGUID) : undefined
  }));
}

// Render
<Dropdown 
  menu={{ items, triggerSubMenuAction: isHoverSupported ? 'hover' : 'click' }}
  trigger={isHoverSupported ? ['hover'] : ['click']}
>
```
