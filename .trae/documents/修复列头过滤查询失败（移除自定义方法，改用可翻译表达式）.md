**问题分析**

* 报错来自 SqlSugar 表达式翻译：`ApplyTextEval(...)` 是自定义方法，被写入 `Where` 的表达式中，SqlSugar 无法将其翻译为 SQL。

* 受影响字段：`storeName/supplierName/productName/itemNumber` 在 `query` 与 `countQuery` 的文本过滤分支使用了 `ApplyTextEval`。

**修复方案**

* 将上述四个字段的过滤改为内联可翻译的字符串操作表达式：`Contains/StartsWith/EndsWith/Equals/NotEqual/IsNullOrEmpty`。

* 覆盖两处：主查询 `query` 与计数查询 `countQuery`，保持与 `ApplyText` 对 StoreRetailPrice 字段一致的语义（null 安全）。

* 不改动数值过滤与集合过滤逻辑。

**具体改动**

* 替换 `ApplyTextEval(st.StoreName, op, v)` 为：

  * contains：`st.StoreName != null && st.StoreName.Contains(v)`

  * startswith：`st.StoreName != null && st.StoreName.StartsWith(v)`

  * endswith：`st.StoreName != null && st.StoreName.EndsWith(v)`

  * equals：`st.StoreName == v`

  * notequal：`st.StoreName != v`

  * blank：`string.IsNullOrEmpty(st.StoreName)`

  * notblank：`!string.IsNullOrEmpty(st.StoreName)`

* 对 `sup.Name、prod.ProductName、prod.ItemNumber` 同样处理。

**额外说明**

* 前端 antd 的 `message` 静态 API 警告为上下文提示，不影响功能；若需消除我可在后续将页面消息改为 `App.useApp()` 或在根组件使用 `<App>` 包裹。

**验证**

* 重新应用列头过滤不会抛出 SqlSugar 异常；后端能返回筛选后的数据。
