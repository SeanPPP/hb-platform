## 问题原因
- 促销保存使用的是自封装 Axios `utils/request.ts`，其响应拦截器在成功时返回的是 `AxiosResponse`（包含 `data` 字段），而非直接返回业务对象。
- 当前 `saveEditor` 中用 `res.success` 判断成功，读取了 `AxiosResponse.success`（不存在），导致前端以为失败；实际上 `res.data.success === true`，后端已保存成功。

## 修复方案
- 统一成功判断读取 `res.data.success`，并按 `res.data.data` 取返回实体：
  - `createPromotion`：`const r = await createPromotion(payload); const ok = !!((r as any)?.data?.success)`
  - `updatePromotion`：`const r = await updatePromotion(id, payload); const ok = !!((r as any)?.data?.success)`
  - 删除、启用等同理：读取 `res.data.success`
- 同时在保存成功时使用 `message.success('保存成功')`，失败时读取 `res.data.message || '保存失败'`
- 可抽出一个小工具函数（就地实现）：`const getApi = (res:any) => ((res?.data?.data) ?? res?.data ?? res)`，用于兼容读取

## 具体改动位置
- 文件：`src/pages/PosAdmin/Promotions/index.tsx`
  - `saveEditor` 成功判断与消息：约 `143-148` 行，改为读取 `res.data.success`
  - `toggleEnable`、`handleDelete` 成功判断：约 `176-189`、`161-174` 行，同步改为读取 `res.data.success`

## 验证
- 新建促销后前端弹出“保存成功”，列表刷新显示新记录。
- 编辑、启用/禁用、删除操作均能正确提示成功/失败。
