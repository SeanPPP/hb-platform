## 目标
- 将 `BatchUpdateDetailsAsync` 从逐条查询/更新改为批量查询与批量更新，降低数据库往返与锁开销，提升性能与稳定性。

## 实施方案
- 一次性查询所有待更新的明细：按 `HGUID` 批量查出 `ContainerDetail` 列表，映射为字典。
- 在内存批量应用更新：遍历 `updates`，将变更写入对应 `detail` 实体，仅标记有变更的实体加入待更新集合。
- 批量更新明细：使用 `Updateable(list).UpdateColumns(...).WhereColumns(it => new { it.DetailCode })` 批量更新指定列。
- 批量更新商品信息：收集所有存在名称变更的 `ProductCode`，一次性查询 `DomesticProduct` 列表，批量改写名称与英文名称后，使用 `Updateable(list).UpdateColumns(...).WhereColumns(it => new { it.ProductCode })` 批量更新。
- 使用事务包裹整个批次，失败回滚。
- 返回 `totalUpdated` 为成功更新的明细条数，`totalRequested` 由控制器保持为 `updates.Count`。

## 验证
- 前端“保存”分片提交将不再超时；后端日志应显示成批处理而非逐条处理。
- 随机抽样验证明细与商品信息是否正确更新。
