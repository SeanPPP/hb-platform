## 背景与约束
- 子表数据源统一为 `ProductSetCode`，现已支持价格编辑，但条码不可编辑、不可新增。
- 需求：在编辑弹窗的“套装/多码”表中支持“新增条码行”，并且后端不做条码唯一性判断。

## 后端新增接口
- 控制器 `BlazorApp.Api/Controllers/React/ReactProductSetCodesController.cs`
  - 新增创建接口：`POST /api/react/v1/product-set-codes/batch-create`
  - 入参：`{ items: Array<{ productCode: string; setItemNumber?: string; setBarcode?: string; setPurchasePrice?: number; setRetailPrice?: number; isActive?: boolean; }> }`
  - 行为：
    - `setItemNumber` 为空时，根据父商品货号自动生成（复用 `ItemNumberHelper.GenerateSetItemNumber` 逻辑）。
    - `setBarcode` 允许为空或重复（按需求不做唯一性判断）。
    - 设置审计字段 `UpdatedAt/UpdatedBy`。
  - 返回：`{ success: boolean, data: { createdIds: string[] }, message?: string }`
- 服务接口 `BlazorApp.Api/Interfaces/React/IProductSetCodeReactService.cs`
  - 新增：`Task<ApiResponse<List<string>>> BatchCreateAsync(List<CreateSetCodeItemDto> items, string updatedBy)`
- 服务实现 `BlazorApp.Api/Services/React/ProductSetCodeReactService.cs`
  - 校验必填：`productCode`、生成或提供的 `setItemNumber`、`setQuantity=1`、`setType` 默认给 2（多码套装）或沿用父商品类型（可选）。
  - 批量插入 `ProductSetCode`，返回创建的 `SetCodeId` 列表。

## 共享DTO（后端）
- 文件 `BlazorApp.Shared/DTOs/ProductSetCodeReactDtos.cs`
  - 新增：
    - `CreateSetCodeItemDto { string ProductCode; string? SetItemNumber; string? SetBarcode; decimal? SetPurchasePrice; decimal? SetRetailPrice; bool? IsActive; }`
    - `BatchCreateSetCodesDto { List<CreateSetCodeItemDto> Items }`

## 前端改造
- 服务层 `ReactUmi/my-app/src/services/multiCodeSet.ts`
  - 新增：`export async function batchCreateSetCodes(data: { items: { productCode: string; setItemNumber?: string; setBarcode?: string; setPurchasePrice?: number; setRetailPrice?: number; isActive?: boolean; }[] }) { return request(`${API_BASE}/batch-create`, { method: 'POST', data }); }`
- 页面 `ReactUmi/my-app/src/pages/PosAdmin/ProductManagement/index.tsx`
  - 子表上方新增按钮：`添加条码`
    - 点击后在 `setCodes` 中追加临时行 `{ tempId, setBarcode: '', setPurchasePrice: undefined, setRetailPrice: undefined, setItemNumber: '' }`，并在编辑 map 中跟踪修改。
  - 列定义：
    - `setBarcode` 改为 `Input` 可编辑；
    - `setPurchasePrice`、`setRetailPrice` 用 `InputNumber` 可编辑；
    - 新增 `setItemNumber` 列（必填），可编辑或留空由后端生成。
  - 保存流程：
    - 拆分新行与旧行：
      - 新行（无 `setCodeId`）构造 `batchCreateSetCodes.items`，`productCode` 使用当前商品编码；`setItemNumber` 为空则允许后端生成。
      - 旧行构造 `batchUpdateBarcodes.items` 与 `batchUpdateSetPrices.items`（若条码也支持编辑，参考先前方案可一并提交）。
    - 依次调用创建→更新，全部成功后关闭弹窗并刷新。

## 校验与交互
- 前端仅做基础校验：
  - `setItemNumber` 文本长度 ≤50；
  - `setBarcode` 文本长度 ≤50；
  - 价格非负。
- 不做条码唯一性校验（按需求）。

## 验证
- 类型为“套装商品(1)”或“多码商品(2)”的编辑弹窗中，点击“添加条码”可新增行并输入条码与价格；保存后列表刷新显示新增数据。
