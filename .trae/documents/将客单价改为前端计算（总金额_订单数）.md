## 实施计划

### 前端修改

#### 1. 修改 DashboardSummary.tsx
- 将客单价从 `data?.averageOrderValue` 改为前端计算：`data?.orderCount > 0 ? data.totalAmount / data.orderCount : 0`
- 对比客单价同样改为前端计算：`data?.compareOrderCount > 0 ? data.compareTotalAmount / data.compareOrderCount : 0`
- 增长率也改为前端计算

#### 2. 修改 StoreSalesRankTable.tsx
- 客单价列渲染改为前端计算：`record.orderCount > 0 ? record.totalAmount / record.orderCount : 0`
- 不再使用后端返回的 `averageOrderValue` 字段

#### 3. 修改 HourlySalesChart.tsx
- 已使用前端计算（`totalAmount / customerCount`），无需修改

#### 4. 类型定义 (salesDashboard.ts) - 可选清理
- 可以保留 `averageOrderValue` 字段以保持兼容性，或者标记为废弃

### 注意事项

⚠️ **供应商表格问题**：
- `SupplierSalesRank` 和 `ChinaSupplierSalesRank` **没有订单数数据**
- 无法计算客单价（总金额/订单数）
- 如果需要在供应商表格显示客单价，需要后端先添加订单数字段

### 计算规则
- 客单价 = `总金额 / 订单数`
- 避免除零错误：当订单数为 0 时，客单价为 0
- 对比客单价同理计算

- ✅ 前端组件：3个文件需要修改
- ❌ 后端代码：无需修改（保持现有 API 返回）
- ❌ 数据库：无需修改
