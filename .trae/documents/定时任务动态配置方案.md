# 定时任务动态配置实现方案

## 目标
将 `ScheduledTaskService` 的硬编码配置改为数据库存储，管理员可在后台页面修改，修改后立即生效。

## 实现步骤

### 1. 创建配置实体
- 在 `BlazorApp.Shared/Models/` 创建 `ScheduledTaskConfig.cs`
- 包含：每小时间隔、每日执行时间、更新时间、更新者

### 2. 创建配置服务
- 创建 `IScheduledTaskConfigService` 接口
- 创建 `ScheduledTaskConfigService` 实现
- 提供：获取配置、更新配置方法
- 实现 `ConfigChanged` 事件用于通知配置变更

### 3. 修改定时任务服务
- 移除 `IOptions<ScheduledTaskOptions>` 依赖
- 注入 `IScheduledTaskConfigService`
- 监听 `ConfigChanged` 事件
- 添加 `RestartTimers()` 方法实现动态重启
- 添加线程安全锁机制

### 4. 创建 API 控制器
- 创建 `ScheduledTaskConfigController`
- GET `/api/ScheduledTaskConfig` - 获取当前配置
- PUT `/api/ScheduledTaskConfig` - 更新配置（需管理员权限）

### 5. 数据库初始化
- 在 `SqlSugarContext` 添加配置表
- 在 `SeedDataService` 添加默认配置初始化

### 6. 注册服务
- 在 `Program.cs` 注册 `IScheduledTaskConfigService`

## 技术要点
- 使用事件机制实现配置变更通知
- 使用锁保证定时器重启的线程安全
- 权限控制：只有管理员可修改配置
- 配置更新后立即重新初始化定时器
