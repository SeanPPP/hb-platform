## 问题概述
- LocalSupplierInvoiceCreate/index.tsx：调用 processBatches 传入 4 个参数导致签名不匹配；进度回调参数隐式 any。
- LocalSupplierInvoiceDetail/index.tsx：在 JSX 中使用 await import，违反“仅允许在异步函数和模块顶级使用 await”。
- LocalSupplierInvoices/index.tsx：未正确导入 VirtualInvoiceGrid；onView/onDelete 的 record 参数隐式 any。
- 其他后端警告为可容忍的非阻断项，本次聚焦 TypeScript 编译错误修复。

## 修复方案
1) 扩展通用批处理工具签名
- 文件：`ReactUmi/my-app/src/utils/batch.ts`
- 变更：将 `processBatches` 签名扩展为 `processBatches<T>(batches: T[][], concurrency: number, worker: (batch: T[]) => Promise<void>, onProgress?: (done: number, total: number) => void)`；内部维护 done 计数并在每批处理后触发 `onProgress?.(done, total)`。
- 目的：兼容页面现有的 4 参数用法，消除“应有 3 个参数”错误，并为进度回调提供明确类型。

2) 修正创建页的调用与类型
- 文件：`ReactUmi/my-app/src/pages/PosAdmin/LocalSupplierInvoiceCreate/index.tsx`
- 变更：
  - 所有 `processBatches` 调用保持 4 参形式，并显式标注进度回调参数类型，例如 `(done: number, total: number) => messageApi.open(...)`，消除隐式 any 报警。
  - 保持 `chunk` 与 `processBatches` 来自 `@/utils/batch` 的导入，不再声明本地重复实现（已删除）。

3) 修正明细页的虚拟化渲染用法
- 文件：`ReactUmi/my-app/src/pages/PosAdmin/LocalSupplierInvoiceDetail/index.tsx`
- 变更：
  - 顶部引入 `VirtualInvoiceDetailGrid`：`import VirtualInvoiceDetailGrid from '@/components/VirtualInvoiceDetailGrid'`。
  - 将 JSX 中的 `await import` 替换为组件用法：`<VirtualInvoiceDetailGrid rows={items} />`，移除非法的 await 表达式。

4) 修正列表页的虚拟化导入与类型
- 文件：`ReactUmi/my-app/src/pages/PosAdmin/LocalSupplierInvoices/index.tsx`
- 变更：
  - 顶部引入 `VirtualInvoiceGrid`：`import VirtualInvoiceGrid from '@/components/VirtualInvoiceGrid'`（若缺失则补齐）。
  - 为回调参数添加类型：`onView={(record: DataType) => {...}}`、`onDelete={async (record: DataType) => {...}}`，消除隐式 any。
