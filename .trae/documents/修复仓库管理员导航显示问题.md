## 修复后台导航菜单不显示的问题

### 问题原因
在 `app.ts` 中的 `layout` 配置中，`menuDataRender` 函数依赖于全局变量 `window.g_initialState` 来获取用户权限信息。这种方式不可靠，可能导致 `initialState` 未正确获取，从而使 `access` 对象为空，导致所有菜单项因权限检查失败而被过滤。

### 修复方案
修改 `d:\Development\cline\blazor\ReactUmi\my-app\src\app.ts` 文件：

1.  更新 `layout` 导出函数的签名，使其接收 Umi 注入的 `initialState` 参数。
2.  在 `menuDataRender` 中直接使用闭包中的 `initialState`，而不是依赖 `window.g_initialState`。

### 代码修改预览

```typescript
// 修改前
export const layout = () => {
  return {
    // ...
    menuDataRender: (menuData: any[]) => {
      // 🔐 获取当前用户权限
      const initialState = (window as any).g_initialState;
      const access = initialState?.access || {};
      // ...
    }
  }
}

// 修改后
export const layout = ({ initialState }: any) => {
  return {
    // ...
    menuDataRender: (menuData: any[]) => {
      // 🔐 获取当前用户权限
      const access = initialState?.access || {};
      // ...
    }
  }
}
```

### 预期效果
- 修复后，`layout` 配置将正确接收并使用 `initialState`。
- `menuDataRender` 能正确获取 `access` 对象。
- 仓库管理员登录后，具备权限的菜单项（如“国内采购”、“货柜管理”等）将正常显示。
