# 修改订单列表统计字段

用户要求修改订单列表 (`StoreOrderListItemDto`) 的字段定义和统计逻辑，并在前端展示。

## 修改计划

1.  **后端 DTO (`StoreOrderDtos.cs`)**:
    - 更新 `StoreOrderListItemDto` 类：
        - `TotalAmount` 属性注释更新为 "预计销售金额 (Order Qty * OEMPrice)"。
        - `ImportTotalAmount` (或 `TotalImportAmount`) 属性注释更新为 "发货金额 (Alloc Qty * OEMPrice)"。
        - 添加 `TotalOrderAmount` (decimal) 属性，表示 "订货金额 (Order Qty * OEMPrice)" (与 TotalAmount 可能重复，但按要求添加)。
        - 添加 `TotalAllocQuantity` (int) 属性，表示 "发货数量 (Alloc Qty)"。

2.  **后端服务 (`StoreOrderReactService.cs`)**:
    - 在 `GetOrderListAsync` 方法中更新字段赋值逻辑：
        - `TotalAmount` = 子查询 Sum(`Quantity` * `OEMPrice`)。
        - `ImportTotalAmount` = `o.OEMTotalAmount` (数据库中存储的是 AllocQty * OEMPrice，即发货金额)。
        - `TotalOrderAmount` = 子查询 Sum(`Quantity` * `OEMPrice`)。
        - `TotalAllocQuantity` = 子查询 Sum(`AllocQuantity`)。
        - 保持 `OEMTotalAmount` = `o.OEMTotalAmount`。

3.  **前端定义 (`storeOrder.ts`)**:
    - 更新 `StoreOrderListItemDto` 接口，添加 `totalOrderAmount` 和 `totalAllocQuantity`。

4.  **前端页面 (`OrderList/index.tsx`)**:
    - 更新 `columns` 定义：
        - `Total Amount` 列标题改为 `Est. Sales ($)` (预计销售金额)。
        - `Amount ExGST` (`importTotalAmount`) 列标题改为 `Send Amount ($)` (发货金额)。
        - 添加 `Order Amount ($)` 列 (`totalOrderAmount`)。
        - 添加 `Send Qty` 列 (`totalAllocQuantity`)。
        - 调整列顺序。

## 验证
- 检查后端编译通过。
- 检查前端页面显示正确的列和数值。
