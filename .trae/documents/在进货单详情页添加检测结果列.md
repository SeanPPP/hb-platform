## 目标
- 让“进货单明细”页展示与“新建进货单”页一致的两列检测结果：
  - 检测结果（供应商+货号）
  - 条码匹配结果

## 改动范围
- 文件：`src/pages/PosAdmin/LocalSupplierInvoiceDetail/index.tsx`
- 仅在 `columns` 数组中新增两列，复用现有 `rowMeta` 中的 `supplierDetect` 与 `barcodeDetect` 数据。

## 实现内容
- 新增列：`检测结果`
  - 数据源：`rowMeta[r.detailGUID]?.supplierDetect`
  - 展示：
    - `error` 存在 → `<Tag color="error">{error}</Tag>`
    - `exists === true` → `<Tag color="green">已存在</Tag>`
    - 其他 → `<Tag>未找到</Tag>`
  - 排序：按 `exists` 布尔值排序（已存在在前）。
- 新增列：`条码匹配结果`
  - 数据源：`rowMeta[r.detailGUID]?.barcodeDetect`
  - 展示：
    - `error` 存在 → `<Tag color="error">{error}</Tag>`
    - `matched === true` → `<Tag color={overTwo ? 'orange' : 'green'}>{overTwo ? '匹配数>2' : '已匹配'}</Tag>` + `匹配数:{matchCount}`
    - 其他 → `<Tag>未匹配</Tag>`
  - 排序：先按 `matched`，再按 `matchCount`。

## 交互与数据
- 按钮“检测”已触发 `onDetectAll`，会填充 `rowMeta` 的 `supplierDetect` 与 `barcodeDetect`；无需改动接口或逻辑。
- 风格与逻辑保持与“新建进货单”页一致（不引入新依赖）。

## 验证
- 进入详情页，点击“检测”，表格应出现两列并展示检测结果；排序与高亮生效。
- 控制台无错误；页面体验一致。
