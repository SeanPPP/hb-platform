## 目标
- 在“自动价格策略管理”页面新增一个测试区域：选择分店/供应商、输入进货价，调用后端评估接口，返回“上浮率”和“零售价”并展示。

## 后端改动
- 新增评估端点（不影响现有CRUD）：
  - 控制器：`ReactPricingStrategiesController`
  - 路由：`POST /api/react/v1/pricing-strategies/evaluate`
  - 入参DTO：`purchasePrice: decimal`，`storeCode?: string`，`supplierCode?: string`
  - 返回DTO：`retailPrice: decimal`，`rate: decimal`（上浮率，按零售价/进货价计算），`strategyId?: string`（命中策略），`rule?: { minPrice, maxPrice, algorithm, startRate, endRate }`
  - 实现：注入并调用 `AutoPricingService.FindStrategyAsync(...)` + `CalculateRetailPrice(...)`，用 `retailPrice / purchasePrice` 计算 `rate`；同时从策略里找命中的规则返回简要信息。
- DTO位置：`BlazorApp.Shared/DTOs/PricingStrategyReactDtos.cs` 或新建对应文件，保持 React 接口一致性。

## 前端改动
- 服务：在 `reactumi/my-app/src/services/pricingStrategyService.ts` 添加 `evaluate` 方法，调用上述端点。
- UI：在 `reactumi/my-app/src/pages/PosAdmin/PricingStrategies/index.tsx` 顶部列表卡片下新增一个 `Card` 或在现有 `Card` 中添加一块“测试区域”。
  - 表单包含：`storeCode`、`supplierCode`（使用现有 `storeOptions`/`supplierOptions`）、`purchasePrice`（`InputNumber`）。
  - 按钮“计算”，点击后调用 `evaluate`，展示返回的 `rate` 与 `retailPrice`，并可显示命中策略/规则的简要说明。
  - 错误时使用 `message.error` 反馈。

## 验证
- 打开 `/pos-admin/pricing-strategies`，在测试区域选择分店/供应商并输入进货价，点击计算后，正确显示上浮率与零售价。
- 检查 Network 请求和响应格式无误；控制台无错误；边界场景（无策略/无规则）返回进货价与 `rate=1`。

## 说明
- 评估端点为只读，不修改任何数据；`rate` 通过 `retailPrice/purchasePrice` 计算，保持后端简单改动；如需严格输出未四舍五入的倍率，可后续在服务层增加返回倍率值。
