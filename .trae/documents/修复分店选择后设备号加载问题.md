## 修复方案

### 1. 后端优化（推荐）
在 `DeviceRegistrationController.cs` 的 `GetDevicesByStoreCode` 方法中添加 AutoMapper 映射：
```csharp
var devices = await _deviceService.GetDevicesByStoreCodeAsync(storeCode);
var deviceDtos = _mapper.Map<List<DeviceListItemDto>>(devices);  // 添加映射
return Ok(new { success = true, data = deviceDtos });
```

### 2. 前端服务类型定义修复
更新 `deviceService.ts` 中的 `DeviceDto` 接口，将属性名改为英文（与后端 DTO 一致）：
```typescript
export interface DeviceDto {
  id: number;
  hardwareId: string;
  systemDeviceNumber: string;  // 英文属性名
  storeCode: string;
  // ...
}
```

### 3. 前端组件使用更新
更新 `index.tsx` 中使用设备数据的地方，改为使用英文属性名：
```typescript
options={devices.map((d: any) => ({ 
  label: d.systemDeviceNumber,  // 改用英文
  value: d.systemDeviceNumber 
}))}
```

### 4. 添加错误处理和日志
在前端 `loadDevices` 函数中添加更详细的错误日志，方便调试：
```typescript
const res = await getDevicesByStoreCode(storeCode);
console.log('API Response:', res);  // 添加日志
if (res?.success) {
  console.log('Devices loaded:', res.data);  // 添加日志
  setDevices(res.data || []);
} else {
  console.error('API returned failure:', res?.message);
}
```
