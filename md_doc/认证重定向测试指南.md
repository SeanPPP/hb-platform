# 认证重定向测试指南

## 测试目标
验证 Blazor WebAssembly 应用中的认证失败自动跳转登录页面功能是否正常工作。

## 已完成的修复
1. ✅ **修复 IAuthorizationPolicyProvider 错误**
   - 在 `Program.cs` 中添加了 `AddAuthorizationCore()` 服务注册
   - 解决了 "Cannot provide a value for property 'AuthorizationPolicyProvider'" 错误

2. ✅ **简化 App.razor 中的未授权处理**
   - 将复杂的未授权UI替换为简单的 `RedirectToLogin` 组件
   - 避免了UI渲染问题，确保快速重定向

3. ✅ **创建 RedirectToLogin 组件**
   - 自动检测当前URL并重定向到登录页面
   - 传递 returnUrl 参数，支持登录后返回原页面
   - 避免在登录页面形成无限重定向

4. ✅ **增强登录页面功能**
   - 支持读取 returnUrl 查询参数
   - 登录成功后自动重定向到原页面或默认仪表板
   - 如果已登录用户访问登录页，自动重定向到目标页面

## 故障排除

### 如果重定向不工作
1. **检查控制台错误**
   - 打开浏览器开发者工具
   - 查看 Console 面板的错误信息

2. **检查网络请求**
   - 查看 Network 面板
   - 确认认证API调用是否成功

3. **检查认证状态**
   - 访问 `/auth-test` 页面查看详细认证信息

### 常见问题
1. **无限重定向循环**
   - 检查 `RedirectToLogin` 组件中的路径判断逻辑
   - 确保登录页面路径正确

2. **返回URL不正确**
   - 检查查询参数编码是否正确
   - 验证 `Microsoft.AspNetCore.WebUtilities` 是否正确导入

3. **授权服务未注册**
   - 确认 `Program.cs` 中包含 `AddAuthorizationCore()`

## 技术实现细节

### 认证流程
1. 用户访问受保护页面 (使用 `[Authorize]` 属性)
2. `AuthorizeRouteView` 检查认证状态
3. 如果未认证，触发 `NotAuthorized` 模板
4. `RedirectToLogin` 组件执行重定向逻辑
5. 重定向到 `/login?returnUrl=原页面路径`
6. 用户登录成功后，返回到原页面

### 关键组件
- **AuthorizeRouteView**: Blazor内置的路由授权组件
- **CustomAuthenticationStateProvider**: 自定义认证状态提供者
- **RedirectToLogin**: 自定义重定向组件
- **Login.razor**: 增强的登录页面，支持returnUrl

### 配置要点
```csharp
// Program.cs
builder.Services.AddAuthorizationCore(); // 必须添加
builder.Services.AddScoped<AuthenticationStateProvider, CustomAuthenticationStateProvider>();
```

```razor
<!-- App.razor -->
<AuthorizeRouteView RouteData="@routeData" DefaultLayout="@typeof(MainLayout)">
    <NotAuthorized>
        <RedirectToLogin />
    </NotAuthorized>
</AuthorizeRouteView>
```
