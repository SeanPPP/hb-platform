# 表格固定列功能测试指南

## 测试环境

- Ant Design Blazor 1.4.3
- .NET 8.0
- 现代浏览器（Chrome, Firefox, Safari, Edge）

## 测试步骤

### 1. 基础功能测试

#### 1.1 固定表头测试
1. 打开商品列表页面
2. 确保有足够的数据（超过600px高度）
3. **预期结果**：表头应该固定在顶部，滚动时表头保持可见

#### 1.2 左侧固定列测试
1. 横向滚动表格
2. **预期结果**：选择列、图片列、货号列应该始终固定在左侧

#### 1.3 右侧固定列测试
1. 横向滚动表格
2. **预期结果**：采购价列应该始终固定在右侧

### 2. 视觉效果测试

#### 2.1 阴影效果
1. 观察固定列的边缘
2. **预期结果**：应该有阴影效果分隔固定列和滚动列

#### 2.2 悬停效果
1. 鼠标悬停在固定列上
2. **预期结果**：固定列应该有背景色变化

### 3. 响应式测试

#### 3.1 桌面端测试
1. 在桌面浏览器中测试
2. **预期结果**：固定列功能正常，滚动流畅

#### 3.2 移动端测试
1. 在移动设备或模拟器中测试
2. **预期结果**：表格应该适配小屏幕，固定列可能隐藏或调整

### 4. 性能测试

#### 4.1 大数据量测试
1. 加载大量商品数据（1000+条）
2. **预期结果**：滚动应该保持流畅，无卡顿

#### 4.2 内存使用测试
1. 长时间使用表格
2. **预期结果**：内存使用应该稳定，无内存泄漏

## 已知问题

### 1. 版本兼容性
- 某些旧版本的Ant Design Blazor可能不支持`Fixed`属性
- 解决方案：升级到1.4.3或更高版本

### 2. 移动端适配
- 在小屏幕设备上，固定列可能影响可读性
- 解决方案：在移动端考虑隐藏部分固定列

### 3. 样式冲突
- 自定义CSS可能与Ant Design默认样式冲突
- 解决方案：使用`!important`确保样式优先级

## 故障排除

### 问题1：固定列不显示
**可能原因**：
- Ant Design版本过低
- CSS样式冲突
- 表格宽度不足

**解决方案**：
1. 检查Ant Design版本
2. 清除浏览器缓存
3. 检查CSS样式

### 问题2：滚动不流畅
**可能原因**：
- 数据量过大
- 浏览器性能问题
- CSS动画过多

**解决方案**：
1. 减少每页数据量
2. 优化CSS动画
3. 使用虚拟滚动

### 问题3：固定列样式异常
**可能原因**：
- CSS样式覆盖
- 浏览器兼容性问题

**解决方案**：
1. 检查CSS优先级
2. 添加浏览器前缀
3. 测试不同浏览器
