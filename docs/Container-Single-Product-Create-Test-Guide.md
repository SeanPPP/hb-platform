# 货柜明细 - 单个商品创建功能测试指南

## 功能说明
在货柜明细页面的表格中，对于检测为"新商品"的商品，可以通过点击"新商品 ➕"标签快速创建单个商品到仓库系统。

## 测试前提条件

1. ✅ 已登录系统
2. ✅ 已打开货柜明细页面（/yiwu-purchase/containers/{ContainerCode}）
3. ✅ 货柜状态为可编辑状态（IsEditable = true）
4. ✅ 已执行"检测商品"操作，确保有检测结果

## 测试用例

### 测试用例 1：成功创建新商品（完整信息）

**测试步骤：**
1. 点击页面上的"检测商品"按钮
2. 等待检测完成，查看表格中的"检测结果"列
3. 找到一个显示"新商品 ➕"标签的商品（绿色标签）
4. 确认该商品的进口价格和贴牌价格都已填写
5. 点击"新商品 ➕"标签

**预期结果：**
- ✅ 显示 loading 状态
- ✅ 弹出成功提示："✅ 成功创建商品：{货号}"
- ✅ 自动重新检测商品
- ✅ 该商品的检测结果标签变为"已存在"（蓝色标签）

---

### 测试用例 2：创建失败 - 缺少贴牌价格

**测试步骤：**
1. 点击页面上的"检测商品"按钮
2. 找到一个显示"新商品 ➕"标签的商品
3. 确认该商品的贴牌价格为空或为 0
4. 点击"新商品 ➕"标签

**预期结果：**
- ✅ 显示错误提示："商品 {货号} 缺少贴牌价格，请先设置后再创建"
- ✅ 商品状态不变，仍然显示"新商品 ➕"

**修复方法：**
1. 双击该商品的"贴牌价格"列单元格
2. 输入有效的价格（例如：15.99）
3. 按 Enter 键或点击其他地方失焦保存
4. 再次点击"新商品 ➕"标签

---

### 测试用例 3：已存在商品点击保护

**测试步骤：**
1. 找到一个显示"已存在"标签的商品（蓝色标签）
2. 尝试点击标签（注意：已存在的商品标签不可点击）

**预期结果：**
- ✅ 标签不可点击（没有 cursor: pointer 样式）
- ✅ 不会触发任何操作

---

### 测试用例 4：创建时商品已存在（并发冲突）

**测试场景：** 在检测后、创建前，该商品被其他用户创建

**测试步骤：**
1. 点击"检测商品"按钮
2. 找到一个"新商品 ➕"标签
3. （模拟：此时另一个用户创建了该商品）
4. 点击"新商品 ➕"标签

**预期结果：**
- ✅ 显示警告提示："⚠️ 商品 {货号} 在创建过程中发现已存在，已跳过"
- ✅ 自动重新检测商品
- ✅ 检测结果标签变为"已存在"（蓝色标签）

---

### 测试用例 5：网络错误处理

**测试场景：** API 调用失败

**测试步骤：**
1. （断开网络或模拟 API 错误）
2. 点击"新商品 ➕"标签

**预期结果：**
- ✅ 显示错误提示："❌ 创建失败：{错误信息}"
- ✅ Loading 状态正确结束
- ✅ 商品状态不变

## 功能特性验证

### 特性 1：价格自动填充
**验证点：**
- 在检测商品时，如果商品已存在且本地贴牌价格为空，会自动填充仓库商品的价格
- 代码位置：2502-2515 行

**测试步骤：**
1. 找到一个贴牌价格为空的商品
2. 点击"检测商品"按钮
3. 如果该商品在仓库中已存在且有贴牌价格

**预期结果：**
- ✅ 自动填充该商品的贴牌价格
- ✅ 显示提示："检测完成！...自动填充价格: {count} 个"

---

### 特性 2：单个创建与批量创建的区别
**单个创建优势：**
- 无需数学验证
- 立即创建，无需确认弹窗
- 快速便捷，适合逐个检查创建

**批量创建优势：**
- 一次性创建多个商品
- 有数学验证保护
- 适合大量商品创建

## 常见问题排查

### 问题 1：点击"新商品 ➕"标签没有反应
**排查步骤：**
1. 检查是否执行了"检测商品"操作
2. 检查浏览器控制台是否有 JavaScript 错误
3. 检查网络请求是否正常
4. 检查货柜是否为可编辑状态

### 问题 2：提示缺少贴牌价格
**解决方案：**
1. 双击"贴牌价格"列单元格进入编辑模式
2. 输入有效价格（必须 > 0）
3. 按 Enter 或失焦保存
4. 确认价格已显示后，再次点击创建

### 问题 3：创建成功但标签仍显示"新商品"
**排查步骤：**
1. 检查是否触发了自动重新检测
2. 手动点击"检测商品"按钮刷新状态
3. 检查后台日志确认商品是否真的创建成功
4. 检查数据库 WarehouseProduct 表
