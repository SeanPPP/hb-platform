# 批量新建商品功能设计方案

## 一、功能概述

支持批量新建商品,特别是套装礼品盒商品(商品类型=1),同时可以快速配置套装明细信息。

## 二、业务规则

### 2.1 商品类型
- **0**: 普通商品 - 单一SKU商品
- **1**: 套装商品 - 由多个商品组成的套装(需要配置套装明细)
- **2**: 多码商品 - 一个商品对应多个条码或规格

## 三、界面设计方案

### 方案一：分步向导式(推荐)

#### Step 1: 基本信息
- 选择供应商
- 输入商品数量
- 选择商品类型(普通/套装/多码)
- 可选：选择前缀代码

#### Step 2: 商品详情
- 批量输入商品信息(表格形式)
- 字段：商品名称、英文名称、规格、价格等

#### Step 3: 套装配置(仅当类型=套装时显示)
- 选择套装规格(套10/套15/自定义)
- 展示套装明细表格
- 支持快速填充默认价格
- 支持手动调整价格

#### Step 4: 确认创建
- 预览所有信息
- 确认并提交

### 方案二：单页表单

#### 区域划分
1. **顶部区域**: 供应商选择、前缀代码
2. **商品列表区域**: 批量商品信息表格
3. **套装配置区域**: 当商品类型=套装时动态展示
4. **操作区域**: 保存、取消按钮

## 四、技术实现要点

### 4.1 前端实现
- 使用Ant Design的Steps组件实现分步向导
- 使用EditableProTable实现可编辑表格
- 动态表单验证
- 套装价格模板快速应用

### 4.2 后端接口
- POST `/api/react/domestic-products/batch-create-with-sets`
- 请求体结构:
```json
{
  "supplierCode": "SUP001",
  "prefixCode": "HB",
  "products": [
    {
      "productName": "礼品盒A",
      "productType": 1,
      "setConfig": {
        "setType": "套10",
        "items": [
          {"domesticPrice": 2.5},
          {"domesticPrice": 2.99},
          ...
        ]
      }
    }
  ]
}
```

### 4.3 数据库设计
- 主表: `DomesticProduct`
- 套装明细表: `DomesticSetProduct`
- 一对多关系: 一个套装商品 -> 多个套装明细

## 五、用户交互流程

```
开始
  ↓
选择供应商
  ↓
选择商品类型 → 是套装? 
  ↓ 是               ↓ 否
选择套装规格      直接填写商品信息
(套10/套15)           ↓
  ↓                确认创建
自动填充默认价格      ↓
  ↓                完成
手动调整(可选)
  ↓
确认创建
  ↓
完成
```

## 六、后续优化建议

1. **模板功能**: 保存常用的套装配置模板
2. **Excel导入**: 支持从Excel批量导入商品和套装信息
3. **快速复制**: 复制已有商品的套装配置
4. **价格批量调整**: 支持按比例批量调整套装价格
5. **审批流程**: 大批量创建时需要审批

## 七、注意事项

1. 套装商品必须先存在基础商品才能创建套装明细
2. 套装价格可以与基础商品价格不同
3. 套装条码和货号需要自动生成,避免冲突
4. 创建失败时需要事务回滚,保证数据一致性
