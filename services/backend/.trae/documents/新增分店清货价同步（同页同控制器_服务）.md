## 目标
- 在现有 React 数据同步页面与 React 专用控制器/服务中新增同步项：
  - DIC_商品清货价表 → HBweb.StoreClearancePrice（按分店并发，分店内分页，批量写入）
- 与现有商品/零售价/一品多码/套装多码保持一致的返回与耗时统计；支持选择分店。

## 后端
- 控制器（复用 DataSyncReactController）新增端点：
  - `POST /api/react-sync/store-clearance-prices`（请求体 `{ selectedStoreCodes?: string[] }`）
  - 返回 `ApiResponse<SyncResult>`；权限 `[Authorize(Roles="Admin")]`
- 服务接口（扩展 IDataSyncReactService）：
  - `Task<SyncResult> SyncStoreClearancePricesFromHqConcurrentAsync(List<string>? storeCodes, int maxConcurrency = 12, int batchSize = 200000)`
- 服务实现（扩展 DataSyncReactService）：
  - 分店并发：每分店独立 HQ/本地连接；分店内分页 JOIN/过滤；查询失败指数退避重试并重建连接
  - 本地清理：仅删除选中分店的 `StoreClearancePrice` 历史数据
  - 映射后批量写入：`Fastest<StoreClearancePrice>().BulkCopyAsync(PageSize=50000)`；插入失败重试
  - 统计：`processed/added/errors/query/insert` 耗时；前端展示耗时与结果
- 映射（新增 React 专用 Profile）：
  - `ReactStoreClearancePriceMappingProfile`
    - `StoreCode ← 分店代码`
    - `ProductCode ← 商品编码`
    - `ClearanceBarcode ← 清货条形码`
    - `ClearancePrice ← 清货价`
    - 可选：`CreatedAt/UpdatedAt ← FGC_CreateDate/FGC_LastModifyDate`
    - 忽略导航属性；对字符串字段进行长度截断（如 50）与 `Trim`

## 前端
- 服务层（`src/services/dataSync.ts`）：
  - 新增 `syncStoreClearancePrices(selectedStoreCodes?: string[])` → `POST /api/react-sync/store-clearance-prices`；`timeout 60–90 分钟`
- 页面（`src/pages/SystemSettings/HqDataSync/index.tsx`）：
  - 新增卡片与按钮：“分店清货价（DIC_商品清货价表 → StoreClearancePrice）”
  - 分店多选（与零售价、一品多码一致），执行同步并显示结果统计与耗时

## 并发与稳健性
- 并发度默认 12（可配置）；分店内分页 `batchSize=200000`；必要时下调并发以适应 HQ 负载
- 查询失败指数退避重试并重建连接；插入失败批次重试；日志输出每页拉取与插入耗时、失败原因与统计
