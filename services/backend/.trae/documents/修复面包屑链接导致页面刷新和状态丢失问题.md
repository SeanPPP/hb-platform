# 解决点击分类面包屑导致分店选择器清空的问题

## 问题分析
用户反馈：点击分类面包屑（`<a>` 标签链接）后，页面刷新，导致分店选择器变为空。
原因：
1.  **HTML 链接行为**: 截图显示面包屑使用的是标准的 `<a href="...">` 标签。
    *   `<a href="/store?category=...">` 会导致浏览器进行**整页刷新 (Full Page Refresh)**。
    *   整页刷新会重新加载 React 应用，导致所有非持久化的状态丢失。
2.  **状态丢失**: `initialState.selectedStore` 虽然存储在全局状态中，但如果它没有被持久化到 `localStorage` 或 `sessionStorage`，刷新后就会重置为初始值（通常是空或 null）。
3.  **代码检查**:
    *   `StoreLayout.tsx` 中使用了 `Breadcrumb` 组件。
    *   检查 `StoreHome/index.tsx`（面包屑渲染的地方），代码显示：
        ```tsx
        <Breadcrumb.Item key={c.categoryGUID}>
          <a href={`/store?category=${c.categoryGUID}`}>{c.categoryName}</a>
        </Breadcrumb.Item>
        ```
    *   这里直接使用了 `<a>` 标签，确实会导致刷新。

## 解决方案
1.  **使用路由跳转**: 将 `<a href="...">` 替换为 React Router 的 `<Link to="...">` 或使用 `history.push` / `useNavigate` 的点击事件。这样可以进行**客户端路由跳转 (Client-side Routing)**，不会触发布局卸载和状态重置。
2.  **持久化分店选择**: 虽然改为客户端路由能解决通过点击面包屑导致的状态丢失，但为了更好的用户体验（防止用户手动 F5 刷新导致丢失），建议确保 `selectedStore` 是持久化的。
    *   `StoreLayout.tsx` 中在加载时会调用 `fetchUserStores`。
    *   `app.ts` 的 `getInitialState` 通常会恢复登录用户信息，但不一定恢复了 `selectedStore`。
    *   在 `StoreLayout.tsx` 中，如果 `initialState.selectedStore` 为空，代码逻辑并没有尝试从 `localStorage` 恢复。

## 实施计划

### 1. 修改 `src/pages/StoreHome/index.tsx`
*   将面包屑中的 `<a href="...">` 替换为 `Link` 组件（来自 `@umijs/max` 或 `react-router-dom`）。
*   或者使用 `onClick={() => history.push(...)}`。

### 2. (可选) 增强分店持久化
*   在 `StoreLayout.tsx` 中，当用户选择分店时，将其保存到 `localStorage`。
*   在 `StoreLayout` 初始化时，尝试从 `localStorage` 读取分店信息并更新到 `initialState`。
*   *注*: 这属于额外优化，首要任务是修复链接刷新问题。

## 预期结果
点击分类面包屑时，页面将平滑切换分类，不会整页刷新，分店选择器状态将保持不变。
