## 调整点
- 条码匹配结果改为单独一列展示，不与供应商+货号检测结果合并，避免信息混杂。

## 前端页面
- 新建页：`PosAdmin/LocalSupplierInvoiceCreate/index.tsx`
- 表单必填：`storeCode`、`supplierCode`、`invoiceNo`。
- 粘贴解析：支持列【货号｜名称或条码｜数量｜价格】，校验数量为整数、价格>0。
- 明细表格列：
  - 固定左侧：图片、货号。
  - 基础列：商品名称、条码、数量、进货价、金额、零售价、折扣率、自动定价、特殊商品、定价浮率、新自动零售价、活动类型、已存在商品数。
  - 结果列（两列）：
    - `供应商+货号检测结果`
    - `条码匹配结果`（单独列，只显示条码检测的命中/未命中、匹配数量、是否>2）
- 按钮：
  - “检测（供应商+货号）”→ 调用供应商+货号检测接口，仅更新对应结果列。
  - “检测（条码）”→ 调用条码检测接口，仅更新“条码匹配结果”列。
  - “保存进货单”→ 创建主表与明细。

## 前端类型与服务
- 类型：在 `src/types/localSupplierInvoice.ts` 增加：
  - `SupplierItemDetectResult { exists, productCode?, productName?, currentPurchasePrice?, currentRetailPrice? }`
  - `BarcodeDetectResult { matched, matchCount, overTwo, productCodes?: string[], productNames?: string[] }`
- 服务：`src/services/localSupplierInvoiceService.ts` 增加：
  - `createInvoice(data)`、`bulkDetectSupplierItem(payload)`、`bulkDetectBarcode(payload)`。

## 后端接口与实现
- 控制器：`ReactLocalSupplierInvoicesController`
  - `POST /api/react/v1/local-supplier-invoices`：创建主表与明细（校验必填与明细合法）。
  - `POST /api/react/v1/local-supplier-invoices/detect/supplier-item`：批量返回供应商+货号检测结果。
  - `POST /api/react/v1/local-supplier-invoices/detect/barcode`：批量返回条码匹配结果（单独列专用数据）。
- 服务：`ILocalSupplierInvoicesReactService` 新增上述 3 方法；实现于 `LocalSupplierInvoicesReactService`：
  - 供应商+货号：`Product.LocalSupplierCode == supplierCode && Product.ItemNumber == itemNumber` 命中后查 `StoreRetailPrice.StoreCode == storeCode && ProductCode == product.ProductCode`。
  - 条码匹配：命中 `Product.Barcode == barcode` 或 `StoreMultiCodeProduct.StoreCode == storeCode && MultiBarcode == barcode`，统计 `matchCount`，`overTwo = matchCount > 2`。
