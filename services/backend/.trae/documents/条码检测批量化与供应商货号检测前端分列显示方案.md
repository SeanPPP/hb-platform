## 目标
- 条码检测批量化：一次性查询产品与一品多码，避免逐条数据库访问，提升性能与稳定性。
- 供应商货号检测分列显示：将当前“供应商+货号检测结果”合并列拆分为多列，信息更清晰。

## 后端改造（批量条码检测）
- 文件：BlazorApp.Api/Services/React/LocalSupplierInvoicesReactService.cs
- 方法：DetectBarcodeAsync(DetectBarcodeRequest dto)
- 实现要点：
  - 收集条码列表：`var barcodes = dto.Items.Select(b => b.Barcode?.Trim()).Where(not empty).Distinct().ToList()`。
  - 批量查询产品：`Product` where `Barcode IN barcodes AND IsDeleted=false` → 投影 `{ Barcode, ProductCode, ProductName }`。
  - 批量查询多码：`StoreMultiCodeProduct` where `StoreCode == dto.StoreCode AND MultiBarcode IN barcodes AND IsDeleted=false` 左联 `Product` → 投影 `{ MultiBarcode as Barcode, ProductCode, ProductName }`。
  - 合并结果并建立 `barcode -> { productCodes[], productNames[], matchCount }` 字典。
  - 返回时按输入顺序生成 `BarcodeDetectResult[]`：`matched = matchCount > 0`、`overTwo = matchCount > 2`、携带 `productCodes`、`productNames`。
  - 边界：空条码列表直接返回对应数量的未匹配结果；对名称 null 做去空与去重处理。

## 前端改造（供应商货号检测分列显示）
- 文件：ReactUmi/my-app/src/pages/PosAdmin/LocalSupplierInvoiceCreate/index.tsx
- 列修改：
  - 移除现有合并列“供应商+货号检测结果”。
  - 新增分列：
    - `检测结果`：Tag 显示“已存在 / 未找到”。
    - `商品编码`：显示 `SupplierItemDetectResult.productCode`。
    - `商品名称`：显示 `SupplierItemDetectResult.productName`。
    - `当前进货价`：显示 `SupplierItemDetectResult.currentPurchasePrice`（两位小数）。
    - `当前零售价`：显示 `SupplierItemDetectResult.currentRetailPrice`（两位小数）。
- 类型：沿用已定义的 `SupplierItemDetectResult`，无需变更。
- 交互：保持禁用逻辑（需选择分店与供应商后检测）与错误提示不变。

## 验证
- 批量条码检测：
  - 多条条码一次请求返回，无 SQL 错误，无逐条循环。
  - 匹配数与超过2的标注正确。
- 分列显示：
  - 检测后各列正确显示编码、名称、进价与零售价；未命中时仅显示“未找到”。

- 后端：仅限条码检测方法内部查询方式；接口契约保持不变。
- 前端：仅修改创建页的列定义；其他页面不受影响。
