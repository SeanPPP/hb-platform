## 销售数据看板实现方案（使用 Hangfire 管理定时任务）

### 一、数据架构设计

#### 1. 统计表设计

**每日销售统计表** (`DailySalesStatistic`)

* `Date` (日期)

* `TotalAmount` (总金额)

* `TotalQuantity` (总数量)

* `OrderCount` (订单数)

* `SkuCount` (SKU数)

* `CustomerCount` (客单数)

* `AverageOrderValue` (客单价)

* `UpdateTime` (更新时间)

**分时销售统计表** (`HourlySalesStatistic`) - 含分店维度

* `Date` (日期)

* `Hour` (小时 0-23)

* `BranchCode` (分店代码，NULL表示全部分店汇总)

* `BranchName` (分店名称，可选)

* `TotalAmount` (总金额)

* `TotalQuantity` (总数量)

* `CustomerCount` (客单数)

* `AverageOrderValue` (客单价)

* `UpdateTime` (更新时间)

**分店销售统计表** (`StoreSalesStatistic`)

* `Date` (日期)

* `BranchCode` (分店代码)

* `BranchName` (分店名称)

* `TotalAmount` (总金额)

* `TotalQuantity` (总数量)

* `OrderCount` (订单数)

* `CustomerCount` (客单数)

* `AverageOrderValue` (客单价)

* `UpdateTime` (更新时间)

**供应商销售统计表** (`SupplierSalesStatistic`) - 不含分店维度

* `Date` (日期)

* `SupplierCode` (供应商代码)

* `SupplierName` (供应商名称)

* `IsDomestic` (是否国内供应商)

* `TotalAmount` (总金额)

* `TotalQuantity` (总数量)

* `StoreCount` (涉及分店数)

* `UpdateTime` (更新时间)

**分店供应商明细表** (`StoreSupplierSalesDetail`) - 可选

* `Date` (日期)

* `BranchCode` (分店代码)

* `SupplierCode` (供应商代码)

* `SupplierName` (供应商名称)

* `IsDomestic` (是否国内供应商)

* `TotalAmount` (总金额)

* `TotalQuantity` (总数量)

* `UpdateTime` (更新时间)

### 四、Hangfire 配置详情

#### Program.cs 配置

```csharp
// 配置 Hangfire
builder.Services.AddHangfire(config =>
{
    config
        .UseSimpleAssemblyNameTypeSerializer()
        .UseRecommendedSerializerSettings()
        .UseSqlServerStorage(
            builder.Configuration.GetConnectionString("DefaultConnection"),
            new SqlServerStorageOptions
            {
                QueuePollInterval = TimeSpan.FromSeconds(15)
            });
});

builder.Services.AddHangfireServer();
builder.Services.AddScoped<SalesStatisticsJobService>();

app.UseHangfireDashboard("/hangfire", new DashboardOptions
{
    Authorization = new[] { new HangfireAuthorizationFilter() }
});
```

#### 定时任务注册

```csharp
// 每小时更新当前小时统计
RecurringJob.AddOrUpdate<SalesStatisticsJobService>(
    "UpdateCurrentHourStatistics",
    service => service.UpdateCurrentHourStatistics(),
    Cron.Hourly);

// 每天凌晨3点全量刷新前一天数据
RecurringJob.AddOrUpdate<SalesStatisticsJobService>(
    "FullRefreshPreviousDay",
    service => service.FullRefreshPreviousDay(),
    "0 3 * * *");
```

#### Hangfire 权限配置

* 创建 `BlazorApp.Api/Hangfire/HangfireAuthorizationFilter.cs`

* 仅管理员可访问 `/hangfire` 仪表板

### 五、Hangfire 任务设计

#### 任务1：更新当前小时统计

* **频率**：每小时执行一次

* **功能**：

  * 聚合当前小时的分时数据（所有分店）

  * 聚合当前小时各分店的分时数据

  * 更新每日汇总数据

#### 任务2：全量刷新前一天数据

* **频率**：每天凌晨3点

* **功能**：

  * 重新计算前一天的每日统计

  * 重新计算前一天的24小时分时统计

  * 重新计算前一天的各分店统计

  * 重新计算前一天的供应商统计

#### 手动触发任务

* 通过 Hangfire Dashboard 手动触发任意任务

* 或创建 API 端点：`POST /api/admin/refresh-statistics?date=`

### 七、Hangfire 优势

1. **可视化仪表板**：访问 `/hangfire` 查看任务执行状态
2. **手动触发**：可以随时手动触发统计更新
3. **执行历史**：查看每次任务的执行日志和结果
4. **失败重试**：自动重试失败的任务
5. **分布式支持**：未来可扩展多实例部署
6. **任务管理**：暂停/恢复/删除定时任务
