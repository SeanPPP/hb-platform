# 实现分类点击递归显示商品

## 问题分析
用户要求点击分类时，应显示该分类及其所有递归子分类下的所有商品。
目前的实现：
1.  `StoreHome/index.tsx` 获取商品时传递了 `categoryGUID: categoryId`。
2.  `StoreLayout.tsx` 的导航菜单点击后，URL 更新为 `/store?category={GUID}`。
3.  后端 API `getStoreOrderProducts` 接收 `categoryGUID`。

关键在于后端 API 的行为。如果后端 API 仅匹配指定的 `categoryGUID` 而不包含子分类，那么前端就需要做调整。
但通常这种逻辑应该在后端处理（即查询时包含所有子分类 ID）。
**如果必须在前端处理**（假设后端只支持精确匹配）：
1.  前端需要获取分类树。
2.  找到当前分类的所有子孙分类 ID。
3.  将这些 ID 列表传给后端（如果后端支持 `categoryGUIDs` 数组）。或者后端已经支持递归查询（通常是默认行为）。

**另一种可能性**：`StoreHome/index.tsx` 中的 `getStoreOrderProducts` 调用传递参数的方式。
目前代码：
```typescript
      const res = await getStoreOrderProducts({
        // ...
        categoryGUID: categoryId || undefined,
        // ...
      });
```
如果后端接口支持递归，这应该足够了。
但如果用户反馈“应该可以显示”，暗示目前可能**没有**显示子分类商品。

让我们假设后端需要前端传递“是否包含子分类”的标志，或者需要前端自己收集所有子分类 ID。
查看 `src/services/storeOrder.ts` 定义：
```typescript
export interface StoreOrderFilterParams {
  // ...
  categoryGUID?: string;
  // ...
}
```
并没有 `includeSubCategories` 这样的参数。

**前端解决方案**:
在 `StoreHome/index.tsx` 中，当有 `categoryId` 时：
1.  获取分类树（已经有了 `getCategoryTree`）。
2.  找到目标分类及其所有子分类的 GUID。
3.  如果后端支持传递多个 ID，则传递数组。如果不支持，可能需要后端修改。
    *   *检查 API 定义*: `StoreOrderFilterParams` 只有一个 `categoryGUID?: string`。
    *   这意味着后端逻辑决定了是否递归。

**如果后端逻辑是“只查当前分类”**，而我们需要“查递归”，通常后端会提供一个 `Recursive` 参数，或者前端无法单纯通过 `categoryGUID` 实现（除非后端改）。
**但是**，还有一种情况：前端 `getStoreOrderProducts` 并没有把所有子分类的商品聚合起来。

**确认用户意图**: "点击分类 应该可以 显示该分类所有商品 包括所有递归子分类"
这听起来像是一个功能需求，暗示目前**只显示了当前分类**的商品。

**如果后端不支持递归**，前端能做什么？
前端无法通过一次 API 调用获取所有子分类商品，除非并发请求（不推荐）。
