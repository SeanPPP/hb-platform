## 实施目标
- 在“供应商管理”页新增“新建供应商”弹窗与创建流程。
- 供应商编码由后端自动生成，格式 `SP###`（三位零填充），从 `SP001` 开始递增。

## 前端改动（ReactUmi/my-app）
- 页面：`src/pages/PosAdmin/SupplierManagement/index.tsx`
  - 工具栏新增按钮“新建供应商”。
  - 打开 `Modal + Form`（vertical）：
    - 字段：`name`（必填）、`status`（Switch，默认启用）、`contactPerson`、`phone`、`email`（格式校验）、`remark`。
    - 不显示编码输入框（只读由后端生成）。
  - 提交：调用 `createLocalSupplier(dto)`；成功后 `message.success("创建成功：编码 SPxxx")`，关闭弹窗并调用 `load()` 刷新列表（保留当前排序与分页）。
- 服务：`src/services/localSupplier.ts`
  - 新增 `createLocalSupplier(data)` 对接 `POST /api/react/v1/local-suppliers`。

## 后端改动（BlazorApp.Api）
- 服务层：`Services/React/LocalSupplierReactService.cs`
  - 在 `CreateAsync` 内生成编码：
    - 事务开始 → 查询最大 `LocalSupplierCode` 前缀 `SP`（如：`SP012`）→ 解析后三位数字 `12` → `+1` → `D3` 零填充 → `SP013`。
    - 插入实体（唯一索引约束 `LocalSupplierCode`）。
    - 并发处理：捕获唯一冲突时重试一次（重新查询+生成）；仍失败则返回错误。
    - 返回 `ApiResponse<LocalSupplierDto>`（包含 `LocalSupplierCode`）。
- 控制器：`Controllers/React/LocalSuppliersController.cs`
  - 保持 `POST /api/react/v1/local-suppliers` 接口不变，返回 `{ success, data, message }`。
- 上下文与索引（已部分完成）：
  - `SqlSugarContext` 中已注册 `HBLocalSupplier` SimpleClient 与 CodeFirst 初始化；确保唯一索引 `IX_LocalSupplier_Code_Unique` 存在。

## 排序联动验证
- 列点击排序后，前端通过 `onChange` 更新 `sortBy/sortOrder` 并触发 `load()`；后端根据字段映射 `localsuppliercode/name/status/...` 返回对应排序。
- 默认排序保留“名称升序”。

## 验证用例
- 连续“新建供应商”两次：编码应为 `SP001`、`SP002`。
- 并发或快速点击创建：无重复编码；若冲突则返回错误并提示。
- 列表刷新显示新记录，支持代码/名称等列排序；隔行色与表格内滚动保持体验。

## 注意事项
- 统一 JSON 序列化使用 camelCase（`localSupplierCode`）。
- Email/长度校验在前端与后端都进行。
- 未来超过 `SP999` 时可扩展为四位或以上零填充（`SP1000`）。
