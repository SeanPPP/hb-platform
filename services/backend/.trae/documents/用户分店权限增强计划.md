# 用户分店权限增强计划

## 需求分析

修改 `GetUserStoresAsync` 方法，使得：
- 如果用户是管理员（Admin）或仓库管理员（Warehouse），返回系统中所有分店
- 普通用户仍然返回其关联的分店列表

## 现状分析

### 相关代码位置
1. **Controller层**: `UsersController.cs` L413-434
   - 端点: `GET /api/users/guid/{guid}/stores`
   - 授权注释已暂时关闭

2. **Service层**: `UserService.cs` L1107-1143
   - 方法: `GetUserStoresAsync(string userGuid)`
   - 当前逻辑：只返回用户通过 UserStore 表关联的分店

3. **角色系统**
   - 用户模型：`User.cs` 包含多对多角色关联
   - 角色模型：`Role.cs` 定义角色信息
   - 用户角色关联：`UserRole.cs`
   - 已有角色：Admin（管理员）、Manager（经理）、WarehouseManager、Warehouse（仓库管理员）

4. **参考实现**
   - `ReactStoreOrderController.cs` L45-54：已有 `IsAdminOrWarehouseAdmin()` 方法
   - 检查角色："Admin" 或 "Warehouse"
   - `ReactStoreOrderController.cs` L61-86：管理员和仓库管理员返回所有分店

### 角色判断逻辑
根据现有代码，管理员和仓库管理员的角色名称为：
- `Admin` - 系统管理员
- `Warehouse` - 仓库管理员

## 修改方案

### 方案概述
修改 `UserService.GetUserStoresAsync` 方法，添加角色判断逻辑：
1. 先获取用户的角色列表
2. 检查用户是否为 Admin 或 Warehouse 角色
3. 如果是特殊角色，返回所有分店
4. 否则，返回用户关联的分店（现有逻辑）

### 需要修改的文件

#### 1. BlazorApp.Api/Services/UserService.cs
**修改内容：**
- 在 `GetUserStoresAsync` 方法中添加角色判断逻辑
- 需要注入 `IStoreService` 依赖（如果未注入）

**具体实现步骤：**
1. 在方法开始处，先查询用户的角色
2. 判断用户角色是否为 "Admin" 或 "Warehouse"
3. 如果是特殊角色：
   - 查询所有分店
   - 为每个分店设置 `IsPrimary = false` 和 `AssignedAt = DateTime.UtcNow`（因为没有关联记录）
4. 如果不是特殊角色：
   - 保持现有逻辑，从 UserStore 表查询关联的分店

**伪代码：**
```csharp
public async Task<ApiResponse<List<UserStoreDto>>> GetUserStoresAsync(string userGuid)
{
    try
    {
        var db = _context.Db;

        // 获取用户角色
        var userRoles = await db.Queryable<UserRole>()
            .InnerJoin<Role>((ur, r) => ur.RoleGUID == r.RoleGUID)
            .Where((ur, r) => ur.UserGUID == userGuid)
            .Select((ur, r) => r.RoleName)
            .ToListAsync();

        // 判断是否为管理员或仓库管理员
        var isAdminOrWarehouse = userRoles.Contains("Admin") || userRoles.Contains("Warehouse");

        List<UserStoreDto> stores;

        if (isAdminOrWarehouse)
        {
            // 返回所有分店
            var allStores = await db.Queryable<Store>()
                .Select(s => new UserStoreDto
                {
                    StoreGUID = s.StoreGUID,
                    StoreName = s.StoreName,
                    StoreCode = s.StoreCode,
                    IsPrimary = false, // 管理员对所有分店无主分店概念
                    AssignedAt = DateTime.UtcNow
                })
                .ToListAsync();
            stores = allStores;
        }
        else
        {
            // 返回用户关联的分店（现有逻辑）
            stores = await db.Queryable<UserStore>()
                .InnerJoin<Store>((us, s) => us.StoreGUID == s.StoreGUID)
                .Where((us, s) => us.UserGUID == userGuid)
                .Select((us, s) => new UserStoreDto
                {
                    StoreGUID = s.StoreGUID,
                    StoreName = s.StoreName,
                    StoreCode = s.StoreCode,
                    IsPrimary = us.IsPrimary,
                    AssignedAt = us.CreatedAt,
                })
                .ToListAsync();
        }

        return ApiResponse<List<UserStoreDto>>.OK(stores, "获取用户分店成功");
    }
    catch (Exception ex)
    {
        // 错误处理...
    }
}
```

#### 管理员返回所有分店的实现
直接查询 Store 表，不关联 UserStore：
```csharp
var allStores = await db.Queryable<Store>()

```

#### 数据结构说明
- `UserStoreDto` 包含：`StoreGUID`, `StoreName`, `StoreCode`, `IsPrimary`, `AssignedAt`
- 管理员返回所有分店时，`IsPrimary` 设置为 `false`，因为没有实际关联记录
- `AssignedAt` 设置为当前时间，保持数据一致性
