## 一、accessKey 如何映射到权限码

`accessKey` 是 `AccessControl` 接口的布尔属性名，不是直接的后端权限码。
每个 `accessKey` 对应 `access.ts` 中一段计算逻辑，展开后对应具体权限码：

| accessKey | 计算逻辑 | 实际后端的权限码 |
|-----------|----------|------------------|
| ✅ 无 accessKey | `return true` | 无（所有人可见） |
| `isAdmin` | `isAdmin === true` | 角色 `Admin` 或 `管理员` |
| `canReadStore` | `isAdmin \|\| hasPermission('Stores.View')` | `Stores.View` |
| `canReadUser` | `isAdmin \|\| hasPermission('Users.View')` | `Users.View` |
| `canReadRole` | `isAdmin \|\| hasPermission('Roles.View')` | `Roles.View` |
| `canReadOrder` | `isAdmin \|\| hasPermission('Orders.View')` | `Orders.View` |
| `canReadProduct` | `isAdmin \|\| hasPermission('Products.View')` | `Products.View` |
| `canManageWarehouse` | `isAdmin \|\| hasPermission('Warehouse.Manage')` | `Warehouse.Manage` |
| `canManageStore` | `isAdmin \|\| hasPermission('Stores.Edit') \|\| hasPermission('Warehouse.Manage')` | `Stores.Edit` 或 `Warehouse.Manage` |
| `canViewReports` | `isAdmin \|\| hasPermission('Reports.View')` | `Reports.View` |

## 三、权限码汇总（按模块）

## 四、常见角色权限配置示例

| 角色 | 需分配的权限码 |

| **Order（订货员）** | 不需要后台权限（进入 `/shop` 前台） |
| **仓库经理** | `Warehouse.Manage` `Products.View` `Orders.View` |
| **店铺经理** | `Stores.Edit` `Stores.View` `Orders.View` `Products.View` `Reports.View` |
| **系统管理员** | `Admin` 角色（一切全开） |
| **报表查看员** | `Reports.View` |
