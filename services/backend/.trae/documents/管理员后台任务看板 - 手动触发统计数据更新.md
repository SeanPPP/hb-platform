# 实现计划：管理员后台任务看板

## 功能概述
为管理员创建一个专用页面，可以手动触发指定分店和供应商、指定日期的销售统计数据更新到中间表。

## 后端实现

### 1. 扩展 `SalesStatisticsJobService.cs`
添加新方法支持按分店和供应商筛选更新：
- `UpdateStoreStatistics(DateTime date, List<string> branchCodes)` - 更新指定分店的统计
- `UpdateSupplierStatistics(DateTime date, List<string> supplierCodes)` - 更新指定供应商的统计

### 2. 创建 `StatisticsJobTriggerController.cs`（新文件）
创建专门的控制器用于手动触发统计任务：
- `POST /api/StatisticsJob/trigger-store` - 触发分店统计更新
- `POST /api/StatisticsJob/trigger-supplier` - 触发供应商统计更新
- `POST /api/StatisticsJob/trigger-daily` - 触发每日统计更新
- `GET /api/StatisticsJob/job-status` - 获取任务执行状态

### 3. 注册服务（`Program.cs`）
注册 `StatisticsJobTriggerService` 到 DI 容器

## 前端实现

### 4. 创建 `ReactUmi/my-app/src/pages/StatisticsJobTrigger/index.tsx`
创建任务触发页面，包含：
- 日期范围选择器（DatePicker.RangePicker）
- 分店多选下拉框（StoreService 获取分店列表）
- 供应商多选下拉框（SupplierService 获取供应商列表）
- 任务类型选择（分店统计/供应商统计/每日统计）
- 触发按钮
- 任务执行状态列表（使用 ProTable 展示 Hangfire 任务历史）

### 5. 创建 `ReactUmi/my-app/src/services/statisticsJob.ts`
创建前端服务文件，包含 API 调用：
- `triggerStoreJob(params)` - 触发分店统计任务
- `triggerSupplierJob(params)` - 触发供应商统计任务
- `triggerDailyJob(params)` - 触发每日统计任务
- `getJobStatus()` - 获取任务状态

### 6. 添加路由配置（`.umirc.ts`）
在系统菜单下添加新路由 `/statistics-job-trigger`

### 7. 添加国际化翻译
在 `locales/zh-CN.ts` 和 `locales/en-US.ts` 添加相关翻译

## 技术要点
- 使用 Hangfire 的 `BackgroundJob.Enqueue()` 触发一次性任务
- 任务执行使用现有的 `SalesStatisticsJobService` 方法
- 前端使用 Ant Design 的 ProTable 和 Form 组件
- 支持任务执行状态查询和实时更新
