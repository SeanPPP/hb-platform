## 目标
- 图片列显示规则：仅当“供应商+货号检测命中”时显示商品图片；即使条码匹配命中但供应商未命中，也不显示图片。
- 紫色高亮：当“供应商+货号命中”且“条码未命中”时，整行紫色高亮。

## 修改文件
- ReactUmi/my-app/src/pages/PosAdmin/LocalSupplierInvoiceCreate/index.tsx

## 具体实现
- 图片列渲染：
  - `const d = r.supplierDetect`，仅在 `d?.exists === true && /^https?:\/\//.test(d.productImage||'')` 条件下使用 `d.productImage`；移除对条码结果图片的回退。
- 行高亮：
  - 在 `rowClassName` 增加紫色条件：`supplierDetect?.exists === true && barcodeDetect?.matched === false` → 返回 `row-purple`。
  - 样式注入：`.row-purple td { background-color: #f9f0ff !important; }`（浅紫色）。
- 现有高亮保留：
  - 黄色：`supplierDetect?.exists === false && barcodeDetect?.matched === true`（警示）
  - 绿色：`supplierDetect?.exists === false`（新商品）
  - 红色：批内重复（条码或货号重复）

## 验证
- 供应商未命中且条码命中：图片列不显示（占位图），整行黄色高亮。
- 供应商命中且条码未命中：整行紫色高亮，图片正常显示（http/https）。
- 其他情况：按既有规则显示高亮与图片。

## 影响范围
- 仅前端创建页逻辑与样式，后端无改动。
