## 问题研判
- 后端控制器对网格请求在服务失败时返回 400；服务失败大概率来源于过滤参数与服务端预期不一致。
- 目前页面仍按名称构造过滤：`filterModel.storeName/supplierName`（index.tsx:140-142），选择下拉时可能传入对象或名称字符串，导致等值比较失败或表达式异常。

## 修复方案
1) 前端改键与取值
- 将表单字段与过滤键从 `storeName/supplierName` 改为 `storeCode/supplierCode`（index.tsx 表单、buildGridRequest）。
- 下拉选项 `value` 用代码：分店 `StoreCode`，供应商 `LocalSupplierCode`；`label` 仍用名称（index.tsx:241-246 已部分完成，调整确保 value=代码）。
- 构造过滤：
  - `if (values.storeCode) filterModel.storeCode = { FilterType: 'text', Type: 'equals', Filter: values.storeCode }`
  - `if (values.supplierCode) filterModel.supplierCode = { FilterType: 'text', Type: 'equals', Filter: values.supplierCode }`

- 本地启动后选择分店下拉，发起 `POST /api/react/v1/store-retail-prices/grid`，应返回 200；列表正确过滤。
