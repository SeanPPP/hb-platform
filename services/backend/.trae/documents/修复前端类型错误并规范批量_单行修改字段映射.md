## 诊断结论
- TypeScript 错误来自 `editedMap.current.set(uuid, current)` 的 `current` 对象不满足 `Partial<StoreRetailPriceUpsertItemDto>` 类型：
  - 我们在前端用的是 camelCase（uuid/storeCode/productCode/supplierCode、discountRate 等）；
  - 但 `StoreRetailPriceUpsertItemDto` 的字段是 PascalCase（UUID/StoreCode/ProductCode/SupplierCode、DiscountRate 等）。
- 报错位置：`index.tsx:99、107、611`（分别是单行编辑保存与批量应用处）。

## 修复方案
- 统一 `editedMap` 存储为 PascalCase，与 `StoreRetailPriceUpsertItemDto` 类型一致：
  - 初始化 `current` 为 `{ UUID, StoreCode, ProductCode, SupplierCode }`；
  - 将字段写入改为映射：
    - `discountRate → DiscountRate`（值为百分比整数）
    - `storeRetailPriceValue → StoreRetailPriceValue`
    - `purchasePrice → PurchasePrice`
    - `isActive → IsActive`
    - `isAutoPricing → IsAutoPricing`
- 保存阶段 payload 处理：
  - 从 `editedMap` 取到 PascalCase 对象直接发送；
  - 在发送前仅将 `DiscountRate` 从百分比转换为小数（/100）。
- 批量应用 `applyBatch()` 同步改造：
  - `current` 使用 PascalCase；
  - 根据选择列映射到 PascalCase 写入；
  - 同步 UI 时仍使用 camelCase 更新表格显示。
