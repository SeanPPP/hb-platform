## 目标
- 在现有 React 同步页面与 React 专用控制器/服务中，新增两项同步：
  - DIC_分店一品多码表 → HBweb.StoreMultiCodeProduct（按分店并发）
  - DIC_一品多码表 → HBweb.ProductSetCode（全量并发读取 + 单事务写入）
- 前端同一个“数据同步”页面新增两块卡片与按钮，显示耗时、开始/结束时间、成功/错误条数与消息；分店一品多码支持选择分店。

## 后端
- 控制器（复用 DataSyncReactController）新增端点：
  - POST /api/react-sync/store-multi-code-products（请求体 { selectedStoreCodes?: string[] }）
  - POST /api/react-sync/product-set-codes
  - 返回 ApiResponse<SyncResult>，保持统一鉴权 [Authorize(Roles="Admin")]。
- 服务接口 IDataSyncReactService 新增：
  - Task<SyncResult> SyncStoreMultiCodeProductsFromHqConcurrentAsync(List<string>? storeCodes, int maxConcurrency = 12, int batchSize = 200000)
  - Task<SyncResult> SyncProductSetCodesFromHqAsync(int hqBatchSize = 50000, int writePageSize = 10000, int maxReadConcurrency = 8)
- 服务实现 DataSyncReactService：
  - StoreMultiCode（分店并发）：
    - 独立 HQ/本地连接；分店内分页 JOIN 拉取，指数退避重试；每页映射 → Fastest<StoreMultiCodeProduct>.BulkCopyAsync(PageSize=50000)
    - 本地清理仅选中分店的数据；汇总 processed/added/errors/query/insert 耗时
  - ProductSetCode（全量）：
    - 清空 ProductSetCode；并发分页从 DIC_一品多码表拉取，映射 → 单消费者事务 BulkCopy（保留原子性与可回滚性）
    - 统计新增/错误与耗时；失败批次退避重试
- React 专用映射 Profile（新增）：
  - ReactStoreMultiCodeProductMappingProfile：
    - StoreCode ← H分店代码；ProductCode ← H商品编码；MultiCodeProductCode ← H多码商品编码 或 H分店多码商品编码；MultiBarcode ← H多条形码；PurchasePrice ← H进货价；MultiCodeRetailPrice ← H一品多码零售价；DiscountRate ← H折扣率；IsAutoPricing ← H是否自动定价；IsSpecialProduct ← H是否特殊商品；IsActive ← H使用状态；CreatedAt/UpdatedAt ← FGC_CreateDate/FGC_LastModifyDate
  - ReactProductSetCodeMappingProfile：
    - ProductCode ← H商品编码；SetProductCode ← H多码商品编号；SetItemNumber ← H多码商品编号（或规则生成）；SetBarcode ← H多条形码（fallback: H主条形码）；SetPurchasePrice ← H进货价；SetRetailPrice ← H一品多码零售价；SetQuantity ← 1；SetType ← 2（多码套装）；Created/Updated ← FGC_CreateDate/FGC_LastModifyDate

  - 新增 syncProductSetCodes() → POST /api/react-sync/product-set-codes（timeout 10–20 分钟）

    - “一品多码（DIC_一品多码表 → ProductSetCode）”：全量同步按钮 + 结果展示
  - 与现有商品/零售价卡片一致的交互与提示；运行中禁用按钮，完成显示统计与耗时
- 菜单与路由保持不变（在“系统设置→数据同步”入口内新增卡片）。

## 并发与稳健性
- StoreMultiCode：按分店并发（默认 12），分店内分页；查询失败指数退避重试并重建连接；批量插入重试与内存回收
- ProductSetCode：并发读取（默认 8）＋单事务消费者写入；失败批次重试；确保原子性
- 前端设置足够超时，失败返回也渲染 SyncResult；必要时可升级后台任务+轮询
