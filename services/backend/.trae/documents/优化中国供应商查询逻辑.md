## 修复 LocalSupplierCode 中国供应商查询逻辑

### 问题
第 1023-1067 行的代码会为**所有 LocalSupplierCode** 查询中国供应商，但实际上**只有 LocalSupplierCode = "200"** 的商品才对应中国供应商。这导致：
1. 查询了大量不必要的 Product 数据
2. 查询了大量不必要的 WarehouseProduct 和 DomesticProduct 数据  
3. 查询了大量不必要的 ChinaSupplier 数据
4. 严重浪费数据库资源和性能

### 解决方案
修改第 1023-1067 行的逻辑：

**方案 1：提前过滤（推荐）**
```csharp
// 只查询 LocalSupplierCode = "200" 的商品
var localSupplier200Codes = allLocalSupplierCodes.Where(c => c == "200").ToList();
if (localSupplier200Codes.Any())
{
    // 只查询 LocalSupplierCode = "200" 的 Product
    var localSupplierProducts = await _context.ProductDb.GetListAsync(p =>
        p.LocalSupplierCode == "200" && !p.IsDeleted
    );
    // ... 后续查询保持不变
}
```

**方案 2：移到循环内部**
将整个 1023-1067 行的代码块移到第 1076 行的 `if (localSupplierCode == "200")` 条件块内。

### 优点
- 减少不必要的数据库查询
- 提升性能（特别是当 LocalSupplierCode 数量很多时）
- 逻辑更清晰，符合业务规则

### 推荐方案
使用**方案 1**，因为：
1. 代码结构更清晰
2. 可以一次性查询所有需要的 ChinaSupplier
3. 避免在循环中重复查询数据库
