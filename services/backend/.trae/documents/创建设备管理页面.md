## 设备管理页面实现计划

### 后端部分

#### 1. 创建 DTO (Data Transfer Objects)
- 创建 `DeviceRegistrationDto.cs` 在 `BlazorApp.Shared/DTOs/` 目录
- 定义：`DeviceRegistrationListDto`, `DeviceRegistrationDetailDto`, `CreateDeviceRegistrationDto`, `UpdateDeviceRegistrationDto`
- 包含字段：ID, 设备硬件识别码, 系统设备编号, 分店代码, 设备类型, 设备系统, 设备状态, 设备授权码, 备注, 审计字段等

#### 2. 创建服务接口和实现
- 接口：`IDeviceRegistrationReactService.cs` 在 `BlazorApp.Api/Interfaces/React/`
- 实现：`DeviceRegistrationReactService.cs` 在 `BlazorApp.Api/Services/React/`
- 方法：GetGridDataAsync, GetByIdAsync, UpdateAsync (按需求不需要创建和删除)
- 编辑字段：设备状态、设备系统、设备类型、备注

#### 3. 创建控制器
- `ReactDeviceRegistrationController.cs` 在 `BlazorApp.Api/Controllers/React/`
- 端点：GET /api/device-registration/grid, GET /api/device-registration/{id}, PUT /api/device-registration/{id}

#### 4. 更新 SqlSugarContext
- 在 `BlazorApp.Api/Data/SqlSugarContext.cs` 添加 POSM 设备表客户端
- 在表初始化中添加 `POSM_设备注册信息表`

#### 5. 更新 Program.cs
- 注册服务：`builder.Services.AddScoped<IDeviceRegistrationReactService, DeviceRegistrationReactService>();`

#### 6. 创建 AutoMapper 配置（如果需要）
- 创建 `ReactDeviceRegistrationProfile.cs` 在 `BlazorApp.Api/Mappings/Profiles/React/`

### 前端部分

#### 1. 创建 TypeScript 类型
- 在 `ReactUmi/my-app/src/types/` 创建 `deviceRegistration.ts`
- 定义设备相关的 TypeScript 接口

#### 2. 创建服务
- 在 `ReactUmi/my-app/src/services/` 创建 `deviceRegistrationService.ts`
- 实现 API 调用方法：getGridData, getById, update

#### 3. 创建页面组件
- 在 `ReactUmi/my-app/src/pages/SystemSettings/DeviceManagement/` 创建 `index.tsx`
- 使用 ProTable 组件展示设备列表
- 实现编辑功能（可编辑设备状态、设备系统、设备类型、备注）
- 状态使用 Tag 组件显示，带颜色区分

#### 4. 创建编辑表单组件
- `UpdateDeviceForm.tsx` 用于编辑设备信息
- 使用 Modal + Form 组件
- 字段：
  - 设备状态：Select 下拉框（待确认/禁用/启用/锁定/未注册）
  - 设备系统：Select 下拉框（Android/iOS/Mac/Windows）
  - 设备类型：Select 下拉框（PDA/Mobile/POS/Admin）
  - 备注：TextArea

#### 5. 添加路由
- 在 `ReactUmi/my-app/.umirc.ts` 的系统设置路由下添加
- 路径：`/system/device-management`
- 权限：`access: 'isAdmin'`

#### 6. 添加国际化翻译
- 在 `ReactUmi/my-app/src/locales/zh-CN.ts` 和 `en-US.ts` 添加设备管理相关翻译

### 权限控制
- 页面访问限制为管理员（`isAdmin`）
- 编辑操作同样限制为管理员
