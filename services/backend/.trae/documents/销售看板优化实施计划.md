# 销售看板优化实施计划

## 一、后端改造 (BlazorApp.Api)

### 1.1 扩展接口和DTO
- 创建 `DateRangeDto` (startDate, endDate, compareStartDate, compareEndDate, compareMode)
- 扩展 `DashboardSummaryDto` 添加同比字段：CompareTotalAmount, CompareTotalQuantity, CompareOrderCount, CompareSkuCount, CompareCustomerCount, CompareAvgOrderValue
- 扩展 `HourlySalesDto` 添加 CompareTotalAmount, CompareTotalQuantity, CompareCustomerCount, CompareAvgOrderValue
- 扩展 `StoreSalesRankDto` / `SupplierSalesRankDto` 添加同比字段和同比百分比
- 新增 `CompareMode` 枚举：ByWeek（按星期）、ByDate（按日期）

### 1.2 修改 Controller
- 所有API添加 `DateRangeDto` 参数替代单 `date`
- 注入 `IUserService` 获取当前用户信息
- 提取用户权限分店代码列表 `List<string> branchCodes`
- 传递 `branchCodes` 和权限标识 `isAdministrator` 到Service层

### 1.3 改造 SalesDashboardReactService

#### GetDashboardSummaryAsync
- 支持日期区间聚合（SUM各字段）
- 支持同比数据查询
- 计算同比百分比

#### GetHourlySalesAsync
- 支持日期区间聚合（按小时SUM）
- 支持同比数据查询
- **改为返回所有分店明细**（移除 BranchCode == "ALL" 过滤）
- 支持供应商过滤（supplierCode参数）
- 支持权限过滤（只返回用户有权访问的分店）

#### GetStoreSalesRankAsync
- 支持日期区间聚合
- 支持同比数据查询
- 权限过滤（管理员看全部，普通用户看自己分店）
- 计算同比百分比

#### GetSupplierSalesRankAsync / GetChinaSupplierSalesRankAsync
- 支持日期区间聚合
- 支持同比数据查询
- 权限过滤（基于用户分店权限）

#### GetSupplierStoreSalesAsync / GetStoreSupplierSalesAsync
- 支持日期区间聚合
- 支持同比数据查询

### 1.4 数据库查询优化
- 日期区间：`WHERE Date >= @start AND Date <= @end`
- 同比数据：查询对比日期范围后合并结果
- 分时数据：`GROUP BY Hour, BranchCode` 而非 `GROUP BY Hour`

## 二、前端改造 (ReactUmi/my-app)

### 2.1 日期选择器组件
- 从 DatePicker 改为 RangePicker
- 默认值：当天（前一天）的当天范围
- 添加快速选择按钮组：
  - 本周（当前周一到今天）
  - 上周（上周一到上周日）
  - 本月（本月1号到今天）
  - 上月（上月完整月份）
- 添加同比模式切换 Toggle：
  - 按星期（2026年第1周 vs 2025年第1周）
  - 按日期（2026-01-01 vs 2025-01-01）
- 显示当前选择和对比日期范围标签

### 2.2 主页面改造
- 从 `useModel` 获取 `currentUser`
- 判断是否管理员（roleNames包含 "Admin" 或检查权限）
- 提取用户分店代码列表（currentUser.stores）
- 日期区间状态：`[startDate, endDate]`
- 对比日期区间状态：`[compareStartDate, compareEndDate]`
- 对比模式状态：`compareMode: 'by-week' | 'by-date'`
- 联动状态：`selectedBranchCode` / `selectedSupplier`
- 权限提示：非管理员显示"您只能查看分店 XXX 的数据"

### 2.3 数据服务层更新
- 所有API函数更新参数：
  - `date` → `dateRange: { startDate, endDate, compareStartDate, compareEndDate, compareMode }`
  - 添加 `branchCodes` 参数（权限过滤）
  - 添加 `supplierCode` 参数（供应商过滤）

### 2.4 汇总卡片改造
- 显示本周期数据：
  - 总销售额、总商品数、订单数、SKU数、客单数、客单价
- 显示对比周期数据（灰色）
- 计算并显示同比百分比：
  - 正增长：绿色 ↑ +10.5%
  - 负增长：红色 ↓ -8.3%
  - 持平：灰色 0%

### 2.5 分时图表改造（多折线图 + 数据点标签）
- 使用 `@ant-design/charts` 的 Line 组件
- X轴：时间（0:00 - 23:00）
- Y轴：销售额（左轴）/ 客单数（右轴）
- **多折线图**：每条线代表一个分店
- **数据点标签**：在每个数据点上显示金额数字
  - 配置 `label` 属性
  - 格式化：$1,234
  - 自动隐藏重叠标签
  - 小数值可省略显示
- 支持同比对比：虚线表示去年同期数据
- 支持供应商过滤：点击供应商后只显示该供应商的各分店数据
- 支持分店过滤：点击分店后只显示该分店数据
- 添加图例开关（显示/隐藏各分店）
- Tooltip显示详细信息：时间、分店、销售额、商品数、客单数、客单价

### 2.6 排行表改造

#### StoreSalesRankTable
- 添加对比数据列：去年同期销售额
- 添加同比百分比列
- 行点击联动：选中后过滤其他组件为该分店数据
- 高亮显示选中行

#### SupplierSalesRankTable / ChinaSupplierSalesRankTable
- 添加对比数据列：去年同期销售额
- 添加同比百分比列
- 行点击联动：选中后过滤其他组件为该供应商的所有分店数据
- 高亮显示选中行

### 2.7 国际化补充
- 快速按钮文本：本周、上周、本月、上月
- 同比模式：按星期、按日期
- 同比标签：本期、去年同期
- 同比指标：同比、增长、下降
- 权限提示文本

## 三、设计优化 (使用 ui-ux-pro-max)

### 3.1 UI改进
- 使用 ProCard 提升视觉层次
- 统一间距：gutter={[16, 16]}
- 统一颜色方案：
  - 增长：#52c41a（绿色）
  - 下降：#ff4d4f（红色）
  - 持平：#8c8c8c（灰色）
  - 主色：#1890ff（蓝色）
- 添加加载骨架屏
- 优化移动端响应式布局

### 3.2 交互优化
- 顶部添加筛选状态栏：
  - 显示当前日期区间
  - 显示对比日期区间
  - 显示联动状态（已选分店/供应商）
  - 添加"清除筛选"按钮
- 联动提示：点击排行表时显示Toast提示
- 权限提示：普通用户顶部横幅提示

### 3.3 可视化细节
- 折线图样式：
  - 实线：本期数据
  - 虚线：去年同期数据
  - 不同颜色区分不同分店
  - 图例可点击切换显示/隐藏
- 数据点标签：
  - 显示在数据点上方
  - 使用紧凑格式：$1.2K、$5.6M
  - 避免重叠：自动隐藏密集区域的标签
  - 鼠标悬停高亮对应数据点
