## 问题摘要
- antd 警告：Input `addonAfter` 已废弃；`message` 静态函数不兼容动态主题
- Handsontable 警告：classic 主题将废弃
- AxiosError：POST `/api/react/v1/product-set-codes/grid` 请求失败（需完善错误上报与排序映射）

## 改动方案
### 1) Handsontable 主题
- 在 HotTable 外层容器添加 `className="ht-theme-default"`，切换到默认主题
- 保留 `licenseKey: 'non-commercial-and-evaluation'`

### 2) antd message 使用方式
- 在页面中使用 `App.useApp()` 获取 `message` 实例：
```tsx
import { App } from 'antd';
const { message } = App.useApp();
```
- 将所有 `message.*` 调用改为此实例，避免静态函数警告
- 在页面 JSX 最高层包裹 `<App>`（仅页面内包裹即可）

### 3) 搜索布局替换
- 将搜索框与按钮组包裹在 `Space.Compact`，避免使用废弃的 `addonAfter`：
```tsx
<Space.Compact>
  <Input.Search ... />
  <Button ... />
  ...
</Space.Compact>
```

### 4) 接口错误诊断与修复
- 增强错误日志：打印 `error.response?.status`、`error.response?.data`，便于定位（401/403/500）
- 排序映射修正：Handsontable `afterColumnSort` 返回列索引，需要映射到字段名：
  - 建立 `index → field` 映射数组，与 `columns` 一致
  - 生成后端需要的 `SortModelDto[]`：`{ colId: fieldName, sort: 'asc' | 'desc' }`
- 失败回退：若排序为空或映射失败，发送空排序，让后端按默认 `updatedAt desc`
