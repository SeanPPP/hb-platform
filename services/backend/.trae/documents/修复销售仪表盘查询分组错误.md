## 修复 `GetSupplierStoreSalesAsync` 中的 SQL 错误

### 问题分析
在 `SalesDashboardReactService.cs` 的 `GetSupplierStoreSalesAsync` 方法中，执行 SQL 查询时抛出异常：
`Microsoft.Data.SqlClient.SqlException (0x80131904): 选择列表中的列 'sales_order.OrderTime' 无效，因为该列没有包含在聚合函数或 GROUP BY 子句中。`

这是因为在 `Select` 语句中使用了 `o.OrderTime.Value.Date`，但该列没有包含在 `GroupBy` 子句中。

### 解决方案
修改 `GroupBy` 子句，将 `o.OrderTime.Value.Date` 加入分组条件。

### 变更计划
1.  **修改 `GetSupplierStoreSalesAsync` 方法**：
    *   在 `GroupBy` 子句中添加 `o.OrderTime.Value.Date`。
    *   为了更准确的供应商筛选，同样引入 `PosmProductSupplierMapping` 表进行连表查询（类似于之前对 `SalesStatisticsJobService` 的修改），以支持 `LocalSupplierCode="200"` 的情况。

2.  **修改 `GetStoreSupplierSalesAsync` 方法**：
    *   同样存在类似的潜在问题（虽然尚未报错），建议一并优化，使用 `PosmProductSupplierMapping` 表进行准确的供应商匹配。

### 代码逻辑预览 (GetSupplierStoreSalesAsync)
```csharp
var query = _posmContext.Db.Queryable<SalesOrder>()
    .LeftJoin<SalesOrderDetail>((o, d) => o.OrderGuid == d.OrderGuid)
    .LeftJoin<PosmProductSupplierMapping>((o, d, m) => d.ProductCode == m.ProductUUID)
    .Where(o => o.OrderTime != null && o.OrderTime.Value.Date == date.Date)
    .Where((o, d, m) => m.LocalSupplierCode == supplierCode || m.ChinaSupplierCode == supplierCode) // 支持双重匹配
    .GroupBy((o, d, m) => new { o.OrderTime.Value.Date, o.BranchCode }) // 添加 Date 到分组
    .Select((o, d, m) => new { ... });
```
