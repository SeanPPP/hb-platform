## 目标
- 仅使用 `HqSqlSugarContext` 作为数据源，将 `CBP_DIC_商品库存表` 全量同步到本地 `WarehouseProduct`。
- 采用分页读取 + 批量写入 + 事务包裹，提供稳定与高性能同步。

## 字段映射
- `WarehouseProduct.ProductCode` ← `H商品编码`（必填；为空记录直接过滤）
- `DomesticPrice` ← `H国内价格`
- `OEMPrice` ← `H贴牌价格`
- `ImportPrice` ← `H进口价格`
- `StockQuantity` ← `H库存`（decimal→int 安全转换：负数/超大值按边界处理）
- `MinOrderQuantity` ← `H最小订货量`（decimal→int 安全转换）
- `StockValue` ← `H库存金额`
- `StockAlertQuantity` ← `H库存预警数`
- `IsActive` ← `H使用状态 == 1`
- `Volume`：暂不映射（后续如需可从商品字典表补齐）

## 后端实现
1. 新增 AutoMapper Profile：`ReactWarehouseProductStockProfile`（`CBP_DIC_商品库存表 → WarehouseProduct`），含数值安全转换与字段截断。
2. 新增服务方法：`SyncWarehouseProductsFromHqAsync(int hqBatchSize = 50000, int writePageSize = 10000)`：
   - `HqSqlSugarContext` 检查连接。
   - 开启事务：清空 `WarehouseProduct` → 分页查询 `CBP_DIC_商品库存表`（过滤 `H商品编码` 非空）→ 映射 → `Fastest<WarehouseProduct>.BulkCopyAsync` 写入。
   - 日志：每批拉取/写入统计与最终汇总（新增/错误/耗时）。
3. 控制器端点：`POST /api/react-sync/warehouse-products`，返回 `ApiResponse<SyncResult>`。

## 验证
- 构建通过；手动调用端点验证同步结果与日志。
- 重点检查：主键 `ProductCode` 不为空；库存/最小订货量类型转换正确；事务成功提交。

## 风险与回滚
- 若数据中存在超长或非法值，批量写入会记录失败批次；可在映射中加强截断与默认值处理。
- 如需回滚，移除新方法与端点即可恢复现状。
