# 修改订货页面默认排序规则

用户希望将订货页面的默认排序规则更改为“按照货号（ItemNumber）排序”。

## 1. 后端修改 (`StoreOrderReactService.cs`)

目前在 `GetPagedListAsync` 方法中，默认排序逻辑是：
```csharp
// 默认按是否有库存排序，然后按更新时间
q = q.OrderBy((p, wp, wc) => wp.StockQuantity > 0 ? 0 : 1)
    .OrderBy((p, wp, wc) => p.UpdatedAt, OrderByType.Desc);
```

**修改计划**：
1.  找到 `StoreOrderReactService.cs` 中的 `GetPagedListAsync` 方法。
2.  修改 `switch (filter.SortBy.ToLower())` 中的 `default` 分支。
3.  将默认排序逻辑改为：优先按 `ItemNumber` 升序排序。
    *   *考虑*: 是否还需要保留“有库存优先”？通常订货页面用户习惯按货号查找，混合库存排序可能会让找货变困难。
    *   *决定*: 根据用户指令“默认按照货号排序”，直接改为按 `ItemNumber` 排序。
4.  同时检查 `if (!string.IsNullOrWhiteSpace(filter.SortBy))` 的 `else` 分支（如果有），也统一修改。

## 2. 前端修改 (`ProductGrid.tsx`)

前端组件中可能也有默认排序的设置。

**修改计划**：
1.  找到 `ProductGrid.tsx`。
2.  检查 `useState('Default')` 的初始值。
3.  检查下拉菜单 `Select` 中的选项，可以将 "Default" 的显示名称改为 "Default (Item No)" 或者直接保留 "Default" 但其含义已变。
4.  *可选*: 如果前端显式传递了 "ItemNumber" 相关的排序参数，确保后端能处理。但目前架构是后端处理 "Default" 逻辑，所以主要改后端即可。

## 3. 验证计划
-   **后端验证**: 编译通过。
-   **前端验证**: 刷新订货页面，在不选择任何排序的情况下，商品列表应按货号升序排列。
