目标
- 修复 isactive/分类/列头过滤变更后仍显示上一次结果的问题，采用 useEffect 统一触发查询，所有查询输入只更新状态，由 effect 集中发起最新请求。

变更范围
- ReactUmi/my-app/src/pages/WarehouseCategories/index.tsx

实施步骤
1) 收敛查询输入为状态
- 保留/新增：page、pageSize、sortBy、sortOrder、tableFilters: Record<string, string[]>、isActiveFilter: 'all'|'1'|'0'、filterCategoryGuids: string[]、includeSubCategories: boolean。

2) 重构 loadTable
- 参数改为只使用当前状态构造请求（不从事件临时传入）：
  - const mergedFilters = { ...tableFilters, ...(isActiveFilter !== 'all' ? { isactive: [isActiveFilter] } : {}) }
  - request: { page, pageSize, sortBy, sortOrder, categoryGuids: filterCategoryGuids, includeSubCategories, filters: Object.keys(mergedFilters).length ? mergedFilters : undefined }
- 执行异步获取后更新：products、total、loading；异常时 message.error。

3) 新增 useEffect 统一触发查询
- 依赖数组：[page, pageSize, sortBy, sortOrder, tableFilters, isActiveFilter, filterCategoryGuids, includeSubCategories]
- effect 内调用 loadTable()，避免在事件里直接调用，杜绝旧状态查询。

4) 仅在事件中更新状态，不直接发请求
- isactive Select onChange：setIsActiveFilter(v); setPage(1)。
- 分类 TreeSelect onChange：setFilterCategoryGuids(guids); setPage(1)。
- Table.onChange：
  - setPage(pagination.current || 1)
  - setPageSize(pagination.pageSize || 默认值)
  - setSortBy(sorter?.field as string | undefined)
  - setSortOrder(sorter?.order as 'ascend'|'descend'|undefined)
  - 将 filters 映射为 Record<string, string[]> 后 setTableFilters(mappedFilters)

5) 保持列过滤 UI 一致
- columns 的 filteredValue 绑定 tableFilters[key]，确保切换分类/isactive 时过滤 UI 不丢失。
- 核对过滤 key 与后端 switch-case 保持一致；如需，增加一个 key 映射函数以统一（例如 productName→productname）。

6) 验证用例
- isactive=1 + 分类A + 名称过滤：仅上架且属于分类A及其子类且名称包含。
- isactive=0 + 条码+供应商名过滤：仅下架且多条件交集。
- isactive=all：不加 isactive 过滤，结果含上下架。
- 分页与排序在上述组合下均正常、每次状态变更后页面数据即时更新。

可选优化
- 如需减少多次状态变更导致的重复请求，可加入小型防抖（如在 effect 内用 setTimeout/clearTimeout），或使用 useRef 跳过首次加载的重复触发；优先保持逻辑简洁再视情况优化。
