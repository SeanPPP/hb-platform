## 修复缺失的菜单翻译键

### 问题
- 路由 `name: 'storeOrder'` 和 `name: 'storeOrderList'` 对应的翻译键缺失
- 当前只有 `menu.warehouse.storeOrder` 和 `menu.warehouse.storeOrderList`
- Umi 根据路由 `name` 自动生成翻译键：`menu.{name}`

### 解决方案
在两个翻译文件中添加顶层键：

**1. 修改 `src/locales/zh-CN.ts`**
- 添加：`'menu.storeOrder': '分店订货'`
- 添加：`'menu.storeOrderList': '分店订货列表'`

**2. 修改 `src/locales/en-US.ts`**
- 添加：`'menu.storeOrder': 'Store Order'`
- 添加：`'menu.storeOrderList': 'Store Order List'`

### 预期结果
- 消除控制台的 React Intl 缺失翻译警告
- 菜单项正确显示中文/英文文本
