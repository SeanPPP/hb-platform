# 修改商品卡片显示和添加购物车逻辑

用户有两个需求：
1.  **Out of Stock 不需要显示**：隐藏商品卡片上的 "Out of Stock" 标签。
2.  **isInStock 也可以添加购物车**：即使商品库存不足（`isInStock` 为 false），也允许用户将其添加到购物车。

## 1. 现状分析

**前端组件**: `src/pages/StoreOrder/components/ProductCard.tsx`

**需求 1: 隐藏 Out of Stock 标签**
当前代码：
```tsx
{!product.isInStock && <Tag color="red">Out of Stock</Tag>}
```
解决方案：直接移除或注释掉这段代码。

**需求 2: 允许 Out of Stock 商品添加到购物车**
当前代码：
```tsx
<Button 
   type="primary" 
   icon={<ShoppingCartOutlined />} 
   onClick={handleAddToCart}
   loading={loading}
   disabled={!product.isInStock} // 这一行阻止了添加
>
   Add
</Button>
```
解决方案：移除 `disabled={!product.isInStock}` 属性。

## 2. 潜在影响

*   **前端**: 用户将能够对任何商品点击 "Add" 按钮。
*   **后端**: 之前在 `StoreOrderReactService.cs` 的 `AddToCartAsync` 方法中，并没有显式检查库存量来阻止添加（只是检查商品是否存在）。所以后端应该默认支持此行为。
    *   在 `AddToCartAsync` 中：
        ```csharp
        var warehouseProduct = await _db.Queryable<WarehouseProduct>()...;
        // ...
        // 并没有检查 warehouseProduct.StockQuantity > 0
        ```
    *   因此，只需要修改前端即可。

## 3. 执行步骤

1.  修改 `src/pages/StoreOrder/components/ProductCard.tsx`。
    *   移除 `Out of Stock` 的 Tag。
    *   移除 Add 按钮的 `disabled` 属性。

## 4. 验证计划

*   **UI**: 确认 Out of Stock 标签不再显示。
*   **功能**: 找到一个库存为 0 的商品，尝试点击 Add 按钮，确认能成功添加到购物车。
