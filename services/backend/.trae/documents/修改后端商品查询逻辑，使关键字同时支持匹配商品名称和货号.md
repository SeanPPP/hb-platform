按您的要求，我将把前端的查询参数改为 `itemNumber`，并相应调整后端以确保搜索功能完整。

### 1. 修改前端
文件：`src/pages/StoreHome/index.tsx`
*   在 `fetchProducts` 方法中，将 `keyword` 参数传递给 `itemNumber` 字段，而不是 `productName`。
    ```typescript
    const res = await getStoreOrderProducts({
      // ...
      itemNumber: keyword || undefined, // 修改此处
      productName: undefined,           //以此为准
      // ...
    });
    ```

### 2. 修改后端
文件：`BlazorApp.Api/Services/React/StoreOrderReactService.cs`
*   为了防止改为 `itemNumber` 后无法搜索商品名称，我将同时增强后端的 `ItemNumber` 过滤逻辑。
*   在 `GetPagedListAsync` 中，修改 `filter.ItemNumber` 的处理块：
    *   将查询条件改为：匹配 `ItemNumber` **OR** `ProductName`。
    *   这样即使用户输入的是商品名称，通过 `itemNumber` 字段传进来也能搜到。

```csharp
// 2. 货号搜索 (增强为 货号 OR 名称)
if (!string.IsNullOrWhiteSpace(filter.ItemNumber))
{
    var keyword = filter.ItemNumber.Trim().ToLower();
    q = q.Where(
        (p, wp, wc) => 
            (p.ItemNumber != null && p.ItemNumber.ToLower().Contains(keyword)) || 
            (p.ProductName != null && p.ProductName.ToLower().Contains(keyword))
    );
}
```
