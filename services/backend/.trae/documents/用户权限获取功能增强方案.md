## 实施方案

### 后端修改（.NET API）

#### 1. 修改 `AuthController.cs` - `GetCurrentUser()` 方法
- 查询用户所有角色的权限列表
- 将权限信息添加到返回的 `UserDto` 中
- 确保返回完整的权限数据

#### 2. 修改 `UserDto.cs` - 添加权限字段
- 在 `UserDto` 中添加 `Permissions` 属性（`List<string>`）
- 确保类型定义与前端一致

#### 3. 修改 `AuthService.cs` - JWT Token 生成
- 在 `GenerateJwtToken()` 和 `GenerateAccessToken()` 方法中添加权限声明
- 将用户所有权限添加到 JWT Claims 中

### 前端修改（React Umi）

#### 1. 更新 `app.ts` - 处理权限数据
- 从 API 响应中提取 `permissions` 字段
- 将权限添加到 `currentUser.permissions` 数组

#### 2. 完善 `access.ts` - 权限检查逻辑
- 确保权限检查函数正确工作
- 根据实际权限系统调整权限验证逻辑

#### 3. 实现菜单权限过滤
- 在 `menuDataRender` 或 `menuItemRender` 中添加权限检查
- 根据用户权限动态显示/隐藏菜单项
- 为每个菜单项配置所需权限

#### 4. 更新类型定义
- 同步后端和前端的类型定义
- 确保类型一致性

### 预期效果

✅ 后端返回完整的用户权限信息
✅ JWT Token 包含权限声明
✅ 前端正确获取和存储权限

✅ 权限检查功能正常工作
