## 总览

* 目标：在“仓库商品管理”页增加“商品明细编辑弹窗”，除商品编码/货号不可改，其它字段均可编辑；并将所有后端实现放置在 React 项目目录内，统一为 `/api/react/v1/...`。

* 目录位置：`ReactUmi/my-app/server`（Node.js + Express + TypeScript）；前端服务层位于 `src/services/*`。

* 数据源：sqlserver   SqlSugarContext。

## 数据模型

* Product（基础商品）

  * 关键：`productCode`(PK, string)、`itemNumber`(唯一, string)

  * 字段：`name`、`nameEn`、`specs`、`material`、`remark`、`packingQty`、`volume`、`grossWeight`、`packingSize`

  * 价格：`RetailPrice`、`importPrice（PurchasePrice）`

  * 供应商：`LocalSupplierCode`(string, 不能为空)、`ChinaSupplierCode`(string, 可空)

* WarehouseProduct（仓库维度）

  * 关键：`productCode`(PK, FK Product)

  * 字段：`isActive`、`minOrderQuantity`、`volume`（仓库维度体积）

  * 价格：`domesticPrice`、`oemPrice`、`importPrice`

  * 分类：`categoryGuid`(可空)

* LocalSupplier（本地供应商）

  * 关键：`guid`(PK)、`LocalSupplierCode`(唯一, string)、`name`、`status`、`contactPerson`、`phone`、`email`、`remark`

* ChinaSupplier（国内供应商）

  * 关键：`guid`(PK)、`supplierCode`(唯一, string)、`supplierName`、`status`、...

## 后端路由（React 目录内）

* Product 基础表

  * `GET /api/react/v1/domestic-products/:productCode`

  * `PUT /api/react/v1/domestic-products/:productCode`

  * `PUT /api/react/v1/domestic-products/batch-update`

  * `PUT /api/react/v1/domestic-products/:productCode/set-items`（如涉及套装）

* WarehouseProduct 仓库维度

  * `POST /api/react/v1/product-warehouse/table`（分页、筛选、排序）

  * `POST /api/react/v1/product-warehouse/batch-update`（更新价格、体积、上架状态）

  * `POST /api/v1/WarehouseProductBatch/bulk-set-status`（仅切换上架/下架）

  * `POST /api/v1/WarehouseProductBatch/batch-update`（更新 `minOrderQuantity`）

* 分类

  * `GET /api/react/v1/warehouse-categories/tree`

  * `POST /api/react/v1/warehouse-categories/:categoryGuid/products/batch-assign`

  * `POST /api/react/v1/warehouse-categories/products/batch-unassign`

* LocalSupplier（新建）

  * `GET /api/react/v1/local-suppliers`

  * `GET /api/react/v1/local-suppliers/active`

  * `POST /api/react/v1/local-suppliers`

  * `PUT /api/react/v1/local-suppliers/:LocalSupplierCode`

* ChinaSupplier（已存在，按现有服务）

  * `GET /api/v1/ChinaSuppliers`、`GET /api/v1/ChinaSuppliers/active` 等\
    复制一份 到 react 目录下 前端重新对接到新控制器

## 弹窗实现（仓库商品页）

* 入口：在 `WarehouseCategories/index.tsx` 增加“操作”列的“编辑”按钮或行双击打开弹窗（现有文件位置：`ReactUmi/my-app/src/pages/WarehouseCategories/index.tsx`）

* 加载数据：

  * 基础商品详情：`getProductById(productCode)`（按编码）

  * 国内供应商：`getActiveChinaSuppliers()`

  * 本地供应商：`getLocalSuppliersActive()`（新服务）

  * 分类树：复用 `treeData`

* 显示逻辑（供应商）：

  * 默认显示“本地供应商”选择；当 `LocalSupplierCode === '200'` 时，隐藏本地供应商选择，显示“国内供应商”选择（label 显示 `supplierName`，value 用 `supplierCode`）

* 表单字段：

  * 只读：`productCode`、`itemNumber`

  * 基础：`name/nameEn/specs/material/remark/packingQty/volume/grossWeight/packingSize/domesticPrice/labelPrice/importPrice`

  * 供应商：`LocalSupplierCode` 或 `SupplierCode` 依据上方逻辑显示

  * 仓库：`isActive/minOrderQuantity/categoryGuid`（`TreeSelect`）

* 提交顺序：

  1. `PUT /api/react/v1/domestic-products/:productCode` 更新基础字段与供应商字段（只读字段忽略）
  2. 仓库维度：若价格/体积/状态有变更，`POST /api/react/v1/product-warehouse/batch-update` 或 `bulk-set-status`
  3. `minOrderQuantity`：`POST /api/v1/WarehouseProductBatch/batch-update`
  4. 分类变更：`batch-assign` 或 `batch-unassign`
  5. 成功后关闭弹窗，`loadTable()` 刷新

## 供应商显示与存储规则

* 表单显示：

  * `LocalSupplierCode !== '200'` → 显示“本地供应商”下拉

  * `LocalSupplierCode === '200'` → 显示“国内供应商”下拉

* 存储与更新：

  * 本地选择提交 `LocalSupplierCode`；国内选择提交 `SupplierCode`

  * 切换到国内时，同时将 `LocalSupplierCode` 设为 `'200'`，保证显示逻辑一致

## 迁移与风险

* 风险：`getProductById` 参数名与类型（将使用 `productCode` 作为路径参数）；避免与数值 ID 混淆

* 一致性：仓库维度价格与基础价格的来源需定义“权威字段”——建议以仓库维度为列表展示价格，基础价格用于商品主数据

* 供应商：如列表当前来源与基础/仓库字段不同步，需统一映射策略（建议以基础 Product 为准）
