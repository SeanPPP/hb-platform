# 添加分店选择与校验 (更新方案)

用户要求在查询按钮后面添加分店选择功能，并且在购物车头部也显示对应的分店名称。

## 1. 需求分析

1.  **分店选择**:
    *   **位置**: 查询按钮后面（`ProductGrid.tsx` 的 Toolbar）。
    *   **数据源**: 当前用户可用的分店列表（代码）。
    *   **默认值**:
        *   如果只有一个分店，默认选中。
        *   如果有多个分店，默认为空。
    *   **交互**: 如果未选择分店，不能添加到购物车，并提示“请选择分店”。

2.  **购物车头部显示分店**:
    *   `CartDrawer.tsx` 头部（Title 区域）显示当前选中的分店名称。

3.  **API**:
    *   需要前端 API 来获取用户分店列表。
    *   目前 `UserService.cs` 有 `GetUserStoresAsync`，对应路由 `/api/users/guid/{guid}/stores`。
    *   我们可以使用 `currentUser` (from `useModel('@@initialState')`) 获取 GUID，然后调用此 API。
    *   或者使用 `BlazorApp.Api/Controllers/UsersController.cs` 的 `/api/users/guid/{guid}/stores`。

## 2. 状态管理调整

目前 `StoreOrderPage` (`index.tsx`) 硬编码了 `storeCode = 'S001'`。我们需要将其改为动态状态。

**方案**:
*   **状态提升**: `selectedStoreCode` 和 `storeList` 状态放在 `StoreOrderPage` (index.tsx) 中管理。
*   **传递**: 将 `selectedStoreCode` 和 `storeList` 传递给 `ProductGrid` 和 `CartDrawer`。
*   **回调**: `ProductGrid` 需要一个回调 `onStoreChange` (虽然是在 `ProductGrid` 内部渲染 Select，但状态在父组件更好，或者反过来，`ProductGrid` 管理状态并通知父组件去刷新购物车)。
    *   考虑到 `ProductGrid` 的 Toolbar 位置，Select 组件将在 `ProductGrid` 中渲染。
    *   为了简单，可以在 `ProductGrid` 中处理 Select，并通过回调通知 `StoreOrderPage` 更新 `storeCode`，进而触发 `fetchCart`。

**修正**: `ProductGrid` 的 Toolbar 是在 `ProductGrid.tsx` 内部。如果要把 Select 放在“查询按钮后面”，那就必须在 `ProductGrid.tsx` 里渲染。
因此：
1.  `ProductGrid.tsx` 负责获取分店列表（或者由父组件获取后传入）。建议由父组件获取，因为 `CartDrawer` 也需要知道分店信息（名称）。
2.  **决定**: 在 `StoreOrderPage` 获取分店列表，并传递给 `ProductGrid`。`ProductGrid` 渲染 Select。
3.  `StoreOrderPage` 管理 `currentStore` (包含 Code 和 Name)。

## 3. 实现步骤

### 3.1 API 准备
在 `src/services/userService.ts` (如果不存在则创建) 或 `storeService.ts` 中添加 `getUserStores`。
由于 `ReactUmi` 默认可能没有 `userService.ts`，我将在 `storeService.ts` 中添加：
```typescript
export async function getUserStores(userGuid: string) {
  return request(`/api/users/guid/${userGuid}/stores`, { method: 'GET' });
}
```
注意：这需要 User GUID。我们可以从 `initialState` 获取。

### 3.2 修改 `StoreOrderPage` (`index.tsx`)
1.  获取 `initialState` 中的 `currentUser`。
2.  `useEffect` 中调用 `getUserStores(currentUser.userid)`。
3.  逻辑：
    *   如果列表长度 1 -> `setStore(list[0])`。
    *   如果列表长度 > 1 -> `setStore(null)`。
4.  将 `store` 对象传递给 `ProductGrid` 和 `CartDrawer`。

### 3.3 修改 `ProductGrid.tsx`
1.  接收 `stores` 列表和 `currentStore` 以及 `onStoreChange` 回调。
2.  在 Toolbar 中渲染 `Select`。
3.  在 `handleAddToCart` 中检查 `currentStore` 是否为空。如果是，`message.error('Please select a store')` 并返回。

### 3.4 修改 `CartDrawer.tsx`
1.  接收 `storeName`。
2.  在 `title` 中显示：`Shopping Cart (${storeName}) ...`。
