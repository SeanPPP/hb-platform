## 目标
- 列表页保持普通 AntD Table。
- 明细页保留虚拟化滚动以提升超大数据量的渲染与滚动性能。

## 技术方案
- 新增独立组件 `DetailVirtualTable`，仅在明细页使用，不影响其他页面。
- 基于 AntD Table 的 `components.body` + `rc-virtual-list` 实现纵向虚拟化：
  - 通过 `renderVirtualList(rows)` 仅渲染可视区行，提供 `rowHeight`、`overscan`。
  - 要求行高固定（如 48），以保证虚拟化准确。
- 兼容策略：
  - 明细页在虚拟化模式下暂不使用固定列；如需要固定列，自动降级为普通 Table（保持现有体验）。
  - 保留筛选、排序、行选择、操作列、图片单元格等交互。

## 实施步骤
1. 新增 `src/components/DetailVirtualTable.tsx`：封装 AntD Table + 虚拟化渲染的表体，仅导出给明细页使用。
2. 明细页 `LocalSupplierInvoiceDetail/index.tsx`：
  - 当数据量超过阈值（如 `items.length > 500`），或显式设置开关时，切换到 `DetailVirtualTable`。
  - 传递 `height`、`rowKey`、`rowHeight`、`overscan` 等参数；否则使用普通 `VirtualTable`（非虚拟）。
3. 验证功能：
  - 大数据滚动帧率与内存占用；筛选、排序、选择、粘贴导入、批量操作与价格分发。
  - 固定列需求时降级验证，保证不影响使用。

## 回退方案
- 如虚拟化影响特定交互或布局，提供页内开关关闭虚拟化，立即回退到普通 Table 渲染。
