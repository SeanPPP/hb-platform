## 问题概览
- 前端 TypeScript 有 2 处构建错误，均在 `SetItemsModal.tsx`（react-data-grid 的编辑渲染器与行变更回调类型不匹配）。
- 后端 C# 有多处可空性与异步相关警告，主要集中在：字符串方法对可能为 null 的对象调用、将可能为 null 的值赋给非可空字段、`async` 方法未使用 `await`。

## 原因分析
- renderEditCell 类型不匹配：react-data-grid 的 `Column.renderEditCell` 期望的是“函数渲染器”，而当前传入了 `React.FC`，TS 判定不可赋值。
- onRowsChange 类型不匹配：回调签名错误地将 `rowIndex` 并入 `rows`，与库定义不符（索引信息应从 `data.indexes` 读取）。
- C# 可空性：对可能为 null 的字符串（如 `ProductCode`、`SupplierCode`、`备注` 等）直接调用 `.Contains/.StartsWith/.EndsWith` 或将其赋值给非可空属性，引发警告与潜在空引用。
- `async` 无 `await`：方法标记为 `async` 但内部无异步操作，导致编译器提示；应改为同步返回 `Task.FromResult(...)` 或增加实际 `await`。

## 修复方案（逐点）
### 前端：`SetItemsModal.tsx`
1) 调整编辑渲染器类型
- 将 `renderEditCell: PriceEditCell` 改为函数包装：`renderEditCell: (props) => <PriceEditCell {...props} />`
- 位置：`ReactUmi/my-app/src/pages/DomesticProducts/SetItemsModal.tsx:614`
- 可选：将 `PriceEditCell` 改为函数签名：`(props: RenderEditCellProps<SetItem>) => ReactNode`

2) 修正行变更回调签名
- 将 `handleRowsChange` 改为：`(updatedRows: SetItem[], data: RowsChangeData<SetItem>) => void`
- 直接 `setSetItems(updatedRows)`；如需索引，用 `data.indexes`
- 移除对 `rowIndex` 的“清洗”逻辑
- 位置：`ReactUmi/my-app/src/pages/DomesticProducts/SetItemsModal.tsx:357-368`

### 后端：可空与异步
