**问题原因**

* 使用 antd v5，`Modal` 需使用 `open` 属性控制显隐。现页“新增分类”弹窗未传递 `open={createVisible}`，导致点击“新增”仅修改状态但不会显示弹窗。

* 影响文件：

  * `src/pages/WarehouseCategories/index.tsx` → “新增分类” `Modal` 仅有 `title/onCancel/onOk`，缺失 `open`。

  * `src/pages/WarehouseProducts/index.tsx` → 同样的“新增分类” `Modal` 缺失 `open`。

**修复方案**

* 为“新增分类”弹窗添加 `open={createVisible}`，并建议增加 `destroyOnClose` 防止表单残留：

  * `Modal open={createVisible} destroyOnClose ...`

* 保留现有按钮逻辑：`onClick={() => setCreateVisible(true)}`，`onCancel={() => setCreateVisible(false)}`，`onOk` 执行创建后关闭并 `form.resetFields()`。

* 校验“编辑分类”弹窗：`open={editVisible}` 已存在，无需修改。

**修改点**

* `ReactUmi/my-app/src/pages/WarehouseCategories/index.tsx`：为“新增分类” `Modal` 添加 `open={createVisible}` 与 `destroyOnClose`。

* `ReactUmi/my-app/src/pages/WarehouseProducts/index.tsx`：为“新增分类” `Modal` 添加 `open={createVisible}` 与 `destroyOnClose`。

**验证步骤**

* 启动前端后，进入仓库商品页面，切换“管理模式”。

* 点击“新增”按钮：弹出“新增分类”表单；填写并提交成功后弹窗关闭，分类树刷新显示新分类。

* 取消或关闭弹窗：弹窗消失，再次打开不携带上次填写的内容（因 `destroyOnClose`）。
