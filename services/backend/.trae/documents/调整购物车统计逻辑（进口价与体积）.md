# 订货页面统计调整开发计划 (V2)

根据用户最新指令，购物车统计需要做以下调整：

1. **移除** 前端的“零售价”（OEMPrice）总计显示。
2. **新增** “进口价格”（ImportPrice）总计显示。
3. **新增** “体积”（Volume）总计显示。

   * **计算规则变更**：单件体积计算依赖于 `DomesticProduct.UnitVolume` 和 `DomesticProduct.PackingQuantity` (单件装箱数)。

   * **计算公式**：

     * 如果 `PackingQuantity > 0`，单件体积 = `UnitVolume / PackingQuantity` ( `UnitVolume` 是整箱体积)&#x20;

     * **关键确认**：用户引用了 `DomesticProduct.cs` 的 `PackingQuantity` 和 `UnitVolume`。

     * 查看 `DomesticProduct.cs` 注释：

       * `UnitVolume`: 是整箱体积

       * `PackingQuantity`: "单件装箱数量...说明：单个包装箱可容纳的商品数量"。

     * 然而，用户说：“`/DomesticProduct.cs#L161-168` 体积 需要按照 单件装箱数 取得单个商品 体积 在 根据数量计算合计体积”。

     * 这通常意味着 `UnitVolume` 字段存储的可能是**整箱体积**（或者外箱体积），或者用户希望用 `外箱体积 / 装箱数` 来计算更精确的单件分摊体积。

     * 结合上下文 `WarehouseProduct.Volume` 也有定义。

     * **策略**：优先使用 `DomesticProduct` 中的数据进行计算。如果 `UnitVolume` 是整箱体积（通常物流数据是按箱给的），那么 `SingleVolume = UnitVolume / PackingQuantity`。

     * **假设**：根据用户描述 "按照 单件装箱数 取得单个商品 体积"，最合理的解释是：数据库里的 `UnitVolume` 代表的是**箱规体积**，所以需要除以 `PackingQuantity` 得到单件体积，再乘以购买数量。

     * **修正**：如果 `UnitVolume` 注释说是 "单件体积"，那为何还要除以装箱数？可能是注释与实际业务数据不符，或者用户指的是 `WarehouseProduct` 里的 Volume 实际上是箱体积？

     * 查看 `WarehouseProduct.cs`，L79 `Volume` 注释也是 "单件体积"。

     * 用户明确指向 `DomesticProduct.cs`。

     * **最终算法**：`TotalVolume = (DomesticProduct.UnitVolume / DomesticProduct.PackingQuantity) * CartItem.Quantity`。 (如果 `PackingQuantity` 为空或0，则直接用 `UnitVolume` 或 0)。

## 1. 后端开发 (BlazorApp.Api & Shared)

### 1.1 修改 DTO

在 `BlazorApp.Shared/DTOs/StoreOrderDtos.cs` 中：

* **`StoreOrderCartItemDto`**:

  * 新增 `ImportPrice` (decimal)

  * 新增 `ImportAmount` (decimal)

  * 新增 `Volume` (decimal?, 单件体积)

  * 新增 `TotalVolume` (decimal?, 小计体积)

* **`StoreOrderCartDto`**:

  * 新增 `TotalImportAmount` (decimal)

  * 新增 `TotalVolume` (decimal)

### 1.2 修改服务层逻辑 (`StoreOrderReactService.cs`)

* **方法**: `GetActiveCartAsync`

* **查询逻辑**:

  * 需要联表 `WareHouseOrderDetails` (d) -> `Product` (p) -> `WarehouseProduct` (wp) -> `DomesticProduct` (dp)。

  * 注意：`WarehouseProduct` 通过 `ProductCode` 关联 `DomesticProduct` (一对一)。

* **计算逻辑**:

  * `ImportPrice` = `d.ImportPrice` (优先取快照) 或 `wp.ImportPrice`。

  * `ImportAmount` = `ImportPrice * Quantity`。

  * `ItemVolume` (单件体积) = `dp.UnitVolume` (如果 `PackingQuantity` > 0 则 `dp.UnitVolume / dp.PackingQuantity`)。

    * *注意*: 需要处理 null 值和除零异常。

  * `TotalVolume` (总) = Sum(ItemVolume \* Quantity)。
