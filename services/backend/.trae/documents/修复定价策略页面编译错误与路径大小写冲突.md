## 问题与原因
- 路径大小写冲突：项目在两种根路径被同时引用（`D:/Development/cline/blazor/ReactUmi/my-app` 与 `d:/Development/cline/blazor/reactumi/my-app`），TypeScript 开启 `forceConsistentCasingInFileNames` 时会报“文件名仅在大小写方面不同”的错误。
- 成功判断类型错误：页面中将 `res.data.success` 用于 `ApiResponse<T>` 的 `data` 部分，`data` 实际是 `PricingStrategyDetailDto` 或 `boolean`，导致错误（行号参考：`src/pages/PosAdmin/PricingStrategies/index.tsx` 第 142、145、162）。
- 组件存在但启动失败：Umi 路由已指向 `@/pages/PosAdmin/PricingStrategies`，页面文件已存在；当前阻塞来自大小写冲突导致的编译失败。

## 解决方案
- 统一根路径大小写
  - 关闭当前编辑器窗口与开发服务器；
  - 仅使用 `D:/Development/cline/blazor/ReactUmi/my-app`（首字母大写）作为工作路径；
  - 清理 Umi 缓存：删除 `node_modules/.cache`；
  - 在大写路径下重新启动：`npm run dev`。
- 修复成功判断逻辑（页面文件）
  - 将以下逻辑统一为读取顶层 `ApiResponse.success`：
    - 创建/更新：`const ok = !!((res as any)?.success)`
    - 删除：`const success = !!((res as any)?.success)`
  - 失败提示文本读取 `message`：`(res as any)?.message || '操作失败'`
- 保持路由与别名稳定
  - `.umirc.ts` 使用 `component: '@/pages/PosAdmin/PricingStrategies'` 已正确；无需再改。
