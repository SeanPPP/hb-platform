# 修改订单明细添加与更新逻辑

根据需求，将订单明细的操作重心从“订货数量”转移到“配货数量(Send Qty)”。

## 后端修改 (BlazorApp.Api)
### StoreOrderReactService.cs
修改 `AddOrUpdateDetailAsync` 方法：
1.  **添加模式 (`isUpdate = false`)**:
    - 忽略传入的 `quantity` 参数。
    - 使用 `MinOrderQuantity` (默认为1) 作为增量。
    - 如果明细已存在：`AllocQuantity += MinOrderQuantity`。
    - 如果明细不存在：`AllocQuantity = MinOrderQuantity`, `Quantity = 0` (保持订货数量为0或根据业务需要设为0)。
2.  **更新模式 (`isUpdate = true`)**:
    - 将传入的 `quantity` 参数直接赋值给 `AllocQuantity`。
3.  **金额计算**:
    - 更新 `OEMAmount` 和 `ImportAmount` 时，基于 `AllocQuantity` 进行计算。

## 前端修改 (ReactUmi)
### OrderDetails/index.tsx
1.  **列定义调整**:
    - `Order Qty` (`quantity`): 移除编辑功能，改为只读展示。
    - `Send Qty` (`allocQuantity`): 添加 `InputNumber` 组件，开启编辑功能，绑定 `handleUpdateQuantity` 事件。
2.  **交互逻辑**:
    - `handleUpdateQuantity` 保持不变，它将把新的 Send Qty 值通过 API 发送给后端（后端已适配接收该值并更新 AllocQuantity）。
