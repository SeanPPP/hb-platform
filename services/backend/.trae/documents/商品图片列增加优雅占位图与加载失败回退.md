## 目标
在商品图片缺失或加载失败时，展示一张好看的占位图，渲染稳定、样式统一、无需新增服务端逻辑。

## 方案
1. 选择占位图
- 优先使用已存在的通用 Logo：`node_modules/@umijs/preset-umi/assets/umi.png`
- 如需更像商品图片的视觉风格，可暂用根目录的测试截图 `test-screenshot.png`（但路径不在静态目录，推荐先用 `umi.png` 保稳）

2. 前端渲染改造（仅修改 `index.tsx` 中图片列 renderer）
- 当 `productImage` 为空：直接使用占位图 URL
- 当图片加载失败（`onerror`）：替换为占位图 URL，避免裂图图标
- 保持现有样式尺寸（`maxWidth: 70px; maxHeight: 48px; objectFit: contain`），并加 `loading="lazy"`、`decoding="async"`

3. 实现要点
- 在文件顶部定义常量 `FALLBACK_IMAGE_URL` 指向占位资源
- 在 renderer 中：
  - `const url = rowData?.productImage || FALLBACK_IMAGE_URL`
  - 为 `img` 绑定 `onerror = () => { img.src = FALLBACK_IMAGE_URL }`
  - 为无图时也正常展示占位

4. 验证
- 手动造一条无图记录（或清空部分 `productImage`），图片列显示占位
- 模拟错误 URL（如拼接错误），首次加载失败后自动切换为占位
- 不影响冻结列与分页性能

## 备注
- 该实现不引入新文件；若你希望使用品牌化占位图，我们可后续在 `src/assets` 添加自定义图片并将常量指向它。
