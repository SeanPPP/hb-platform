## 问题回顾
- DetailDataGrid：
  - 错误1：react-data-grid 无默认导出，需改用命名导出 `DataGrid`。
  - 错误2：`ColumnsType<T>` 可能为分组列，`dataIndex` 不一定存在；需兼容 `key`/`dataIndex`。
  - 错误3：`Column<T>` 不存在 `formatter` 属性，需改用 `renderCell`。
  - 错误4：`onSelectedRowsChange` 的参数 `set` 需显式类型，避免隐式 any。
- DetailVirtualTable / VirtualBody：
  - 错误1：rc-virtual-list 的 `itemKey` 函数签名只接收一个参数，当前传了 `(row, idx)`。
  - 错误2：`overscan` 不是 rc-virtual-list 的合法属性，应移除。

## 修复方案
- DetailDataGrid.tsx：
  1) 改为 `import { DataGrid } from 'react-data-grid'`，并保留 `Column`、`RenderCellProps` 的命名导入。
  2) 构建 RDG 列时，兼容取键：`const key = (col as any).dataIndex ?? (col as any).key ?? ''`；若为空则跳过该列。
  3) 使用 `renderCell` 代替 `formatter`，保持与 AntD 的 `render` 一致：`renderCell: ({ row }) => col.render ? col.render(row[key], row) : String(row[key] ?? '')`。
  4) 为 `onSelectedRowsChange` 的参数标注类型：`(set: Set<React.Key>) => ...`。
- DetailVirtualTable.tsx & VirtualBody.tsx：
  1) `itemKey` 改为单参：`(row: any) => row?.key ?? undefined`（依赖 React 子节点的 key）。
  2) 移除所有 `overscan` 相关传参与类型，保持 rc-virtual-list 的标准 props。
  3) 若继续保留该文件，仅用于非生产调试；实际明细页以 DataGrid 为主（行数>250时）。

## 额外稳定性
- 明细页列宽：确保所有列均设置 `width`（现有列已普遍设置），RDG 使用固定行高 48，横向滚动由容器控制。
- 列表页选择器事件：已替换为 `onOpenChange`，无需改动。
