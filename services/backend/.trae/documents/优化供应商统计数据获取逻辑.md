## 修改供应商统计数据获取方法

### 目标
修改 `SalesStatisticsJobService.cs` 中的 `UpdateStoreSupplierStatistics` 方法，不再从主数据库查询 `Product`、`WarehouseProduct` 等表来获取供应商信息，而是直接使用 POSM 数据库中的 `PosmProductSupplierMapping` 表。

### 变更计划
1.  **修改 `UpdateStoreSupplierStatistics` 方法**：
    *   修改 POSM 数据库查询，联接 `SalesOrder`、`SalesOrderDetail` 和 `PosmProductSupplierMapping` 表。
    *   联接条件：`SalesOrderDetail.ProductCode == PosmProductSupplierMapping.ProductUUID`。
    *   直接在查询结果中获取 `LocalSupplierCode` 和 `ChinaSupplierCode`。
2.  **移除主数据库查询逻辑**：
    *   移除查询 `_context.ProductDb` 的代码。
    *   移除查询 `_context.WarehouseProduct` 的代码。
    *   移除查询 `_context.ChinaSupplierDb` (用于逻辑判断的部分)。
3.  **实现供应商解析逻辑**：
    *   如果 `LocalSupplierCode == "200"` 且 `ChinaSupplierCode` 不为空，则供应商为 `ChinaSupplierCode`（国内供应商）。
    *   否则供应商为 `LocalSupplierCode`。
4.  **保留供应商名称获取**：
    *   仍然需要从主数据库获取供应商名称（`HBLocalSupplier` 和 `ChinaSupplier`），但仅根据最终解析出的 `SupplierCode` 列表进行批量查询，这是一个轻量级操作。

### 代码逻辑预览
```csharp
var query = _posmContext.Db.Queryable<SalesOrder>()
    .LeftJoin<SalesOrderDetail>((o, d) => o.OrderGuid == d.OrderGuid)
    .LeftJoin<PosmProductSupplierMapping>((o, d, m) => d.ProductCode == m.ProductUUID)
    .Where(o => o.OrderTime.Value.Date == targetDate)
    .Select((o, d, m) => new {
        Date = o.OrderTime.Value.Date,
        BranchCode = o.BranchCode,
        LocalSupplierCode = m.LocalSupplierCode,
        ChinaSupplierCode = m.ChinaSupplierCode,
        TotalAmount = d.ActualAmount,
        TotalQuantity = d.Quantity
    });
// ...后续内存分组和处理
```
