## 原因与目标
- 远程图片被防盗链/CORP限制，浏览器报 `blocked by CORS` 与 `net::ERR_FAILED 200 (OK)`。仅在后端放开 CORS 并不能改变第三方站点的限制。
- 需要两部分：后端全局允许跨域（满足“允许所有跨域”的要求），同时通过同源图片代理绕过远程站点的防盗链与 CORS。

## 后端改造（BlazorApp.Api）
1. 全局 CORS 开启
- 在启动文件添加：`services.AddCors` 注册默认策略 `AllowAnyOrigin().AllowAnyMethod().AllowAnyHeader()`。
- 在中间件管道启用：`app.UseCors()` 放到 `UseAuthentication` 之前。
- 验证：前端站点与任何来源调用 API 都不再受跨域限制。

2. 图片代理接口
- 新增 `GET /api/react/v1/image-proxy?url=...`，同源返回图片，避免浏览器跨域拦截。
- 用 `HttpClient` 拉取远程图片，附带 `Referer: https://www.malmar.com.au/`、`User-Agent` 与 `Accept: image/*`。
- 白名单控制：允许 `https://www.malmar.com.au/img/products/thumb/` 前缀，拒绝其他域。
- 返回 `Content-Type: image/*` 的流；非图片或失败返回 404。设置合理超时与尺寸上限。

## 前端改造（ReactUmi）
- 将图片列的 `src` 修改为代理地址：`/api/react/v1/image-proxy?url=<encodeURIComponent(remoteUrl)>`。
- 保留失败回退占位图；无需设置 `crossOrigin`。
