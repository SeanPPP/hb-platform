## 问题分析
- 当前实现在 `DetectSupplierItemAsync` 中对每个明细循环分别查询 `Product` 和 `StoreRetailPrice`，属于 N+1 次数据库访问，明细较多时易触发 30s 超时。

## 改造方案
- 一次性批量查询：
  1) 收集有效的 `itemNumbers`（去重、去空、Trim）。
  2) 批量查询商品：`Product.LocalSupplierCode == supplierCode && Product.ItemNumber IN itemNumbers && IsDeleted == false`，同时 select 出 `ItemNumber`、`ProductCode`、`ProductName`，并构建 `Dictionary<string, (code,name)>`，键为 `ItemNumber`。
  3) 收集命中商品的 `productCodes`，批量查询价格：`StoreRetailPrice.StoreCode == storeCode && ProductCode IN productCodes && IsDeleted == false`，select 出 `ProductCode`、`PurchasePrice`、`StoreRetailPriceValue`，并构建 `Dictionary<string, (purchase, retail)>`，键为 `ProductCode`。
  4) 按输入顺序生成 `SupplierItemDetectResult[]`：
     - 当 `itemNumber` 为空 → `Exists=false, Error="货号为空"`
     - 当 `itemNumber` 未命中商品 → `Exists=false`
     - 当命中商品 → `Exists=true`，并填充 `ProductCode`、`ProductName`、对应分店价格（价格可能为空）。

## 代码改动（示意）
- 替换 `foreach` 中的每条查询为两次批量查询：
```csharp
var items = (dto.Items ?? new List<DetectSupplierItem>())
    .Select(x => x?.ItemNumber?.Trim())
    .ToList();
var validItemNumbers = items.Where(s => !string.IsNullOrWhiteSpace(s))
    .Distinct()
    .ToList();

var products = await db.Queryable<Product>()
    .Where(p => p.LocalSupplierCode == dto.SupplierCode && validItemNumbers.Contains(p.ItemNumber) && p.IsDeleted == false)
    .Select(p => new { p.ItemNumber, p.ProductCode, p.ProductName })
    .ToListAsync();
var prodByItem = products.ToDictionary(x => x.ItemNumber, x => new { x.ProductCode, x.ProductName });

var productCodes = products.Select(x => x.ProductCode).Where(c => !string.IsNullOrEmpty(c)).Distinct().ToList();
var prices = await db.Queryable<StoreRetailPrice>()
    .Where(x => x.StoreCode == dto.StoreCode && productCodes.Contains(x.ProductCode) && x.IsDeleted == false)
    .Select(x => new { x.ProductCode, x.PurchasePrice, Retail = x.StoreRetailPriceValue })
    .ToListAsync();
var priceByCode = prices.ToDictionary(x => x.ProductCode, x => new { x.PurchasePrice, x.Retail });

var results = new List<SupplierItemDetectResult>(items.Count);
foreach (var raw in dto.Items ?? new List<DetectSupplierItem>())
{
    var itemNumber = raw?.ItemNumber?.Trim();
    if (string.IsNullOrWhiteSpace(itemNumber))
    {
        results.Add(new SupplierItemDetectResult { Exists = false, Error = "货号为空" });
        continue;
    }
    if (!prodByItem.TryGetValue(itemNumber, out var prod))
    {
        results.Add(new SupplierItemDetectResult { Exists = false });
        continue;
    }
    var price = prod.ProductCode != null && priceByCode.TryGetValue(prod.ProductCode, out var pr) ? pr : null;
    results.Add(new SupplierItemDetectResult
    {
        Exists = true,
        ProductCode = prod.ProductCode,
        ProductName = prod.ProductName,
        CurrentPurchasePrice = price?.PurchasePrice,
        CurrentRetailPrice = price?.Retail,
    });
}
return ApiResponse<List<SupplierItemDetectResult>>.OK(results);
```

## 注意事项
- 使用 `Contains(list)` 进行 `IN` 批量查询，确保列表不为空时再执行；空列表直接返回 `Exists=false` 的结果。
- 保持返回顺序与输入一致，前端索引渲染正确对应。
- 对 `itemNumber` 做 `Trim`，避免因空格导致未命中。

## 验证
- 明细在百~千行规模下，数据库访问固定为 2 次；接口响应显著加速，超时风险降低。
- 随机抽样比对逐条与批量结果一致性。

## 影响范围
- 仅改动 `DetectSupplierItemAsync` 方法的查询方式，不涉及 DTO/Controller 接口约定。
- 其他功能不受影响。
