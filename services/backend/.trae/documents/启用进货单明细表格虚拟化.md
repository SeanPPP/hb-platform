## 选型与原则
- 采用 `rc-virtual-list` 对 AntD Table 的 `tbody` 进行虚拟化，保持现有 Table API 与交互一致。
- 保留排序、筛选、行选择、固定列、横向滚动、操作列、图片单元格等功能；不使用分页。

## 技术设计
- 在 `VirtualTable.tsx` 增加虚拟化能力：新加 `virtual`（默认 true）、`rowHeight`（默认 48）、`overscan`（默认 20）。
- 使用 `components={{ body: VirtualBody }}` 替换表体；`VirtualBody` 内部用 `rc-virtual-list` 渲染 `rows`，只创建可视范围 DOM。
- 高度控制：继续沿用 `height` → 映射到 `scroll.y`；`rc-virtual-list` 以 `height` 为容器高度。
- 行键：强制依赖 `rowKey`，用于虚拟列表的 `itemKey`；无键时回退非虚拟模式。
- 固定列与横向滚动：保留 `columns.fixed` 与 `scroll.x`；验证在 `components.body` 下固定列正常（rc-table 会为固定列生成独立 `tbody`，组件替换对齐）。

## 主要改动点
1. 新增文件 `VirtualBody.tsx`：
   - 接口：`({ data, children, itemHeight, height, overscan }) => ReactNode`
   - 逻辑：用 `VirtualList` 包裹 `rows`，`row` 高度用 `itemHeight` 固定，支持 `overscan`。
2. 修改 `VirtualTable.tsx`：
   - 读取 `virtual/rowHeight/overscan/height`，构造 `components.body=VirtualBody`，并保证 `scroll={{ ... , y: height }}`。
   - 当 `virtual=false` 或缺少 `rowKey` 时，自动降级为原生 `Table`。
3. 明细页接入：`LocalSupplierInvoiceDetail` 保持列定义不变，仅传递 `rowKey`、`height`、`scroll.x`、`rowSelection`；可设置 `rowHeight`=48 与 `overscan`=20。

## 验证事项
- 数据量：构造 1万+ 行，测试滚动流畅度与内存占用。
- 交互：筛选/排序/选择/固定列/粘贴导入/批量操作/图片预览均验证。
- 边界：行高不一致的单元格（含 `Image`）统一使用固定高度，避免虚拟列表错位。
- 兼容：检验 `rowSpan/colSpan` 未使用；如出现固定列与虚拟体差异，针对固定列 `tbody` 同步应用 `VirtualBody`。

## 风险与回退
- 若固定列在虚拟化下存在布局问题，短期回退该页为普通滚动（关闭 `virtual`），保留现有体验。
- 如需极致性能，可在独立分支尝试 `react-window` 或迁移 `react-data-grid`/`ag-grid`。
