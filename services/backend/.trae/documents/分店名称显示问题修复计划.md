# 分店名称显示问题修复计划

用户反馈配货单显示的仍然是分店代码而不是名称。这说明 `storeName` 状态变量没有被正确赋值为分店名称，或者 `getStores` API 返回的数据结构与预期不符。

## 问题分析

在 `PickingList/index.tsx` 中：
```typescript
const storeRes = await getStores({ search: orderData.storeCode, page: 1, pageSize: 1 });
// @ts-ignore
if (storeRes.data?.items && storeRes.data.items.length > 0) {
    // @ts-ignore
    const storeInfo = storeRes.data.items[0];
    setStoreName(storeInfo.storeName);
} else {
    setStoreName(orderData.storeCode);
}
```

可能的原因：
1.  `getStores` 没有找到该分店。
2.  `getStores` 返回的数据结构不是 `data.items`。
3.  `storeInfo.storeName` 字段名不正确（可能是 `name` 或其他）。
4.  搜索逻辑问题（例如精确匹配 vs 模糊匹配）。

## 诊断与修复

我将采取以下步骤：
1.  **检查 `getStores` 的实现和返回类型**：查看 `storeService.ts` 确认数据结构。
2.  **调试/修正代码**：
    *   在 `PickingList/index.tsx` 中增加日志（可选，或者直接根据服务定义修正）。
    *   通常 `getStores` 返回的是 `PagedList`，包含 `items`。
    *   分店对象通常有 `storeName` 和 `storeCode`。
    *   如果 `getStores` 搜索不到（例如因为 search 是模糊匹配而代码太短匹配了多个？或者完全不匹配），则会回退到 `storeCode`。

假设 `getStores` 是标准的列表查询。如果它返回空，说明没找到。
我们将尝试使用 `getStoreByCode` (如果存在) 或者确保 `getStores` 能正确找到。

**备选方案**:
如果 `getStores` 不可靠，我们可以检查后端是否有提供分店信息的接口，或者在 `orderData` 中是否已经包含了分店名称（虽然通常只存 code）。
从 `OrderDetails/index.tsx` 的代码来看：
```typescript
const resp: any = await getUsedBranches();
const list: any[] = resp?.data?.data ?? resp?.data ?? resp ?? [];
const store = list.find((s: any) => s?.code === orderData.storeCode);
const name = store?.name ?? '';
```
这里使用了 `getUsedBranches` 来获取分店名称。这可能是一个更可靠的方法，或者 `getStores` 的用法需要调整。

**修正计划**:
我们将改用 `getUsedBranches` 来获取分店名称，就像 `OrderDetails` 页面那样，因为它专门用于获取订单相关的分店信息，且似乎在详情页工作正常。

## 实施步骤

1.  **修改 `PickingList/index.tsx`**:
    *   引入 `getUsedBranches`。
    *   替换 `getStores` 的调用逻辑，改用 `getUsedBranches` 查找匹配的分店。

    ```typescript
    import { getOrderDetail, StoreOrderCartDto, getUsedBranches } from '@/services/storeOrder'; // 注意 getUsedBranches 在 storeOrder 中定义
    // ...
    // 在 fetchData 中
    try {
        const branchRes = await getUsedBranches();
        // @ts-ignore
        const branches = branchRes.data || branchRes; 
        // 检查返回结构，通常是 List<BranchDto>
        const branch = branches.find((b: any) => b.code === orderData.storeCode);
        if (branch) {
            setStoreName(branch.name);
        } else {
            // 如果没找到，尝试 getStores 作为备选，或者直接显示 Code
            // 之前的 getStores 可能因为权限或参数问题失败
            setStoreName(orderData.storeCode);
        }
    } catch (e) { ... }
    ```

    *   **注意**: `getUsedBranches` 是获取 *Used* Branches，如果这个订单是新创建的且分店之前没下过单，可能不在列表里吗？后端实现是 `Select Distinct StoreCode from WareHouseOrder`，所以只要订单存在，就应该能查到。但如果订单刚创建，事务可能未提交？不，查询时订单已存在。
    *   不过，为了保险，保留 `getStores` 逻辑作为 fallback，或者修正 `getStores` 的使用。
    *   让我们先看看 `getStores` 为什么失败。可能是因为 `search` 参数。
    *   最稳妥的方式是：先试 `getUsedBranches`，如果不行，直接显示 Code。
