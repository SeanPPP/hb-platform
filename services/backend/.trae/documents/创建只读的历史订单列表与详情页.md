我将按照您的要求，基于 `src/layouts/StoreLayout.tsx` 布局来实现历史订单功能。

### 1. 创建页面文件
*   **创建目录**：`src/pages/StoreHistoryOrders`
*   **列表页 (`index.tsx`)**：
    *   开发适用于移动端的订单列表。
    *   展示关键信息：订单号、日期、状态、金额。
    *   点击行跳转至详情页。
*   **详情页 (`Detail.tsx`)**：
    *   **只读模式**：仅展示数据，没有任何编辑操作。
    *   包含：订单头部信息（状态、配送费、备注等）和商品明细列表。

### 2. 配置路由 (在 `StoreLayout` 下)
*   修改 `.umirc.ts`，将新路由添加在 `StoreLayout` 的 `routes` 数组中，确保新页面自动应用商店布局：
    ```typescript
    // 示意
    {
      component: '@/layouts/StoreLayout',
      routes: [
        { path: '/store', component: './StoreHome' },
        { path: '/store/orders', component: './StoreHistoryOrders' },         // 新增列表
        { path: '/store/orders/:id', component: './StoreHistoryOrders/Detail' } // 新增详情
      ]
    }
    ```

### 3. 集成入口
*   修改 `src/layouts/StoreLayout.tsx`，在现有的移动端导航结构中（如 Account 菜单或底部导航）添加跳转到 `/store/orders` 的链接。
