# 导航查询角色问题诊断指南

## 问题描述
在用户登录时，使用 `.Includes(u => u.Roles)` 进行导航查询，但角色数据没有被正确加载。

## 可能的原因

### 1. 数据库中没有角色数据
- 角色表（Role）为空
- 用户角色关联表（UserRole）为空
- 角色数据与用户数据不匹配

### 2. 导航属性配置问题
- SqlSugar导航属性配置不正确
- 外键关系不匹配
- 数据类型不一致

### 3. 数据库连接或权限问题
- 数据库连接失败
- 表不存在
- 权限不足

## 常见问题

### Q1: 角色数据存在但导航查询仍为空
**A**: 检查UserRole表中的关联数据是否正确，确保UserGUID和RoleGUID匹配。

### Q2: 初始化后仍无法查询到角色
**A**: 检查导航属性配置，确保User.cs中的Roles属性配置正确：

```csharp
[Navigate(typeof(UserRole), nameof(UserRole.UserGUID), nameof(UserRole.RoleGUID))]
public List<Role> Roles { get; set; } = new();
```

### Q3: 数据库表不存在
**A**: 确保数据库表已正确创建，可以重新运行数据库初始化：

```bash
# 强制重建数据库（会删除所有数据）
FORCE_RECREATE_DB=true dotnet run --project BlazorApp.Api
```
